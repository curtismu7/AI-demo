Design notes

  • One header for every page. The standalone banking-header band above the TopNav is gone (it was the dead-space block). The in-page dashboard-toolbar row is also gone — its controls live inside this header now.
  • Themed by manifest CSS vars. The gradient background, text color, and the active-link underline all read from the four vars at the top of :root. Each row above uses a real manifest's values via data-theme.
  • Context chip + nav. The chip (Customer / Admin / Setup) is always visible. The 3-tab nav replaces the existing admin-only Customer · Admin · Setup quick switcher and is now visible to admins on every page.
  • Dashboard-only controls auto-hide. Theme picker, layout toggle, HITL Controls, Reset Demo render only on /dashboard — they make no sense on Setup or generic admin pages. (Same conditional rule we used last turn, just inside this new bar.)
  • Token pill + user menu always render. Token pill states: green (≥ 5min), amber (< 5min), red (expired → Sign in). The user menu replaces the previous orphan "Log Out" button — all account actions live here.
  • Height. 60px on all routes. Mobile breakpoint at 1100px collapses nav labels to icons and hides the user name.
