Mode C · Notifier. FAB is not a chat surface at all — it's a system-tray-style result indicator. Most recent tool result, link to the Inspect tab for the full audit. Good if the dashboard already has the full agent and the FAB's job is "let me see the last thing without scrolling back." Worst for users who actually want to ask another question without scrolling.
Recommendation
Mode B (Teleport) for v1. It's the cheapest to build (no dual-state machine, no sync indicator), the easiest to explain ("the FAB takes you to the agent"), and degrades gracefully — on non-dashboard routes it becomes the existing popover behavior with no design change.

Mode A (Mirror) is genuinely useful if users are expected to keep the agent in view while scrolling down to read transactions or activity below the fold. Defer to v1.1 if usage data shows that's the actual behavior.

Mode C (Notifier) is the riskiest — it solves a different problem (notifications) using a control that users will read as "chat." Skip unless there's a strong product reason.

Why five tabs (not four, not three)
All is the default because a learning viewer wants the whole story chronologically — token ↔ MCP interleaved by timestamp tells the "what happened?" story better than two stacked silos. Tokens and MCP are filters on the same feed for viewers who already know what they want to inspect.

Standards turns the demo into an RFC index — every protocol that fired this round-trip is one click from its spec. Critical for technical demos where the audience asks "is this real RFC 8693?" — the Standards tab is the receipt.

Errors is the safety valve. When everything succeeds it's dim with `· 0`. When anything fails it lights up — rose for hard failures, gold for HITL pauses. The viewer never has to hunt for the failure card in a long feed.

Why these four refinements
Search turns the All tab from a passive feed into an investigation tool. A viewer can type "tools/call" to see only MCP, or paste a tx ID to scrub the feed for that one event. The chip row preserves the most-useful pre-built filters.
