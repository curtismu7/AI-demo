Accent (`#003366`) reserved for: page title, section headings, card titles, form labels, primary action buttons (`.pingone-test-button--primary`). Not for body text, badge text, or table data cells.

Fix button color (`#d32f2f`) is reserved exclusively for: "Fix: Create Banking Resource Server" and "Fix: Add Missing Scopes" buttons. No other element in the page uses this fill.

Primary visual anchor for the fix panel: the red-fill fix button(s) (`.pingone-test-button--fix`) which are the only elements using `#d32f2f` fill on the page.

## Interaction States

### Fix: Create Banking Resource Server button

| State | Visual |
|-------|--------|
| Default (shown only when banking RS missing) | `.pingone-test-button--fix` — red fill, white text, label "Fix: Create Banking Resource Server" |
| Loading / in-progress | Button disabled (`opacity: 0.55`), label "Creating…", no spinner (consistent with existing button disabled pattern) |
| Success | Button hidden; inline `.scope-fix-result` with green text "Banking resource server created. Re-running verification…" |
| Error | Button re-enabled; inline `.scope-fix-result` with red text showing error message + "Check worker app management API scopes in PingOne." |

### Fix: Add Missing Scopes button

| State | Visual |
|-------|--------|
| Default (shown only when RS exists but scopes missing) | `.pingone-test-button--fix` — red fill, white text, label "Fix: Add Missing Scopes" |
| Loading | Button disabled, label "Adding scopes…" |
| Success | Button hidden; inline result "N scope(s) added. Re-running verification…" in green |
| Error per-scope | Show which scopes failed inline (e.g., "banking:admin: already exists — skipped") |

### Auto re-run after fix

After any successful fix operation, the page automatically calls `verifyAssets()` again. No user action required. Show `notifyInfo('Re-running scope verification…')` toast during this re-run.

### Scopes tab — banking RS identification

| State | Visual |
|-------|--------|
| Banking RS found | Row has `.asset-table-row--banking-rs` highlighted left border (4px solid `#003366`) + label "(banking)" appended to name |
| Banking RS not found | Show inline note below table: "Could not identify banking resource server. Showing first resource server." in amber `.test-card-detail__fix` block |
| No resource servers | Tab shows empty state: "No resource servers found in PingOne." in `.asset-card-error` text color |

## Layout Contract

The fix UI integrates into the existing pingone-test page structure without restructuring:

```
.pingone-test-section (Asset Verification section)
  .asset-verification-header
    .asset-verification-description
    .asset-verification-actions
      [Re-run Verification button]       ← existing
  .asset-table-wrapper (AssetTable)
    .asset-tabs
      [Apps] [Resource Servers] [Scopes]  ← existing tabs
    [tab content per active tab]
  .scope-fix-panel                        ← NEW: shown only when scopes tab active + missing scopes
    h4 "Scope Alignment Issues Detected"
    p "The following canonical scopes are missing..."
    .asset-table (missing scopes list)    ← reuse existing table styles
    .test-card-actions
      [Fix: Create Banking Resource Server button] ← shown if RS missing
      [Fix: Add Missing Scopes button]             ← shown if RS exists but scopes missing
    .scope-fix-result                     ← shown after fix attempt
```

The fix panel is conditionally rendered: shown only when the Scopes tab is active AND `assetVerification.missingCanonicalScopes.length > 0`. Hidden when all canonical scopes are present.
