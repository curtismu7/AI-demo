# Phase 999.1 — UI Design Contract

> Visual and interaction contract for the two D-06 UI surfaces: the Token Chain transport pill and the read-only admin h2 inspector. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** This is a transport/backend phase. The ONLY UI surfaces are the two D-06 deliverables. Both REUSE existing visual patterns — no new design system, no new component family. This contract exists to lock label text, the 3 transport states, placement, empty/default states, and refresh behavior so the executor has zero design ambiguity. It does NOT introduce new tokens; it pins the contract to the project's already-shipped values.

### Transport pill — state → existing class mapping (LOCKED)

The pill is the SAME visual family as `StatusBadge`. Add a sibling `TransportPill` function; do NOT add new CSS classes — reuse the shipped `.tcd-badge--*` set:

| Transport state | Pill label text | Existing CSS class | Color result | When |
|-----------------|-----------------|--------------------|--------------|------|
| h2 negotiated (h2 / h2c) | `HTTP/2` | `tcd-badge--active` | green (#dcfce7→#bbf7d0, text #14532d) | h2c successfully negotiated against MCP server :8083 |
| WebSocket (default path) | `WebSocket` | `tcd-badge--exchanged` | slate/grey (#e2e8f0→#cbd5e1) | **DEFAULT / most common** — D-04: WS is the default demo transport |
| HTTP/1.1 (configured h2, negotiated down) | `HTTP/1.1` | `tcd-badge--skipped` | muted grey (var(--brand-light-gray), #666) | h2 was configured but ALPN fell back to h1 — visually de-emphasized to signal "degraded from intent" |
| Unknown / pending | `Transport pending` | `tcd-badge--waiting` | muted grey, no spinner | transport not yet resolved for this step / pre-flight; never blank |

Rationale for the WS=`exchanged` (neutral slate) vs h2=`active` (green) split: green communicates "the multiplexed path the presenter wants to showcase is live"; slate communicates "normal default, nothing wrong." h1-fallback uses the dimmest treatment (`--skipped`) so a presenter can instantly point and say "we asked for HTTP/2 but the connection negotiated down." This is the teaching contract — clarity over flourish.

**Pill label decision (Claude's Discretion D-06 resolved):** Use full protocol names `HTTP/2`, `WebSocket`, `HTTP/1.1` — NOT `h2` / `h2c` / `mux`. Rationale: a conference audience reads `HTTP/2` instantly; `h2` requires insider knowledge. The educational goal (D-06, PROJECT.md) wins. The internal `h2c` distinction is shown in the admin inspector, not the customer pill.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (inspector) | `Refresh` (plain — no emoji, matches `admin-vault-page__refresh` "Refresh status" precedent) |
| Pill: h2 state | `HTTP/2` |
| Pill: WS state | `WebSocket` |
| Pill: h1-fallback state | `HTTP/1.1` |
| Pill: unknown state | `Transport pending` |
| Inspector page title (`h1`) | `HTTP/2 Bridge Inspector` |
| Inspector intro (`p`, muted) | `Live view of the BFF→MCP HTTP/2 connection pool. HTTP/2 is opt-in: set MCP_SERVER_HTTP2_URL to enable it. When unset, the BFF uses the default WebSocket transport and this page shows no active sessions — that is expected.` |
| Empty state heading (no h2 sessions) | `No active HTTP/2 sessions` |
| Empty state body (no h2 sessions + next step) | `The BFF is using the default WebSocket transport (MCP_SERVER_HTTP2_URL is not set). To demo HTTP/2 multiplexing, set MCP_SERVER_HTTP2_URL=http://localhost:8083, restart the BFF, then trigger an MCP tool call from the agent.` |
| Fallback-event row label | `HTTP/1.1 fallback` (rendered in warning color; do NOT use a red/❌ glyph in the cell — color + text label only) |
| Fallback-event banner (if any fallback events present) | `⚠️ One or more HTTP/2 connections negotiated down to HTTP/1.1. The MCP server may not be speaking h2c on the configured port. Tool calls still work over HTTP/1.1, but without multiplexing.` |
| Error state (inspector fetch failed) | `❌ Could not load HTTP/2 pool state. The BFF may be unreachable or the inspector route is not mounted. Confirm the BFF is running and retry.` (matches `AdminVaultPage` `❌ ${message}` banner pattern) |
| Loading state | `Loading…` (verbatim from AdminVaultPage precedent) |
| Destructive confirmation | None — both surfaces are strictly READ-ONLY. No mutating actions, no confirmations. |

**Empty state is the PRIMARY case, not an afterthought (D-04).** Default demo runs WS, so the inspector's most-common rendering IS the empty state. It must read as "this is expected and here is how to turn h2 on," NOT as an error. The empty state copy is instructional (teaches the presenter the exact env var + restart + trigger sequence). It uses neutral/muted styling (`admin-vault-page__intro` color `#64748b`), never error red.

### Inspector pool table — column contract (LOCKED)

When ≥1 h2 session exists, render a table (reuse `admin-vault-page__status` list/section styling; a simple `<table>` inside the section is acceptable, 14px rows, `#f1f5f9` code-span background for monospace values):

| Column | Source | Rendering |
|--------|--------|-----------|
| Pool key | `http2McpBridge` pool Map key (`{url, 16-char token prefix}`) | monospace `<code>`; the 16-char prefix is safe to show (RESEARCH.md A7 — identical PingOne JWT header prefix, no leak). Do NOT render full tokens. |
| Negotiated protocol | `session.alpnProtocol` (`h2` / `http/1.1`) | text: `HTTP/2` (neutral) or `HTTP/1.1` (warning color row) |
| Active streams | pool entry `pendingStreams` | integer, right-aligned |
| Reuse count | pool entry connection-reuse counter | integer, right-aligned |
| Last used | pool entry `lastUsed` | localized relative/timestamp string |
| Status | derived | `Healthy` (h2, neutral) or `HTTP/1.1 fallback` (warning color) — text + color only, no glyph |

Fallback rows are distinguished from healthy h2 rows by the warning color treatment (`#fffbeb` row background / `#78350f` text, matching `admin-vault-page__rotate-hint`), NOT by an icon. A healthy h2 row uses the default white row.

### Inspector placement decision (Claude's Discretion D-06 resolved)

**New standalone admin route + page**, mirroring Phase 269.1 vault precedent exactly: one new route file (`/api/admin/h2`, gated by `authenticateToken` + admin/`requireAdmin` per RESEARCH.md V4), one mount line, one React page `AdminH2InspectorPage.jsx` under `<AdminRoute>`, added to `AdminSideNav` as a text-only entry (no emoji — sidebar label `HTTP/2 Bridge`, matching the text-only "Vault" sidebar precedent noted in REGRESSION_PLAN). NOT a sub-section of an existing page — keeps the surface scoped, isolates the new route's blast radius, and matches the most recent shipped admin precedent.

### Refresh behavior decision (Claude's Discretion D-06 resolved)

**Manual refresh button, static otherwise** — NO auto-polling. Mirrors `AdminVaultPage` exactly (mount-time fetch via `useEffect`, plus a "Refresh" button calling the same fetch). Rationale: (a) consistency with the established admin-page pattern the executor is already copying; (b) a presenter explicitly clicks "Refresh" after triggering a tool call, which is a clearer teaching beat than a silently mutating table; (c) zero risk of a polling timer adding load or noise to the demo. Optional/deferred: a future phase may add opt-in polling — out of scope here.

## Hard Constraints (executor MUST honor)

1. **REGRESSION_PLAN §0 — NO emoji** in any UI text except `⚠️ ✅ ❌`. The transport pill is plain text only (`HTTP/2` / `WebSocket` / `HTTP/1.1` / `Transport pending`) — never an emoji glyph or icon font. Only the inspector banners/error/loading copy may use `⚠️`/`❌` exactly as the AdminVaultPage precedent does.
2. **Reuse, do not invent.** Pill = sibling of `StatusBadge` using existing `.tcd-badge--*` classes (no new CSS classes). Inspector = clone of AdminVaultPage structure/CSS (`apiClient` only; no raw axios/fetch — token-custody rule).
3. **Read-only.** Neither surface mutates state. No POST, no confirmations, no destructive actions.
4. **Empty/default is primary.** D-04: WS is default → pill's most-common state is `WebSocket`, inspector's most-common render is the instructional empty state. Both must read as expected, never as failure.
5. **UI build gate.** `cd banking_api_ui && npm run build` must exit 0 after the `TokenChainDisplay.js` and new admin page edits (REGRESSION_PLAN §0 / CLAUDE.md non-negotiable #3).
6. **Educational clarity > visual flourish.** A presenter must be able to point at the pill and the inspector and explain HTTP/2 multiplexing without extra UI ornamentation.
