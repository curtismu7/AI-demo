<objective>
When a user opens Config → Token Validation and runs the health check while introspection is not configured, show a clear inline setup guide explaining exactly what env vars to set and how to get the values from PingOne — instead of just a hint string.

Also show a proactive banner when the validation mode is introspection but the health check hasn't been run yet, to surface the setup need immediately.
</objective>

<tasks>

## Task 1 — Add introspection setup guide to ConfigTokenValidation.tsx

**File:** `banking_api_ui/src/components/ConfigTokenValidation.tsx`

After the health status block that currently shows `'not_configured'` as "⚙ Not Configured" + hint, add an `IntrospectionSetupGuide` component (inline, no new file) that renders when `healthStatus.status === 'not_configured'`.

The guide renders three numbered steps:

**Step 1 — Find your introspection endpoint**
> PingOne provides the introspection endpoint at:
> `https://auth.pingone.com/{environment-id}/as/introspect`
> Find your Environment ID: PingOne Admin → Environments → (select env) → Overview → copy the ID from the URL or "Environment ID" field.

**Step 2 — Create a Worker Application in PingOne**
> 1. In PingOne Admin, go to Applications → Applications
> 2. Click ➕ Add Application → select Worker
> 3. Name it (e.g. "Banking Demo Introspection Worker")
> 4. Under Configuration, copy the Client ID and Client Secret
> 5. Enable the application

**Step 3 — Add to your .env file**
```
PINGONE_INTROSPECTION_ENDPOINT=https://auth.pingone.com/{env-id}/as/introspect
PINGONE_WORKER_CLIENT_ID=your-worker-client-id
PINGONE_WORKER_CLIENT_SECRET=your-worker-client-secret
```
Then restart the server.

**Also add a proactive banner**: If `validationMode === 'introspection'` and `healthStatus === null` (health check not yet run), show a soft info banner:
> "Introspection mode selected. Run "Test PingOne Connection" to verify your configuration, or see the setup steps below if you haven't configured the env vars yet."

Keep the banner brief — the full guide only appears after the health check returns `not_configured`.
