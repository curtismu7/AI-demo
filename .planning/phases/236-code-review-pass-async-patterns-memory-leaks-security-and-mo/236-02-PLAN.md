Read server.js in full. It is 1639 lines — read the entire file. Apply all four review dimensions:

1. ASYNC PATTERNS — focus areas:
   - `process.on('uncaughtException', ...)` and `process.on('unhandledRejection', ...)` handlers: Are they present? Do they log and exit cleanly or swallow errors?
   - Express route registrations using `async (req, res) => {...}` without try/catch: these will cause unhandled promise rejections in Express 4 (Express 5 adds auto-catch but this appears to be Express 4).
   - Any route handler that does `someAsyncFn()` without `await` and without `.catch()`.
   - `app.use()` middleware that is async but not wrapped: missing error propagation to `next(err)`.
   - Server startup sequence: `server.listen()` callback — does it `await` anything? Is the DB/session store ready before the server accepts connections?

2. MEMORY LEAKS — focus areas:
   - `setInterval` / `setTimeout` at module level: Is the handle stored? Is there a `SIGTERM` handler that calls `clearInterval`?
   - `process.on('exit', ...)` / `process.on('SIGTERM', ...)`: Is graceful shutdown wired, and does it clean up intervals and close the server?
   - EventEmitter `.on()` registrations on `process` or `app`: Do they accumulate on hot reloads in dev? (Less critical in prod but worth flagging.)
   - Any in-memory store (arrays, Maps) that grows without bound at server scope.

3. SECURITY — focus areas:
   - `helmet()` usage: Is it present? Are any helmet defaults disabled (e.g. `contentSecurityPolicy: false`)?
   - CORS configuration: Is `origin` a wildcard `*` for routes that carry credentials? Are credentials-bearing routes using `credentials: true` with a wildcard origin (not allowed by browsers but flag if present)?
   - Rate limiting: Is `express-rate-limit` applied to auth routes?
   - Any `req.query` or `req.body` fields used directly in object keys, file paths, or log messages without sanitization.
   - `console.log` statements that print full request objects, tokens, or session contents.
   - `eval()` or `new Function()` anywhere.

4. MODERN JS STANDARDS:
   - `var` declarations.
   - `.then().catch()` chains that could be async/await.
   - Repeated access to the same nested property without destructuring.
   - `require()` calls inside functions (lazy loading) — flag if not intentional.
