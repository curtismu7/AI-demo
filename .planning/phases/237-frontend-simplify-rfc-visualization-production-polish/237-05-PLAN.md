# Plan 237-05 — Build Verification & Backend Integration Test

## Objective

Verify that all Phase 237 work is correct with a running backend. This plan requires
a human to run the backend (`./run.sh start` or `./run-bank.sh`) and confirm the UI
flows work end-to-end. The executor should fix any issues found before marking done.

## Pre-conditions

- Plans 237-01 through 237-04 are complete
- Backend is startable via `./run.sh` or `./run-bank.sh`

## Tasks

### T1 — Final build check
```bash
cd banking_api_ui && npm run build
```
Exit code must be 0. Fix any errors before proceeding.

### T2 — RFC link spot-check
Open the UI and verify these links open the correct external URLs:
- [ ] RFC 8693 panel → https://www.rfc-editor.org/rfc/rfc8693
- [ ] RFC 9728 panel → https://www.rfc-editor.org/rfc/rfc9728
- [ ] RFC 7521 panel → https://www.rfc-editor.org/rfc/rfc7521
- [ ] MCP panel → https://spec.modelcontextprotocol.io/specification/2025-11-25/
- [ ] Token chain exchange arrow → RFC 8693 link

### T3 — Token chain live flow (2-Token Exchange)
With backend running and a user logged in:
- [ ] Trigger an agent tool call (e.g. "what's my balance?")
- [ ] Token chain updates: shows User Token → GW Token → Backend Token
- [ ] Each hop shows its audience value
- [ ] Exchange arrows show "RFC 8693" badge
- [ ] Claims panel shows `act` claim with annotation on the GW/backend tokens

### T4 — ID Token 2-Token Exchange flow
- [ ] Enable `ff_id_token_exchange` feature flag via `/feature-flags`
- [ ] Trigger an agent call
- [ ] Token chain header shows "ID Token 2-Token Exchange Flow"
- [ ] Claims panel correctly shows id_token claims at Hop 0

### T5 — RFC 9728 live metadata
With backend running:
- [ ] Open RFC 9728 education panel
- [ ] Click "Fetch Live Metadata"
- [ ] BFF metadata appears with `resource`, `authorization_servers` fields
- [ ] Status badge shows green for reachable services, yellow/red for unreachable

### T6 — Exchange flow name consistency check
Search in the rendered UI (not source) for any remaining "2-Exchange", "two-token",
or "Two Token Exchange" text that was not renamed. Fix if found.

### T7 — REGRESSION_PLAN.md update
Add an entry to REGRESSION_PLAN.md §4 (Bug Fix Log) covering:
- CSS variable rename (--chase-* → --brand-*)
- RFC link additions
- Token chain RFC annotations
- Exchange flow canonical naming

## Acceptance

- `npm run build` exits 0
- All RFC links verified working
- Token chain shows RFC 8693 annotations and audience hops in a live session
- RFC 9728 live fetch returns real data from the running BFF
- REGRESSION_PLAN.md updated
