# Plan 237-02 — RFC Education Panels: Links, Naming, JWT Examples

## Objective

Make every RFC reference in the UI clickable and educational. Add decoded JWT examples
at each exchange hop. Rename exchange flows to their canonical names everywhere.

## RFC link constants

Add a single source-of-truth file `src/config/rfcLinks.js`:

```js
export const RFC_LINKS = {
  RFC_8693:   { label: 'RFC 8693', url: 'https://www.rfc-editor.org/rfc/rfc8693',   title: 'OAuth 2.0 Token Exchange' },
  RFC_9728:   { label: 'RFC 9728', url: 'https://www.rfc-editor.org/rfc/rfc9728',   title: 'OAuth 2.0 Protected Resource Metadata' },
  RFC_7521:   { label: 'RFC 7521', url: 'https://www.rfc-editor.org/rfc/rfc7521',   title: 'Assertion Framework for OAuth 2.0' },
  RFC_7636:   { label: 'RFC 7636', url: 'https://www.rfc-editor.org/rfc/rfc7636',   title: 'Proof Key for Code Exchange (PKCE)' },
  RFC_6749:   { label: 'RFC 6749', url: 'https://www.rfc-editor.org/rfc/rfc6749',   title: 'OAuth 2.0 Authorization Framework' },
  MCP_SPEC:   { label: 'MCP 2025-11-25', url: 'https://spec.modelcontextprotocol.io/specification/2025-11-25/', title: 'Model Context Protocol Specification' },
};
```

Add a reusable `<RfcLink rfc="RFC_8693" />` component in `src/components/shared/RfcLink.js`
that renders as `<a href={url} target="_blank" rel="noopener noreferrer">{label}</a>`
with a small external-link icon.

## Tasks

### T1 — Create rfcLinks.js + RfcLink component

### T2 — RFC 8693 Panel (`education/RFC8693Panel.js`)
- Replace raw "RFC 8693" text strings with `<RfcLink rfc="RFC_8693" />`
- Add a "Two Exchange Hops" tab with:
  - **Hop 0** — User token (what the browser session holds):
    ```json
    { "sub": "user-abc123", "aud": "olb-resource.bxf.com",
      "scope": "banking:read banking:write", "iss": "https://auth.pingone.com/..." }
    ```
  - **Hop 1 (2-Token Exchange)** — GW-scoped delegated token after Exchange #1:
    ```json
    { "sub": "user-abc123", "act": { "sub": "agent1-client-id" },
      "aud": "https://mcp-gw.bxf.com", "scope": "banking:read banking:write ai_agent" }
    ```
  - **Hop 2** — Backend-scoped token after Exchange #2:
    ```json
    { "sub": "user-abc123", "act": { "sub": "agent1-client-id" },
      "aud": "https://mcp-olb.bxf.com", "scope": "banking:read banking:write" }
    ```
  - Each hop shows which claims changed and why (aud narrowed, act added/preserved)
  - Label: "Example payloads — your live tokens appear in the Token Chain panel above"

- `ExchangeModeToggle.js`: update toggle label copy to match canonical names

### T4 — RFC 7521 Panel (`education/JwtClientAuthPanel.js`)
- Add private_key_jwt assertion example with annotated fields:
  ```json
  {
    "iss": "agent1-client-id",      // client_id — who is authenticating
    "sub": "agent1-client-id",      // same as iss for client auth
    "aud": "https://auth.../token", // token endpoint URL
    "jti": "unique-request-id",     // one-time use — prevents replay
    "iat": 1714000000,              // issued at (Unix)
    "exp": 1714000300               // expires in 5 min (max recommended)
  }
  ```
- Add link to RFC 7521 §4.2 and OIDC Core §9
