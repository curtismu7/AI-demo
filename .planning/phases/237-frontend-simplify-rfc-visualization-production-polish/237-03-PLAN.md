# Plan 237-03 — Token Chain: Live RFC Annotations & Hop Labels

## Objective

`TokenChainDisplay.js` is the live window into what tokens exist during a session.
It should tell a developer exactly which RFC governs each step, what the audience is
at each hop, and where the `act` claim comes from.

## Current state

- 1306-line component — does work but has no RFC context at the exchange arrows
- `idTokenMode` prop controls display but the label is generic
- Claims panel highlights `act` but does not explain the RFC context
- No links from the chain view to the spec

## Tasks

### T1 — RFC badge component
Create `src/components/shared/RfcBadge.js`:
- Small pill badge: `[RFC 8693]` that links to the spec section
- Props: `rfc` (key from `rfcLinks.js`), `section` (optional, e.g. `"§4"`)
- Renders: `<a href="..." ...>RFC 8693 §4</a>` styled as a monospace badge

### T2 — Exchange arrow annotations in TokenChainDisplay
Between each token node in the chain, the exchange arrow already exists.
Add to each arrow:
- RFC badge for the exchange (`[RFC 8693]`)
- Label showing the exchange type: "2-Token Exchange" or "ID Token 2-Token Exchange"
- The `audience` the resulting token is scoped to (e.g. `→ aud: mcp-gw.ping.demo`)

Structure (3 arrows total in full agent flow):
```
[User Token]
  aud: olb-resource.ping.demo
      ↓  RFC 8693 — 2-Token Exchange #1
         → aud: mcp-gw.ping.demo
[GW Delegated Token]
  sub: user-abc  act: {sub: agent1}
      ↓  RFC 8693 — Exchange #2 (Gateway re-exchange)
         → aud: mcp-olb.ping.demo
[OLB Backend Token]
  sub: user-abc  act: {sub: agent1}
```

### T3 — Claims panel improvements
In the `ClaimsPanel` sub-component:
- `aud` claim row: add tooltip "Audience — which resource server accepts this token (RFC 8693 §3)"
  with `<RfcBadge rfc="RFC_8693" section="§3" />`
- `act` claim row: add tooltip "Delegation chain — which agent is acting on behalf of the user (RFC 8693 §4.1)"
  with `<RfcBadge rfc="RFC_8693" section="§4.1" />`
- `sub` claim row: add tooltip "Subject — the human user this token represents"
- `scope` claim row: add tooltip "Authorized scopes for this audience"
- Show `act` chain inline: `sub: user-abc → act: { sub: agent1-client }` as a visual delegation arrow

### T4 — Flow name in chain header
The chain panel header currently shows a generic title.
- When `idTokenMode=false`: show "2-Token Exchange Flow"
- When `idTokenMode=true`: show "ID Token 2-Token Exchange Flow"
- Add a small `[?]` info icon that opens the RFC 8693 panel

### T5 — Build check
- `cd banking_api_ui && npm run build` → exit 0

## Acceptance

- Each exchange arrow in the token chain has an RFC 8693 badge with a working link
- Each exchange arrow shows the target audience value
- `act` claim in the claims panel has RFC annotation and tooltip
- Chain header uses canonical flow names
- `npm run build` exits 0
