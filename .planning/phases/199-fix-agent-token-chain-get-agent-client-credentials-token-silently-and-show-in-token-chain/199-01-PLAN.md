<objective>
Proactively fetch the agent client-credentials token on the BFF and expose it as a decoded token event so the Token Chain panel can show the "agent actor token" step before any MCP tool call fires.

Purpose: Currently the agent CC token only appears in the token chain AFTER an MCP 401 triggers RFC 8693 exchange. Users asking "show my accounts" see only the user token in the chain — the agent identity is invisible until first tool use. This adds a silent prefetch so the full delegation chain is visible immediately.
Output: New BFF endpoint + UI prefetch calling it on TokenChainPanel mount.
</objective>

Add a new route GET /api/tokens/agent-cc-preview to banking_api_server/routes/tokens.js.

Place it after the existing /session-preview route.

The route should:
1. Check if AGENT_OAUTH_CLIENT_ID is configured (configStore.getEffective('pingone_mcp_token_exchanger_client_id') || process.env.AGENT_OAUTH_CLIENT_ID). If not set, return a "not configured" placeholder event so the UI can show a helpful hint:
   ```json
   { "tokenEvents": [{ "id": "agent-cc-not-configured", "label": "Agent Actor Token (CC) — Not Configured", "status": "skipped", "decoded": null, "explanation": "AGENT_OAUTH_CLIENT_ID is not set. Configure PingOne MCP Token Exchanger credentials in Admin → Config to enable dual-token exchange.", "rfc": "RFC 8693 §2.1" }] }
   ```
2. If configured: call `await oauthService.getMcpExchangerToken()` — wrap in try/catch.
3. On success: decode with `decodeJwtClaims(token)` from agentMcpTokenService. Return:
   ```json
   { "tokenEvents": [{ "id": "agent-actor-token-prefetch", "label": "Agent Actor Token (CC) — prefetched", "status": "active", "decoded": { ...decodedClaims }, "explanation": "Client-credentials token for the AI Agent (${clientId}). This token is used as the actor_token in RFC 8693 Token Exchange when a banking tool is invoked via MCP. It carries the agent's identity — the resulting MCP access token will have act.client_id proving which agent performed the delegation.", "rfc": "RFC 8693 §2.1" }] }
   ```

4. On fetch error (PingOne 401/400 etc): return a "failed" status event with the error message so the UI shows a diagnostic rather than crashing:
   ```json
   { "tokenEvents": [{ "id": "agent-actor-token-prefetch", "label": "Agent Actor Token (CC) — fetch failed", "status": "failed", "decoded": null, "explanation": "Could not fetch agent CC token: <err.message>. Check AGENT_OAUTH_CLIENT_SECRET and PingOne app config.", "rfc": "RFC 8693 §2.1" }] }
   ```
