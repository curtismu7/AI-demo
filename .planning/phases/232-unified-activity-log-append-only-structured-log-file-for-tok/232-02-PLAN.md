<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authorize route → appEventService | Transaction type/amount/userId from request body written to log metadata |
| agent message → appEventService | User message snippet (first 120 chars) written to log metadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-232-05 | Information Disclosure | authorize metadata.amount + userId | accept | Same data already returned in JSON response to admin; no new disclosure boundary crossed |
| T-232-06 | Information Disclosure | agent_prompt message slice | mitigate | Log only first 120 chars of user message (`String(message).slice(0, 120)`); full prompt not persisted; no tokens or credentials in metadata |
| T-232-07 | Tampering | authorize decision logged | accept | logEvent is observability only; no access-control decision is based on the logged value; gate result is determined by evaluateTransaction before logEvent is called |
| T-232-08 | Spoofing | userId in metadata | accept | userId comes from req.user (set by authenticateToken middleware) or session — same trust level as the route itself |
</threat_model>
