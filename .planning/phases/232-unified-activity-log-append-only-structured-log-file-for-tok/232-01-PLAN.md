<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| logEvent() → filesystem | Event data written to local file; no external network call |
| ACTIVITY_LOG_FILE env var | Operator-controlled path; must not escape server process's working directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-232-01 | Tampering | ACTIVITY_LOG_FILE path | mitigate | `path.resolve()` normalises the path; the file is written server-side only; no user-supplied input reaches this value |
| T-232-02 | Denial of Service | fs.appendFileSync in hot path | accept | Demo context, low traffic; non-fatal try/catch ensures one slow disk cannot block event logging; production would use async write |
| T-232-03 | Information Disclosure | NDJSON log file | accept | Log file contains the same data as the in-memory ring buffer already exposed to admin UI; no raw tokens are written (metadata only, per existing logEvent contract) |
| T-232-04 | Spoofing | category string values | accept | Categories are string constants checked by UI for display only; no access control decision is based on category value |
</threat_model>
