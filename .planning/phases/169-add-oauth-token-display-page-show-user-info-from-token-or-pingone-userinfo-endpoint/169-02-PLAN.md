  <action>
Create or update banking_api_server/routes/oauth.js with a new route:

**Route:** `POST /api/oauth/userinfo`

**What it does:**
1. Extract the user's OAuth token from the request (from Authorization header or session)
2. Call PingOne userinfo endpoint using the token:
   - Endpoint: `https://auth.pingone.com/{environment_id}/as/userinfo`
   - Method: POST with header: `Authorization: Bearer {token}`
   - Returns: User profile (emails, names, phone, preferences, etc.)

3. Return merged/enriched data to frontend:
   ```json
   {
     "source": "PingOne userinfo",
     "data": {
       "sub": "...",
       "email": "...",
       "given_name": "...",
       "family_name": "...",
       "phone_number": "...",
       "locale": "...",
       "email_verified": true,
       ...
     },
     "timestamp": "2026-04-16T..."
   }
   ```

**Error handling:**
- If token invalid/expired: return 401 with { error: "Invalid or expired token" }
- If PingOne unreachable: return { source: "PingOne userinfo", error: "Service unavailable", statusCode: 503 }
- Frontend will gracefully show JWT data only if this fails

**Security:**
- Extract token from session (server-side, not client token string)
- No token leakage in logs
- Use existing JWT validation from app's oauth service
- Restrict to authenticated users only (check session)

**Example implementation pattern:**
```javascript
router.post('/api/oauth/userinfo', authenticateSession, async (req, res) => {
  try {
    const token = req.session?.access_token || req.headers.authorization?.replace('Bearer ', '');
    if (!token) return res.status(401).json({ error: 'No token' });

    const response = await axios.post(
      `https://auth.pingone.com/${process.env.P1_ENV_ID}/as/userinfo`,
      {},
      { headers: { Authorization: `Bearer ${token}` } }
    );

    res.json({
      source: 'PingOne userinfo',
      data: response.data,
      timestamp: new Date().toISOString()
    });
  } catch (err) {
    res.status(err.response?.status || 500).json({
      source: 'PingOne userinfo',
      error: err.message,
      statusCode: err.response?.status
    });
  }
});
```

**Export function:**
```javascript
export async function fetchEnrichedUserInfo() {
  try {
    const response = await fetch('/api/oauth/userinfo', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      credentials: 'include' // Include session cookies
    });

    if (!response.ok) {
      return {
        source: 'PingOne userinfo',
        error: `HTTP ${response.status}`,
        data: null
      };
    }

    return await response.json();
  } catch (err) {
    return {
      source: 'PingOne userinfo',
      error: err.message,
      data: null
    };
  }
}
```

**Requirements:**
- Service is optional (frontend can work without it)
- Graceful error handling (returns structured error, not thrown exception)
- Includes credentials: 'include' so Cookie/session is sent to BFF
- Returns { source, data, error?, timestamp? } structure
- Does NOT throw exceptions (OAuthTokenDisplayPage handles null data gracefully)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Token ↔ BFF | BFF holds Bearer token, calls PingOne userinfo endpoint |
| BFF ↔ Frontend | BFF returns enriched user data (not raw token) |
| BFF ↔ PingOne | Bearer token used for Authentication to PingOne service |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-169-04 | Information Disclosure | BFF userinfo route | Mitigate | Token held server-side only; never returned to client |
| T-169-05 | Man-in-the-Middle | BFF → PingOne | Mitigate | Use HTTPS (PingOne endpoint is HTTPS only) |
| T-169-06 | Spoofing | Enriched data display | Mitigate | Display "from PingOne" label; show timestamp |
| T-169-07 | Denial of Service | userinfo endpoint call | Mitigate | Add timeout (5s), rate limit client requests, graceful fallback |
</threat_model>
