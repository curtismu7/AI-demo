  <action>
Create a new React component `OAuthTokenDisplayPage.tsx` that:

1. **Retrieves the current user's OAuth token** — Use existing authService pattern to fetch token from session/localStorage

2. **Decodes JWT claims** (if not already decoded) — Use `jose` library or manual Base64 decoding to extract payload
   - Do NOT leave raw token exposed
   - Extract these key fields: `sub`, `iss`, `aud`, `scope`, `exp`, `iat`, `given_name`, `family_name`, `email`, `preferred_username`

3. **Groups information logically into sections** — Use existing app card/grid patterns (similar to PingOneTestPage):
   - **Identity & Profile:** username, email, given_name, family_name, subject (sub)
   - **Authorization:** scope (split by space), audience, environment, organization
   - **Validity:** issued_at (iat), expires_at (exp), time remaining
   - **Provider:** issuer (iss), key_id, algorithm

4. **Visual layout:**
   - Header: "OAuth Token Information" with success indicator
   - Use Tailwind grid or card layout (match existing admin pages style)
   - Each section in a distinct card/panel
   - Friendly labels (not raw JWT field names)
   - Render expiry time in human readable format (e.g., "Expires in 59 minutes 12 seconds")

5. **Error handling:**
   - If no token found: Show "No active OAuth session" message
   - If token expired: Show expiry warning
   - If token invalid: Show error badge

6. **NOT included in this task** (those are Wave 2):
   - Do NOT call PingOne userinfo endpoint yet
   - Do NOT show admin-only data enrichment
   - Do NOT add edit/refresh buttons

**Example structure:**
```jsx
const OAuthTokenDisplayPage = () => {
  const tokenData = getTokenClaims(); // From authService
  
  return (
    <div className="p-6">
      <h1>OAuth Token Information</h1>
      
      <section className="grid gap-4">
        <Card title="Identity">
          {tokenData.given_name} {tokenData.family_name}
          Email: {tokenData.email}
          Subject: {tokenData.sub}
        </Card>
        
        <Card title="Authorization">
          Scopes: {tokenData.scope.split(' ').map(...)}
          Audience: {tokenData.aud}
        </Card>
        
        <Card title="Token Validity">
          Issued: {formatDate(tokenData.iat)}
          Expires: {formatDate(tokenData.exp)}
          Remaining: {calculateTimeRemaining(tokenData.exp)}
        </Card>
      </section>
    </div>
  );
};
```

Use existing Tailwind classes from the app. No CSS modifications needed.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Token ↔ Frontend | Token claims displayed; raw token never shown to user |
| Frontend ↔ User | User views only their own token (session-authenticated) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-169-01 | Information Disclosure | OAuthTokenDisplayPage | Mitigate | Never display raw JWT token; only show decoded claims |
| T-169-02 | Spoofing | Token display | Accept | Page displays what's in user's own token; no external data sources |
| T-169-03 | Denial of Service | Token parsing | Mitigate | Handle malformed tokens gracefully; show error state |
</threat_model>
