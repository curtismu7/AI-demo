<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authorization header → Token decode | Bank API receives Bearer token, decodes to JWT |
| JWT claims → Authorization decision | JWT must include `act` claim for agent requests |
| Act claim → Audit log | Actor identity (act.sub) logged for accountability |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-10 | Spoofing | Token act claim | mitigate | Token created by PingOne, act claim added by TokenExchangeService (not user-settable). Backend validates act claim is present. |
| T-172-11 | Tampering | Act claim in token | mitigate | JWT signature verified by token decode (existing infrastructure assumed). Token cannot be modified without invalidating signature. |
| T-172-12 | Repudiation | Agent requests | mitigate | Act.sub logged (Task 2), audit trail captures which agent performed which action. |
| T-172-13 | Information Disclosure | Error messages | mitigate | Error messages describe validation failure ("missing 'act' claim") without leaking internal structure. Generic 401 for malformed tokens. |
| T-172-14 | Denial of Service | Validation logic | mitigate | Validation is fast JWT decode + claim check. No unbounded loops or external calls in validation path. |
| T-172-15 | Elevation of Privilege | Accepting pass-through | mitigate | Hard fail (D-04) — pass-through tokens (without act) are rejected. User cannot bypass delegation by omitting act claim. |

</threat_model>
