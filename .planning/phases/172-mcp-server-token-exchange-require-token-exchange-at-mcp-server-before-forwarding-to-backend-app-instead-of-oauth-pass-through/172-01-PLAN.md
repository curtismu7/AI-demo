## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cache → TokenExchangeService | Cache passes userId/scopeSet to exchange service; service returns token with `act` claim |
| Cache → BankingToolProvider | Provider requests scopes via getScopesForTool(), checks cache for token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-01 | Tampering | tokenCacheService.ts | mitigate | Cache entries include expiresAt; expired tokens never returned. Cache is in-memory, not serialized. |
| T-172-02 | Information Disclosure | toolScopeMap.ts | accept | Scope mapping is config, not secret. Narrowed scopes limit blast radius of token compromise. |
| T-172-03 | Denial of Service | tokenCacheService.ts | mitigate | No cache size limit in Task 1; Plan 02 should add max entries or LRU eviction. |

</threat_model>
