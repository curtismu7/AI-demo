<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| BankingToolProvider → TokenExchangeService | Passes user token to service; expects delegation token with `act` claim back |
| BankingToolProvider → BankingAPIClient | Token contains `act` claim; no modification |
| BankingAPIClient → Backend API | Bearer token in Authorization header; backend validates `act` claim |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-172-04 | Tampering | BankingToolProvider token caching | mitigate | Cache keys include userId; cache never returns token for different user. Expiry enforced (Task 1). |
| T-172-05 | Spoofing | TokenExchangeService exchange | mitigate | Token exchange to PingOne via HTTPS. PingOne returns act claim; not settable by user. Backend validates act (Plan 03). |
| T-172-06 | Repudiation | Token exchange | mitigate | Structured logging (Task 3) logs all exchange attempts with userId, toolName, success/failure. |
| T-172-07 | Information Disclosure | Cached tokens | mitigate | Cache is in-memory, not persisted. Expires after token lifetime. Hard fail prevents leaking error details to user (D-04). |
| T-172-08 | Denial of Service | tokenCache.set() unbounded | mitigate | Plan 02 does not add cache size limit (deferred to Plan 04 if needed). Current risk: one user with many scope combinations could fill memory. |
| T-172-09 | Elevation of Privilege | Token exchange scopes | mitigate | Tool-to-scope map is immutable config (toolScopeMap.ts). User cannot request additional scopes (least-privilege per D-03). |

</threat_model>
