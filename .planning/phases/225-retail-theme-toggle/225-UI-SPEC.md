# Phase 225 — UI Design Contract

> Visual and interaction contract for the retail-theme-toggle phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none — hand-rolled CSS variables |
| Preset | not applicable — IndustryBrandingContext + industryPresets.js |
| Component library | none — custom CSS classes on HTML/React elements |
| Icon library | none — text labels only (no emoji in production UI per CLAUDE.md) |
| Font | System font stack: `-apple-system, BlinkMacSystemFont, "Segoe UI", "Roboto", sans-serif` (from `--font-primary` in index.css) |

**Design system note (source: RESEARCH.md + index.css):** Theming is applied at runtime by
`IndustryBrandingContext.applyIndustryId("retail")` which sets CSS custom properties on
`document.documentElement`. The retail preset is the third entry in `industryPresets.js`. No
component library, no Tailwind, no shadcn. New CSS classes follow the `ud-` BEM-style prefix
convention established in `UserDashboard.css`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, stock-status pill padding |
| sm | 8px | Product name to price gap, card button internal padding |
| md | 16px | Default inter-element spacing, form group margin, card inner padding (inherited) |
| xl | 24px | Section heading margin-bottom, cart summary section padding |
| 2xl | 32px | Gap between product grid and orders section |
| 3xl | 48px | Major section breaks (not expected at current scope) |

Exceptions:

- Banner height: 44px — minimum touch target for the toggle button per WCAG 2.5.8

**Product card inner padding** inherits from the existing `.ud-account-card` rule (20px in
`index.css` via `--card-padding`) — this phase does not redeclare it. The executor will reuse
the existing CSS class, not write a new padding rule.

**Product card grid gap** inherits from the existing `.ud-accounts-row` rule (14px in
`UserDashboard.css` line 2944) — this phase does not redeclare or change it. Frozen inherited
values that the executor will not write do not belong in the spacing contract.

## Typography

All values are pre-defined in `:root` (index.css) and are reused for retail mode without new
font definitions. Source: index.css lines 66–74.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (`--font-size-body`) | 400 | 1.6 (`--line-height-body`) | Product descriptions, stock labels, order dates, cart subtotal label |
| Label | 12px | 600 | 1.4 | Stock-status badge text, SKU, category pill, "Add to Cart" button text |
| Heading (card) | 18px (`--font-size-h3`) | 600 | 1.2 | Product name on card, order item name |
| Display (price) | 22px (`--font-size-h2`) | 600 | 1.1 | Product price, cart total amount |

Note on "Add to Cart" button: uses Label size (12px, weight 600) to match existing `.ud-card-btn`
class (UserDashboard.css line 2957: `font-size: 0.72rem; font-weight: 600`).

**12px vs 14px distinction:** 12px is strictly for badges, pills, and SKU identifiers — compact
metadata elements where visual density is expected. 14px is used for all running text, including
product descriptions, stock labels, order dates, and subtotal labels. Do not use 12px for any
sentence-form copy.

## Color

### Retail Preset Palette (source: CONTEXT.md decisions + RESEARCH.md Pattern 3)

| Role | Value | CSS Variable | Usage |
|------|-------|-------------|-------|
| Dominant (60%) | `#FFFFFF` | `--chase-white` (unchanged) | Page background, card body surfaces, hero background |
| Secondary (30%) | `#0046BE` | `--app-primary-red` (overridden by retail preset) | Dashboard header gradient, nav bar background, product card header strip |
| Accent (10%) | `#FFE000` | not a named root var — retail-only accent | See "Accent reserved for" below |
| Destructive | `#dc2626` | `--app-primary-red-mid` (banking value; unchanged in retail mode) | Not used in retail scope; reserved for existing error flows |

**Accent reserved for:** cart total badge background, "Add to Cart" call-to-action button
background on the primary/featured product card only, mode-toggle banner active-state indicator.
Accent (`#FFE000`) is NOT used for body text, general borders, or hover states. Text on
`#FFE000` backgrounds must be `#000000` (black) for WCAG AA contrast (ratio 19.6:1).

## Product Card Layout Contract

Primary focal point of the retail dashboard: the Cart Summary card at top-center, followed by
the product grid.

Each product card extends `.ud-account-card` with modifier `.retail-product-card`:

```
┌─────────────────────────────┐
│ [Category pill]    [Stock badge] │  ← 8px gap between, 8px top padding
│                                  │
│  Product Name                    │  ← 18px, weight 600
│  SKU: BB-XXXXX                   │  ← 12px, weight 400, color #666666
│                                  │
│  $1,299                          │  ← 22px, weight 600, color #0046BE
│                                  │
│  [Add to Cart]                   │  ← .ud-card-btn (12px, weight 600)
└─────────────────────────────┘
```

Card background: white (`#FFFFFF`) with `box-shadow: 0 4px 14px rgba(0,0,0,0.15)` (same as
`.ud-account-card`). Border-radius: 14px (same as `.ud-account-card`). Padding: 20px (matches
`--card-padding` from index.css).

The "Add to Cart" button background for the featured/primary product card (first in list) uses
accent yellow `#FFE000` with text `#000000`. All other cards use `--app-primary-red` (#0046BE)
background with white text — matching the `.ud-card-btn` glass-morphism pattern for non-primary.

## RetailModeBanner Layout Contract

Positioned: top of `UserDashboard` main content area, directly below TopNav, full-width.
Height: 44px (minimum touch target). Padding: 0 16px.
Background: retail-ON state = `#0046BE`; banking state = `#f5f5f5` (`--chase-light-gray`).

```
┌──────────────────────────────────────────────────────┐
│  Retail Mode — BX Electronics    [Switch to Banking]  │  ← retail ON (blue bg, white text)
└──────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────┐
│  Banking Mode — BX Finance       [Switch to Retail]   │  ← banking (gray bg, dark text)
└──────────────────────────────────────────────────────┘
```

Toggle button: `border-radius: 6px`, padding `6px 12px`, font-size 13px, weight 600.
Retail-ON: button background `#FFE000`, text `#000000`.
Banking state: button background `var(--chase-navy)`, text `white`.

The banner is always visible when the user is on the Dashboard (both modes). It is not shown
on Admin, Config, Education, or Marketing pages.

## Interaction States

### "Add to Cart" button

| State | Visual |
|-------|--------|
| Default | Blue fill `#0046BE`, white text (or yellow `#FFE000`/black for primary card) |
| Hover | `--app-primary-red-hover: #003ca6` (blue darkens by ~10%) |
| Pressed | `opacity: 0.85` |
| In cart (already added) | Label changes to "In Cart", background `#166534` (green), white text |
| Disabled (out of stock) | `opacity: 0.4`, `cursor: not-allowed` |

### Banner toggle button

| State | Visual |
|-------|--------|
| Default (retail ON) | Yellow `#FFE000`, black text |
| Hover (retail ON) | `#e6ca00` (yellow -5%), black text |
| Default (banking) | Navy `var(--chase-navy)`, white text |
| Hover (banking) | `#003770` (navy hover, from index.css `--btn-primary-bg-hover`), white text |
| Focus | `outline: 2px solid #0046BE; outline-offset: 2px` |

### Mode transition

When the banner toggle is clicked:

1. `PATCH /api/admin/feature-flags { updates: { ff_retail_mode: !current } }` fires
2. On success: `applyIndustryId("retail" | "bx_finance")` + `setRetailMode(!current)`
3. No page reload — DOM CSS vars update synchronously via IndustryBrandingContext
4. No loading spinner required (operation is fast; BFF configStore write)
5. On error: show toast "Could not update retail mode. Try again." (uses existing `notifyError`)

## Accessibility Contract

- All interactive elements (Add to Cart, banner toggle) must have accessible labels
- "Add to Cart" aria-label: `"Add {product name} to cart"` when product name is not inline
- Banner toggle aria-pressed: true/false reflecting current mode
- Color-only information is never the sole differentiator — stock badges use both color and
  text label ("In Stock", "Low Stock", etc.)
- Minimum touch target 44px height on banner toggle (set via `min-height: 44px`)
- Cart total and subtotal amounts use `aria-live="polite"` region so screen readers announce
  changes when items are added
