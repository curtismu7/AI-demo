<objective>
Remove redundant `!important` declarations in `BankingAgent.css` by fixing the 3 clear categories where better CSS specificity eliminates the need.

Purpose: 67 `!important` declarations make theming and override work brittle. Most are specificity hacks. This plan removes the safe, mechanical wins — leaving the legitimately necessary ones untouched.

**Legitimately necessary — do NOT touch:**
- Lines 10-20: `body > .banking-agent-float-root` block — fights `ping-end-user-nano` and other third-party globals that hide portaled elements
- Lines 418, 430: `tcd-root`/`tcd-events` overrides — fighting the token chain drawer's own rules
- Lines 1659-1683: media-query overrides for `right/left/width/height` — React sets inline styles; `!important` is the only way to win
- Lines 2064: `.ba-mode-inline { position: relative !important }` — must override the component's own `position: fixed`
- Lines 2931, 2944, 2952: split-column overrides — fighting React inline styles and tcd rules

**The three removable categories:**

### Category A — `.banking-agent-msg-bubble--toolsteps` (4 !important, lines 2111-2114)
Current:
```css
.banking-agent-msg-bubble--toolsteps {
  background: transparent !important;
  border: none !important;
  padding: 4px 0 8px !important;
  max-width: 100% !important;
}
```
Fix: use the parent class `.banking-agent-msg.tool-progress` to increase specificity (2 classes vs 1):
```css
.banking-agent-msg.tool-progress .banking-agent-msg-bubble--toolsteps {
  background: transparent;
  border: none;
  padding: 4px 0 8px;
  max-width: 100%;
}
```

### Category B — `.ba-popout-mode .ba-left-col` (1 !important, line ~200)
Current:
```css
.banking-agent-panel.ba-popout-mode .ba-left-col {
  display: none !important;
}
```
Fix: add the `.banking-agent-panel` class to raise specificity:
```css
.banking-agent-panel.ba-popout-mode .ba-left-col {
  display: none;
}
```
Wait — check if the conflict is with `:not(.ba-mode-inline)` — both have the same specificity. Fix: add parent `.banking-agent-float-root` wrapper:
```css
.banking-agent-float-root .banking-agent-panel.ba-popout-mode .ba-left-col {
  display: none;
}
```

### Category C — `.App--marketing-page` FAB/dock blocks (~18 !important, lines 3472-3567)
These selectors (``.App--marketing-page .banking-agent-fab``, `.App--marketing-page .embedded-banking-agent`) already have higher specificity (2 classes) than the rules they're overriding (1 class). Remove `!important` from all properties in these blocks.

**Exception**: The `html[data-theme="dark"] .App--marketing-page .global-embedded-agent-dock-wrap` rule — keep its !important because it overrides the base `.App--marketing-page` rule which also has !important. This pair must both keep !important or both lose it.

Output: ~23 fewer `!important` declarations. All visual behavior identical.
</objective>
