<objective>
Clean up `App.css`: remove 13 redundant `-webkit-text-fill-color` lines and audit 26 `!important` declarations for safe removal.

Purpose: `-webkit-text-fill-color` overrides `color` in WebKit/Blink. When both are set to the same value (which is the case throughout App.css), the `-webkit-text-fill-color` line is redundant — `color` already works in all modern browsers. Removing it reduces noise and eliminates potential confusion when theming.

**Pattern in App.css:**
```css
color: #fff !important;
-webkit-text-fill-color: #fff !important;  /* ← redundant — same value as color */
```
The `-webkit-text-fill-color` is only non-redundant when you want a gradient `background-clip: text` effect — none of these rules do that.

**!important audit:**
The 26 `!important` in App.css are mostly on nav link `color` rules. Since nav links sit inside scoped selectors (`.app-nav a`, `.sidebar-link`, etc.), the `!important` is often unnecessary — the parent-scoped selector already wins. Remove where the selector specificity is sufficient; keep where fighting Bootstrap or React-Router link styles.

Output: 13 fewer lines (webkit), ≥6 fewer `!important`.
</objective>
