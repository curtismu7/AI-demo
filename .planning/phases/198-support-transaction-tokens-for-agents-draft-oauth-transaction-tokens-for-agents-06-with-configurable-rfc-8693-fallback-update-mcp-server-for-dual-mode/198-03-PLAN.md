  <ul className="tcp-list">

    <li><strong>Transaction Scope</strong> — What operation the agent is authorized to perform. Example: <code className="tcp-code">check_balance</code> or <code className="tcp-code">transfer_funds</code></li>
    <li><strong>Agent Attestation</strong> — Optional cryptographic proof that the agent (not a spoofed client) initiated the request</li>
    <li><strong>Audit Trail Integration</strong> — Every agent action is tagged with transaction ID, enabling compliance logging and troubleshooting</li>
  </ul>

  <ol className="tcp-ordered-list">
    <li>User logs in via OAuth → Receives user access token</li>

      <ul>
        <li>Sends: User access token + agent identity + transaction scope</li>
        <li>Optionally signs the request (agent attestation)</li>
      </ul>
    </li>
    <li>PingOne validates and issues: <strong>Transaction Token</strong>
      <ul>
        <li>Contains: Transaction ID, transaction scope, user identity, agent identity</li>
        <li>Optionally includes: agent signature, transaction timestamp</li>
      </ul>
    </li>
