<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → BFF | Unauthenticated users cannot set credentials |
| BFF → PingOne Management API | Worker token must have sufficient scopes (app:write) |
| BFF → configStore | Credentials stored server-side, never sent to client in responses |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-205-01 | I (Integrity) | Credentials API endpoint | mitigate | Require admin/staff session; validate all inputs with zod schema; reject unknown credential keys |
| T-205-02 | S (Spoofing) | Worker token input | mitigate | Display visual confirmation of token scope in modal; require user to copy token from PingOne UI |
| T-205-03 | R (Repudiation) | Credential updates | mitigate | Log all credential updates (key names only, not values) with session info to audit log |
| T-205-04 | D (Denial) | Modal DOS | accept | Modal is admin-only; rate limit /api/config/credentials/set to 10 req/min per session |

</threat_model>
