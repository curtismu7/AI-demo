  <action>
Create `WebMcpPanel.js` as a React functional component (~150-200 lines):

**State:**
- `tools` (array) — loaded from `listMcpTools()`
- `selectedTool` (object|null) — currently selected tool
- `params` (object) — input parameters for selected tool
- `loading` (boolean) — loading tools or calling tool
- `result` (object|null) — last tool call result
- `streamEvents` (array) — SSE events during tool call
- `error` (object|null) — `{ message, details }` for hybrid error display

**Behavior:**
1. On mount, call `listMcpTools()` to populate tool list
2. Tool list renders as a clickable list/cards showing tool name + description
3. Selecting a tool shows its `inputSchema` as form fields (render JSON schema properties as simple text inputs — prototype level per D-05)
4. "Call Tool" button generates a `flowTraceId` (uuid), opens SSE stream via `openMcpToolStream()`, then calls `callMcpTool()` with the tool name, params, and trace ID
5. Stream events appear in a live log area (scrolling, monospace)
6. Final result displayed in a formatted output section
7. **Hybrid errors (D-03):** On error, show a user-friendly message ("Tool call failed — check connection or permissions") with an expandable `<details>` block containing the full error object (status, message, stack if available)
8. **Config-driven tools (D-03):** Tool list comes from MCP server via BFF — inherently configuration-driven

**CSS:** Create `WebMcpPanel.css` with styling consistent with existing dashboard panels. Use existing CSS variable patterns from the codebase (`--color-*`, `--radius-*` etc. if they exist, otherwise standard clean styling). Include:
- Panel container with border and padding
- Tool list with hover states
- Parameter form with labeled inputs
- Stream log area (dark bg, monospace, max-height with scroll)
- Result display area
- Error display with expandable details

**Export** as default export. Do NOT add routing — this is a panel component rendered conditionally.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → BFF | All tool calls proxied; user input (tool params) crosses this boundary |
| Feature flag | Controls UI visibility; admin-only toggle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-173-04 | Injection | WebMcpPanel param inputs | mitigate | BFF POST /api/mcp/tool already validates tool name as string; MCP server validates params against tool schema. Browser input rendered with React (auto-escapes XSS). |
| T-173-05 | Information Disclosure | Error details | mitigate | Technical details hidden in `<details>` expandable block (D-03 hybrid); full stack traces only shown to developers who expand. No server secrets exposed — only MCP tool error messages. |
| T-173-06 | Elevation of Privilege | Feature flag bypass | accept | Flag controls UI visibility only; BFF session + token exchange enforce actual authorization. Bypassing flag just shows the panel — tool calls still require valid session + MCP token. |
</threat_model>
