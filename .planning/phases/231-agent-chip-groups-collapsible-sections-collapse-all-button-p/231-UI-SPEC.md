## Scope Summary

Four changes ship together as one cohesive redesign of the BankingAgent left rail:

1. **Remove Learn & Explore chips from inline panel** â€” the `showLearnMore` toggle and the `EDUCATION_COMMANDS` list inside the left rail are removed entirely from the inline `.ba-left-col` render path.
2. **LangGraph regex heuristic** â€” every chip label in `ACTION_GROUPS` and every `EDUCATION_COMMANDS` entry is registered as a heuristic pattern so none fall through to the LLM.
3. **Collapsible chip group sections** â€” the existing `ba-action-group` / `ba-group-header` / `ba-group-content` pattern is kept but receives a count badge, a collapse-all/expand-all control, and a height cap so the chip area never pushes the response panel.
4. **Chip discovery popout ("All actions")** â€” a full-coverage modal triggered from a small button in the left-rail footer. Shows all chips from `ACTION_GROUPS` plus all 60+ entries from `EDUCATION_COMMANDS`, organized by group, searchable, keyboard-accessible.

## Spacing / Layout Constraints

### Left-rail chip area height cap (critical)

The chip area (`.ba-left-col` scroll container) must never increase the panel's overall height. The existing `.ba-left-col` is already `overflow-y: auto` â€” this contract confirms that no new wrapper element may use `overflow: visible` or add to the flex column's intrinsic height.

Specific rule: the expanded chip area for all groups combined must be visually bounded by the `.ba-left-col` scroll container. No chip content may escape onto the response panel side.

### Group count badge

## Chip Discovery Popout â€” "All Actions" Modal

### Trigger

The `âŠž All actions` button in the left-rail footer. Not the `âšˇ` button (which continues to serve the existing "Learn & Explore" popup for backwards compatibility â€” see removal note below).

### Removal of inline Learn & Explore

- The `â–ľ Learn more` toggle button and the `EDUCATION_COMMANDS` list inside the left-rail ARE REMOVED from the authenticated inline render path.
- The `âšˇ` bottom-bar button and its `.ba-commands-popup` panel ARE ALSO REMOVED. The `âšˇ` button and `ba-cmd-btn` are replaced by the "All actions" popout.
- `EDUCATION_COMMANDS` items are exclusively accessible via the discovery popout.

### Popout structure

The popout renders as a **positioned overlay anchored to the agent panel**, not a document-body modal. It appears above or beside the left-rail depending on the panel's current mode:

- **Floating mode** (`.banking-agent-panel:not(.ba-mode-inline)`): popout expands as an overlay layered on top of the panel, anchored at the bottom-left of the left rail, growing upward. `position: absolute`, `bottom: 0`, `left: 0`, `z-index: 1` (above panel content, below panel `z-index: 10059`).
- **Inline/embedded mode** (`.ba-mode-inline`): popout renders as a slide-in panel covering the left rail's full width and height. `position: absolute; inset: 0`.

Container dimensions:
- Floating: `width: 320px`, `max-height: calc(100% - 40px)`, `overflow-y: auto`
- Inline: `width: 100%`, `height: 100%`, `overflow-y: auto`

Background: `var(--ba-surface)`. Border: `1px solid var(--ba-border)`. Border-radius: `12px` (floating) / `0` (inline, fills container). Box-shadow: `0 8px 32px rgba(10,20,50,0.45)` (floating only).

### Popout header

```
[ âŠž All actions ]                      [ âś• ]
```

Label: "All actions" at `font-size: 13px`, `font-weight: 600`, color `var(--ba-text)`.
Close button: `âś•` (U+2715), `width: 28px`, `height: 28px`, `border-radius: 6px`, no background, color `var(--ba-muted)`. On hover: background `rgba(65,105,225,0.12)`, color `var(--ba-text)`.

Padding: `12px 16px 8px`. Border-bottom: `1px solid var(--ba-border)`.

### Search input

Below the header, before the first section group.

Placeholder: `Search actionsâ€¦`
Size: `width: 100%`, `font-size: 12px`, `padding: 8px 12px`, `border-radius: 8px`. Background `var(--ba-bg)`. Border `1px solid var(--ba-input-bd)` (maps to `--ba-input-bd: rgba(65,105,225,0.45)`). Color `var(--ba-text)`.

Search behavior: live filter on chip label text as the user types. Case-insensitive substring match. When no results match, show the empty state (see Copywriting below). Section headers for empty groups are hidden when search is active and that group has no matches.

Keyboard: `Escape` clears search and returns focus to trigger button if search is empty; `Escape` when search is already empty closes the popout.

### Section groups in popout

Groups displayed in order: Account, Transaction, Admin, Testing, Learn & Explore.

Each group:
- Section header: `font-size: 10px`, `font-weight: 600`, `letter-spacing: 0.08em`, `text-transform: uppercase`, color `var(--ba-muted)`. Padding `8px 16px 4px`. Uses existing `.ba-commands-section` class.
- Chips: `display: flex; flex-wrap: wrap; gap: 4px; padding: 0 14px 8px`. Uses existing `.ba-chips` + `.ba-chip` classes. No new chip styles needed.
- "Learn & Explore" group chips use existing `.ba-chip--learn` class.

All 60 `EDUCATION_COMMANDS` entries are shown in the Learn & Explore section. The four `ACTION_GROUPS` (account: 3, transaction: 4, admin: 2, testing: 4) are shown in their respective sections.

### Keyboard navigation

| Key | Behavior |
|-----|----------|
| `Tab` / `Shift+Tab` | Move focus through search input then chips in DOM order |
| `Enter` / `Space` | Activate focused chip |
| `Escape` | Close popout, return focus to `âŠž All actions` button |
| Arrow keys | Optional (nice-to-have, not required for phase) |

`aria-label` on the popout container: `"Action browser"`. `role="dialog"`, `aria-modal="true"`. Close button `aria-label="Close action browser"`. Search input `aria-label="Search actions"`. Section headers use `role="heading"` `aria-level="3"`.

## LangGraph Regex Heuristic Contract

This is a non-visual requirement that the executor must implement alongside the UI changes. Including it here so the planner has a single reference.

Every string in `ACTION_GROUPS` `label` fields and every `EDUCATION_COMMANDS` `label` field must produce a corresponding regex pattern in the LangGraph heuristic matcher (in `bankingAgentLangGraphService.js` or equivalent). The pattern must be case-insensitive and must match the label as a substring of the user's natural language input.

No new UI elements are introduced for this requirement. The executor should verify by sending each chip label as a chat message and confirming it routes via heuristic (not LLM fallback).
