<objective>
Fix a concurrent-session data hazard: MCPTool stores session_id and agent_token as mutable PrivateAttr instance state. Tool instances are shared across sessions. Two concurrent sessions race on set_session_context(), causing session A's _arun() to read session B's session_id — a security-class correctness bug (one user's banking calls execute under another user's session).

Purpose: Eliminate the race by replacing both PrivateAttr fields with module-level ContextVars, using the exact pattern already established for _current_tracer (WR-06 fix). The fix is surgical — no API changes to callers, no behavioral changes, only the storage mechanism changes from shared mutable instance state to task-local ContextVar.

Output: Updated mcp_tool_provider.py with two new ContextVars and no PrivateAttrs for session context; test suite extended with concurrent-isolation proof; REGRESSION_PLAN.md §4 entry added.
</objective>
