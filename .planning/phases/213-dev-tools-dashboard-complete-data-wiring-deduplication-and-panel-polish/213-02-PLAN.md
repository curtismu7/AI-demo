<objective>
Restore the Banking Agent side results panel — the floating card that shows accounts, transactions, and balance data to the left of the agent when the agent returns structured data.

**Root causes identified:**
1. `resultsPanelStyle` returns `undefined` for inline mode (`isInline=true`) when `agentBounds` is null (before ResizeObserver fires). This means the CSS fallback `right: 444px; bottom: 96px` applies — which positions the panel based on a floating-agent layout and may place it off-screen in inline (column) mode.

2. The `agentDisplayMode === 'panel'` guard means if the user ever set the Config page to "Full Page" mode, the `ResultsPanel` is never set. The original intent was: 'panel' → side card, 'fullpage' → update main dashboard. But users have reported the panel "got lost" — likely because Config defaulted or they switched to fullpage. The fix: ALWAYS set the `resultPanel` state (show the side card), and ALSO dispatch the dashboard event. The side card and dashboard update are not mutually exclusive.

3. `resultsPanelStyle` for inline mode needs a safe fallback that computes from `panelRef.current` bounds when `agentBounds` is null.

Purpose: Ensure the side results card appears reliably after every agent response that returns structured banking data.
Output: Working ResultsPanel in both float and inline agent modes.
</objective>

In the `resultsPanelStyle` useMemo (lines ~1732-1776), replace the inline fallback:

Current:
```js
if (isInline) return undefined;
```

Replace with a computed fallback that reads `panelRef.current` directly (synchronous, no observer needed):
```js
if (isInline) {
  // Try to read panel bounds synchronously from the ref
  const rect = panelRef.current?.getBoundingClientRect();
  if (rect) {
    const topEdge = Math.max(8, rect.top);
    const maxH = Math.min(window.innerHeight - topEdge - 16, 520);
    const desiredLeft = Math.max(8, rect.left - resultsPanelWidthPx - 12);
    return {
      position: 'fixed',
      left: Math.min(desiredLeft, window.innerWidth - resultsPanelWidthPx - 8),
      top: topEdge,
      bottom: 'auto',
      right: 'auto',
      maxHeight: maxH,
      zIndex: 10058,
    };
  }
  // Truly no rect yet — hide to avoid CSS flash at wrong position
  return { position: 'fixed', left: -9999, top: -9999, zIndex: -1 };
}
```

**Effect:** When `isInline=true` and `agentBounds` is null (rare pre-ResizeObserver window), we still compute from `panelRef`. If the ref isn't mounted yet, we use off-screen coords (-9999) to prevent the CSS `right: 444px` flash. Once ResizeObserver fires and sets `agentBounds`, the correct position is used.

Also add `panelRef` to the `useMemo` dependency array (it's a ref — stable, but needed for the synchronous read):
```js
}, [dragPos, isExpanded, isInline, resultsPanelWidthPx, agentBounds]);
```
→ No change needed to the dependency array since refs are stable — just add a comment:
```js
// panelRef is stable (useRef), agentBounds drives re-computation

```
