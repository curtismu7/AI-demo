**Step 2 — Normalize mfaTest.js (GREEN phase for mfaTest):**

Read the full `mfaTest.js` to understand the current `result._debug.request` passthrough pattern. The current code does:

```js
resBody.pingoneRequest = result._debug.request;
```

The risk is that `result._debug` may be undefined if `mfaService` encountered an error before reaching PingOne. Normalize with a safe accessor helper at the top of the file (after the require statements):

```js
/**
 * Normalize a PingOne debug request object for UI trace display.
 * Ensures method, url, body keys are always present when a call was made.
 * @param {object|undefined} debugReq — from result._debug.request or err._debug.request
 * @returns {{ method: string, url: string, body: object|null }|undefined}
 */
function normalizePingoneRequest(debugReq) {
  if (!debugReq || typeof debugReq !== 'object') return undefined;
  return {
    method: debugReq.method || '',
    url: debugReq.url || '',
    body: debugReq.body !== undefined ? debugReq.body : null,
    ...(debugReq.contentType ? { contentType: debugReq.contentType } : {}),
  };
}
```

Then replace every assignment `resBody.pingoneRequest = result._debug.request` with:
```js
resBody.pingoneRequest = normalizePingoneRequest(result._debug?.request);
```

And every `errBody.pingoneRequest = err._debug.request` with:
```js
errBody.pingoneRequest = normalizePingoneRequest(err._debug?.request);
```

Do this for ALL occurrences in the file (there are ~12 based on grep output).
