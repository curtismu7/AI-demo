# Phase 214-01 Plan — PingOne Authorize Test Page

## Objective

Wire up the `AuthzTestPage.jsx` component (already authored) as a fully navigable,
styled, routed page that lets admins and developers test PingOne Authorize policy
decisions without submitting a real banking transaction.

## Background

`AuthzTestPage.jsx` was created in the working tree but has three gaps:
1. No CSS (`AuthzTestPage.css` import resolves to a missing file → build error)
2. No React Router route in `App.js`
3. No nav link in `AdminSideNav.jsx`

Backend routes already exist and work:
- `GET  /api/authorize/test-status`  — engine status + thresholds (no auth required)
- `POST /api/authorize/test-evaluate` — evaluate a transaction (no auth required)

## Deliverables
