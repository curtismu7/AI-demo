Create new service `callbackDispatcher.js` to dynamically register callbacks:

```javascript
const configStore = require('./configStore');

/**
 * Dynamic callback route registration.
 * 
 * Handles:
 * 1. Admin OAuth callback: GET /api/auth/oauth/callback (default)
 * 2. User OAuth callback: GET /api/auth/oauth/user/callback (default)
 * 
 * Supports custom paths (Federate: /oauth2/callback, Auth0: /callback)
 */
function registerCallbacks(app, oauthRouter, oauthUserRouter) {
  const adminPath = configStore.getEffective('oauth_admin_callback_path');
  const userPath = configStore.getEffective('oauth_user_callback_path');
  
  console.log('[callback-dispatcher] Registering OAuth callbacks:', {
    admin: adminPath,
    user: userPath,
  });
  
  // Admin callback — handler from routes/oauth.js
  if (adminPath) {
    app.get(adminPath, (req, res, next) => {
      // Delegate to oauth router's built-in callback handler
      oauthRouter(req, res, next);
    });
  }
  
  // User callback — handler from routes/oauthUser.js
  if (userPath) {
    app.get(userPath, (req, res, next) => {
      // Delegate to oauthUser router's built-in callback handler
      oauthUserRouter(req, res, next);
    });
  }
}

module.exports = { registerCallbacks };
```

```markdown
# OAuth Callback Path Configuration

Configure OAuth callback paths based on your IDP's requirements.

## Default (PingOne)

```bash
OAUTH_ADMIN_CALLBACK_PATH=/api/auth/oauth/callback
OAUTH_USER_CALLBACK_PATH=/api/auth/oauth/user/callback
```

## Federate

Federate typically expects all callbacks on a single path:

```bash
OAUTH_ADMIN_CALLBACK_PATH=/oauth2/callback
OAUTH_USER_CALLBACK_PATH=/oauth2/callback
```

Then in your Config UI, set redirect URIs:
- Admin: `https://your-app.example.com/oauth2/callback`
- User: `https://your-app.example.com/oauth2/callback`

## Auth0

Auth0 callback is typically on the app root:

```bash
OAUTH_ADMIN_CALLBACK_PATH=/callback
OAUTH_USER_CALLBACK_PATH=/callback
```

Set redirect URIs in Auth0 dashboard:
- Admin: `https://your-app.example.com/callback`
- User: `https://your-app.example.com/callback`

(Auth0 routes internally based on client_id)

## Okta

Okta uses a standard callback path:

```bash
OAUTH_ADMIN_CALLBACK_PATH=/oauth/callback
OAUTH_USER_CALLBACK_PATH=/oauth/callback
```

## Important Notes

- **Login paths** (`/api/auth/oauth/login`, `/api/auth/oauth/user/login`) are NOT configurable — keep them fixed per OAuth spec
- **Callback paths** only — these are where the IDP redirects the user after authentication
- The **redirect_uri** in your Config UI must exactly match the callback path URL (including domain, port, path)
```
