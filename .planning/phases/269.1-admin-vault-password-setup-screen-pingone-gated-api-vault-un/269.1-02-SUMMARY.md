# Phase 269.1 Plan 02: Mount /api/admin/vault routes Summary

Three admin-gated HTTP routes (`GET /status`, `POST /unlock`, `POST /rotate`) shipped at `/api/admin/vault/*`. Each handler is `authenticateToken + requireAdmin`-gated, Vercel-guarded with 503, audit-logged via the 4-field allowlist, and protected against the spoofing / brute-force / oracle / mutex / path-traversal STRIDE threats enumerated in the plan. The unlock route is rate-limited at 5 attempts per 5 minutes per session sub; the rotate route is serialized via a module-scoped mutex and re-verifies `currentPassword` against the file before re-wrapping DEKs.

## What Was Built

### `banking_api_server/routes/adminVault.js` — 3 handlers + 2 middleware

| Element | Lines | Purpose |
|---------|-------|---------|
| Vercel bypass middleware | `router.use(...)` | Returns 503 `{error: 'vault_disabled_serverless', message}` when `process.env.VERCEL === '1'` — runs AFTER outer authenticateToken (mounted in server.js) but BEFORE per-handler requireAdmin |
| `unlockLimiter` | `rateLimit({...})` | 5 attempts / 5 min, `keyGenerator: req.user?.sub \|\| req.ip`, mirrors adminConfig.js shape |
| `safeAudit(entry)` | best-effort wrapper | Calls `recordAudit(auditPath(), entry)`; swallows audit-write failures so the primary op still responds |
| `resolveVaultPath()` | helper | `process.env.VAULT_PATH \|\| DEFAULT_VAULT_PATH` — read lazily per request so tests can vary it |
| `rotateInProgress` | module-scoped `let` | Mutex flag set before rotate work, cleared in finally — second concurrent rotate hits 409 |
| `GET /status` | requireAdmin → handler | Returns `{unlocked, entriesLoaded, vaultFilePresent, vaultPath: path.basename(...)}` — never the full path, never entry names. NOT audited (read-only metadata) |
| `POST /unlock` | requireAdmin → unlockLimiter → handler | Validates body, calls `unlockVaultAtRuntime`, maps errors: VaultAuthError/VaultIntegrityError → 401 opaque, VAULT_FILE_NOT_FOUND → 404, other → 500. One audit line per outcome (op:'unlock', caller:'adminVault') |
| `POST /rotate` | requireAdmin → handler | Validates body (length ≥ 12, ≠ currentPassword), checks `isVaultUnlockedThisProcess()` (423 if false), checks mutex (409 if held), re-opens vault with currentPassword (401 if wrong), then `handle.rotate(newPassword) + handle.save() + handle.close()`. Audit line on every outcome |

## STRIDE Threat Register — Disposition

| Threat | Disposition | Mitigation Evidence |
|--------|-------------|---------------------|
| T-269.1-01 Spoofing/EoP — calls without auth | mitigated | Outer `authenticateToken` in server.js mount + per-handler `requireAdmin` (3 occurrences in adminVault.js); regression tests 3 (401 no req.user) + 4 (403 customer role) |
| T-269.1-02 Tampering (CSRF) — cross-origin POST | accept (existing posture) | sameSite cookie + httpOnly + admin OAuth gate + JSON body content-type defeats CSRF without explicit token middleware; same posture as every other /api/admin/* route; Plan 04 documents in docs/vault.md |
| T-269.1-03 Info Disclosure — password in logs/response/audit | mitigated | Route NEVER logs req.body. recordAudit 4-field allowlist physically rejects a `password` field. Regression tests 10 + 18 use SENTINEL `SENTINEL-PWD-DO-NOT-LOG-001` and assert it never appears in res.body, recordAudit args, or any console.log/warn/error |
| T-269.1-04 Spoofing — brute force on /unlock | mitigated | express-rate-limit ^7.1.5 — 5 attempts/5min keyed by `req.user?.sub \|\| req.ip`; 6th returns 429 without calling unlockVaultAtRuntime; regression test 17 asserts `mock.calls.length === 5` at 6th attempt |
| T-269.1-05 EoP/Tampering — rotate without re-verify | mitigated | Rotate handler calls `vaultLib.openVault(path, currentPassword)` BEFORE `handle.rotate(newPassword)`. Defense in depth even if in-memory `isVaultUnlockedThisProcess()` is stale; integration test 4 asserts wrong currentPassword returns 401 even after a successful unlock |
| T-269.1-06 Tampering — concurrent rotate | mitigated | Module-scoped `rotateInProgress` boolean set/cleared in try/finally; second concurrent call hits the early-return 409; integration test 6 asserts `Promise.all([r1, r2]).sort() === [200, 409]` |
| T-269.1-07 Info Disclosure — BFF unlocked but MCP Gateway still on env password | accept (out of scope) | Plan 04 documents this in docs/vault.md ("After rotate, update env or restart gateway"); no propagation in this phase |
| T-269.1-08 Info Disclosure — password in memory after handler | mitigated | Express discards req references at response end; route handler never stores `req.body.password` on module scope or session; no caching; regression test 18 SENTINEL asserts no leak via console or audit |
| T-269.1-09 Tampering/InfoDisc — path traversal via vault path | mitigated | `vaultPath` ONLY from `process.env.VAULT_PATH \|\| DEFAULT_VAULT_PATH`; route NEVER reads vaultPath from req.body; GET /status returns `path.basename(vaultPath)` — never the full path; regression test 2 asserts `res.body.vaultPath === 'secrets.vault'` for VAULT_PATH=`/very/deep/path/secrets.vault` |
| T-269.1-10 Info Disclosure — wrong-pw vs tampered-file oracle | mitigated | Both VaultAuthError + VaultIntegrityError map to 401 `'vault: open failed (bad password or tampered file)'` — IDENTICAL string. Regression test 8 asserts byte-equal `body.message` across both paths |

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Fixed test-order issue in regression test 1 + 2**
- **Found during:** Task 2 GREEN verification
- **Issue:** Tests 1 and 2 called `getMocks()` BEFORE `buildApp()`. Since `buildApp()` does `jest.resetModules()`, the test's mock references pointed to a stale module instance while the router loaded fresh mock instances — `isVaultUnlockedThisProcess.mockReturnValue(true)` had no effect on what the route actually called.
- **Fix:** Re-ordered to `buildApp(); getMocks();` and removed the unconditional `delete process.env.VAULT_PATH` from `buildApp()` so individual tests can set VAULT_PATH for the route's lazy `resolveVaultPath()` lookup. Also added explicit `delete process.env.VAULT_PATH` in `afterEach` to prevent leakage between cases.
- **Files modified:** banking_api_server/tests/routes/adminVault.regression.test.js
- **Commit:** 92b76560
