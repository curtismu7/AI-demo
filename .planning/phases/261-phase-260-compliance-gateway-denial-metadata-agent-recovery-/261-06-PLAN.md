**Add a new route `GET /api/demo/claim-diagnostics`** that:

1. **Requires admin or authenticated session** — use the same auth middleware as `GET /api/demo/may-act/diagnose`

2. **Runs the existing may_act diagnose logic** — reuse or call the existing diagnose handler function (extract the inner logic into a shared function if needed, or call the handler directly). Do NOT duplicate the entire diagnose logic — call the existing implementation.

3. **Adds session token scope/audience check**:
   ```javascript
   const sessionToken = req.session?.userAccessToken || req.session?.accessToken || null;
   let sessionTokenCheck = { present: false, scopes: [], audience: null };
   if (sessionToken) {
     try {
       // Decode without verification (diagnostic only — not a security check)
       const parts = sessionToken.split('.');
       const payload = parts[1] ? JSON.parse(Buffer.from(parts[1], 'base64').toString('utf8')) : {};
       const scopeStr = payload.scope || payload.scp || '';
       sessionTokenCheck = {
         present: true,
         scopes: typeof scopeStr === 'string' ? scopeStr.split(' ').filter(Boolean)
                 : Array.isArray(scopeStr) ? scopeStr : [],
         audience: payload.aud || null,
       };
     } catch { /* non-fatal — token may not be JWT */ }
   }
   ```

4. **Return aggregated response**:
   ```javascript
   return res.json({
     timestamp: new Date().toISOString(),
     checks: {
       mayAct: mayActResult.checks.userAttribute,    // from existing diagnose
       appMapping: mayActResult.checks.appMapping,   // from existing diagnose
       sessionToken: sessionTokenCheck,
     },
     diagnosis: mayActResult.diagnosis,
     nextStep: mayActResult.nextStep,
     overallPass: mayActResult.checks.userAttribute?.pass && mayActResult.checks.appMapping?.pass && sessionTokenCheck.present,
   });
   ```

5. **Error handling**: if the may_act diagnose logic fails (e.g. PingOne unreachable), return `{ error: 'diagnostic_unavailable', message: err.message }` with HTTP 503 — do NOT crash the route.

**SECURITY**: The endpoint decodes the JWT payload for diagnostic display only. It does NOT verify the signature (that's handled elsewhere). Mark the response with a comment that this is diagnostic-only data — not an auth decision.

       setClaimDiagLoading(false);
