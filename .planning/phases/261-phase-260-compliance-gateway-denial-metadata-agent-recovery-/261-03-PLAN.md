Find the main message handler (the POST route for `/message` or equivalent — likely around line 51+). Locate the catch block that currently emits `res.status(500).json({ error: errorMessage })` around line 195.

**Add typed error handling BEFORE the catch-all 500** in every POST handler that calls the agent service:

```javascript
// D-03: Structured recovery responses
if (err && err.name === 'LoginRequiredError') {
  return res.status(401).json({
    error: 'login_required',
    requiredScopes: Array.isArray(err.requiredScopes) ? err.requiredScopes : []
  });
}
if (err && err.name === 'HitlRequiredError') {
  return res.status(403).json({
    error: 'hitl_required',
    challengeId: err.challengeId || '',
    challengeType: err.challengeType || 'consent',
    expiresAt: err.expiresAt || ''
  });
}
if (err && err.name === 'ScopeRequiredError') {
  return res.status(403).json({
    error: 'scope_required',
    requiredScopes: Array.isArray(err.requiredScopes) ? err.requiredScopes : []
  });
}
// Existing catch-all continues below
```

Apply this pattern to EVERY catch block that currently falls through to the 500 handler — check if there are multiple POST routes (message, task, etc.) and apply consistently.

**IMPORTANT:** Do NOT remove or modify:
- The existing `res.status(401).json({ error: 'Session expired', agentInitRequired: true })` — that is a different 401 (no session, not gateway denial)
- The existing `res.status(403).json({ error: 'User denied consent', consentDenied: true })` — that is the existing consent decline
- The `res.status(428)` HITL path — that is the existing in-BFF HITL flow

The new 401/403 structured responses are ONLY for errors propagated up from the agent service (LoginRequiredError / HitlRequiredError from runAgentTask).

**Validate requiredScopes**: before using `err.requiredScopes` in a response, ensure each scope is a string matching `/^[a-z][a-z0-9:_-]*$/` — if any entry fails, replace with an empty array. This prevents arbitrary string injection into the response.
