## Component Specs

### 1. GatewayConsentModal (`GatewayConsentModal.js` + `.css`)

**Component name rationale:** Follows naming convention `{Context}ConsentModal` ‚Äî `GatewayConsentModal` (cf. `AgentConsentModal`, `TransactionConsentModal`).

**Props interface:**
```js
GatewayConsentModal({
  show,            // boolean
  challengeId,     // string ‚Äî from error.data.challengeId
  challengeType,   // 'consent' | 'step_up' ‚Äî from error.data.challenge_type
  expiresAt,       // string ISO ‚Äî from error.data.expiresAt
  onApprove,       // () => void ‚Äî user approved
  onDismiss,       // () => void ‚Äî user cancelled
})
```

**Variant: `consent`**
- Title: "Action Requires Your Approval" (üõ°Ô∏è icon)
- Body: "The agent is requesting permission to proceed. Review the details below before approving."
- CTA: "Approve" (primary, `#3b82f6` fill, white text) | "Cancel" (ghost)
- No MFA sub-component rendered

**Variant: `step_up`**
- Title: "Identity Verification Required" (üîê icon)
- Body: "The agent is requesting elevated access. Verify your identity to continue."
- Delegates MFA mechanism to `FidoStepUpModal` (FIDO preferred) with `fallbackToOtp={true}` to support OTP fallback ‚Äî same pattern as `BankingAgent.js` lines 1550‚Äì1570
- Outer component owns the title, context text, and dismiss button; inner modal handles biometric/OTP interaction
- CTA: rendered by inner modal (no duplicate button)
- HITL badge: visible (match `.acm-hitl-badge` from `AgentConsentModal.css`)

**Layout:**
- Draggable floating card ‚Äî use existing `useDraggablePanel` hook (pattern from `AgentConsentModal.js`)
- Default size: 400w √ó auto (no fixed height ‚Äî content-driven)
- `createPortal` to `document.body`
- `z-index: 9995` (below Toastify `10100`, above agent panel `9980`)
- Closes on Escape key (`document.addEventListener('keydown', ...)`)

**CSS class prefix:** `gcm-` (e.g., `gcm-card`, `gcm-drag-handle`, `gcm-title`, `gcm-body`, `gcm-footer`)
