<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Gateway → response headers | X-Gw-Audit-Trail is an internal BFF↔Gateway header — never reaches the browser |
| toolArgs → PingOne Authorize | Transaction amount from JSON-RPC body forwarded to policy engine |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-259-03-01 | Spoofing | X-Gw-Audit-Trail header | accept | Header is on the internal gateway→BFF channel (not client-facing); BFF strips before responding to browser |
| T-259-03-02 | Tampering | toolArgs.amount from JSON-RPC body | mitigate | Amount is passed to PingOne Authorize for policy evaluation — Authorize makes the authorization decision, not the amount field itself |
| T-259-03-03 | Information Disclosure | auditTrail in X-Gw-Audit-Trail | mitigate | Contains only decision metadata (active bool, sub, decision string) — no raw token values |
| T-259-03-04 | Denial of Service | Introspection adding latency to every request | mitigate | 30 s cache per token (Plan 01); skipped when endpoint not configured |
</threat_model>
