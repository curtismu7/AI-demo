<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Gateway → BFF | X-Gw-Audit-Trail header on internal network hop |
| BFF → Browser | tokenEvents array in JSON response — audit metadata only, no raw tokens |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-259-04-01 | Information Disclosure | gwAuditTrail in tokenEvents | mitigate | tokenEvents contain only decision metadata (active bool, decision string, sub, exp) — never raw bearer tokens |
| T-259-04-01 | Elevation of Privilege | hitl_required gateway bypass | accept | The 428 response triggers the step-up modal — the user MUST complete MFA before the transaction proceeds. The BFF does not auto-approve on 428. |
| T-259-04-03 | Tampering | X-Gw-Audit-Trail header injection | accept | Header is on an internal channel (BFF→gateway, both under operator control); attacker cannot reach this channel from the browser |
</threat_model>
