<objective>
Add a startup RuntimeError guard to the Python langchain_agent that refuses to start
if SKIP_TOKEN_SIGNATURE_VALIDATION=true is set in any environment other than "development"
or "test". Verify the guard with focused pytest cases. Inspect session_manager.py to
confirm session→user binding is enforced at creation time and document the finding in
REGRESSION_PLAN.md §4.

Purpose: The Node.js BFF (demo_api_server/server.js line 42) already has this guard.
The Python langchain_agent currently does not read SKIP_TOKEN_SIGNATURE_VALIDATION at
all — if the var is set accidentally in staging/production, JWT tokens would pass
without signature checks with no warning. This plan closes the gap.

Output:
  - langchain_agent/src/config/settings.py — guard added to validate_environment_config()
  - langchain_agent/tests/test_config_settings.py — 4 new test cases for the guard
  - REGRESSION_PLAN.md §4 — finding documented
</objective>
