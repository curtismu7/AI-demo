# Plan 249-01 — Disable gateway passthrough; fix MCP Server audience config

## Objective

The gateway is currently running in passthrough mode (`MCP_GW_PASSTHROUGH_TO_MCP_SERVER=true`),
which means it forwards the inbound token (aud=mcpgateway.ping.demo) unchanged to the MCP Server.
As a result, the MCP Server has been configured to accept aud=mcpgateway.ping.demo tokens
(`MCP_SERVER_RESOURCE_URI=mcpgateway.ping.demo`), bypassing the per-hop audience isolation
that is the entire purpose of the gateway layer.

This plan disables passthrough and fixes the audience configuration so each hop carries
only the audience for that specific service — the core RFC 8693 per-hop contract.

## Context

Current broken state:
- `demo_mcp_gateway/.env`: `MCP_GW_PASSTHROUGH_TO_MCP_SERVER=true`
- `demo_mcp_server/.env`: `MCP_SERVER_RESOURCE_URI=mcpgateway.ping.demo` (wrong — set to match passthrough token)
- Token chain: BFF→Gateway (aud=mcpgateway.ping.demo) → Gateway→Server (SAME aud=mcpgateway.ping.demo) → Server→BankingAPI (Step 9 exchange → enduser.ping.demo)

Target correct state:
- Gateway re-exchanges: takes gateway-scoped token → issues server-scoped token
- Token chain: BFF→Gateway (aud=mcpgateway.ping.demo) → Gateway re-exchange → Gateway→Server (aud=mcpserver.ping.demo) → Server calls BankingAPI
- MCP Server validates aud=mcpserver.ping.demo

## Prerequisites (Manual — not automated by this plan)

Before running this plan's code changes, the PingOne MCP Gateway app (`3fc5ec99`) must have a
CC grant for the MCP Server resource (`mcpserver.ping.demo`) with scopes `read` and `write`.
This is a PingOne Console action. The RESEARCH.md notes this was attempted but the scope-name
collision limited the grant to `ai:agent:read` only.

The scope-topology.json shows `Super Banking MCP Server` has `mirroredScopes` (read, write, etc.)
— these are the scopes the gateway should request in its re-exchange. If PingOne blocks the grant
due to name collision, Option A is: check if the gateway's own scopes (mcp:invoke) on mcpgateway
resource prevent it from requesting read/write on mcpserver. The RESEARCH.md "Post-Fix State"
section describes this constraint. The plan tasks handle both paths.

### Task 2 — Fix MCP Server audience in demo_mcp_server/.env

<read_first>
- demo_mcp_server/.env
- demo_mcp_server/src/middleware/mcpTokenValidator.js
- demo_mcp_server/src/server/HttpMCPTransport.ts (lines 434–450 — MCP_GATEWAY_MODE enforcement)
</read_first>

<action>
In `demo_mcp_server/.env`, change:
  `MCP_SERVER_RESOURCE_URI=mcpgateway.ping.demo`
to:
  `MCP_SERVER_RESOURCE_URI=mcpserver.ping.demo`

Also add the gateway mode enforcement flags so the MCP Server rejects tokens whose aud is
the gateway audience (D-05 anti-bypass):
  `MCP_GATEWAY_MODE=true`
  `MCP_UPSTREAM_RESOURCE_URI=mcpserver.ping.demo`
  `MCP_GW_RESOURCE_URI=mcpgateway.ping.demo`

These env vars are read by `HttpMCPTransport.ts` at line 435–441. When `MCP_GATEWAY_MODE=true`,
the transport calls `enforceUpstreamContract(claims, { upstreamAudience, gatewayAudience })` which
rejects tokens with aud=mcpgateway.ping.demo (D-05) and requires aud=mcpserver.ping.demo.

The `mcpTokenValidator.js` uses `MCP_SERVER_RESOURCE_URI` as the `expectedAudience` in
`validateTokenAtGateway()`. Update this env var to mcpserver.ping.demo so both paths are aligned.

Note: The introspection client (GW_INTROSPECTION_CLIENT_ID=d3f8fead) was chosen because it issues
mcpgateway.ping.demo tokens. After this change, the gateway will issue mcpserver.ping.demo tokens
to the MCP server — confirm the gateway client (3fc5ec99) can introspect those tokens, OR keep
using d3f8fead as the introspection client if d3f8fead is the exchanger that issues the
mcpserver tokens. Do not change GW_INTROSPECTION_CLIENT_ID until this is confirmed.
</action>

<acceptance_criteria>

- `grep "MCP_GATEWAY_MODE" demo_mcp_server/.env` returns `MCP_GATEWAY_MODE=true`
- `grep "MCP_GW_RESOURCE_URI" demo_mcp_server/.env` returns `MCP_GW_RESOURCE_URI=mcpgateway.ping.demo`
- `grep "MCP_UPSTREAM_RESOURCE_URI" demo_mcp_server/.env` returns `MCP_UPSTREAM_RESOURCE_URI=mcpserver.ping.demo`
