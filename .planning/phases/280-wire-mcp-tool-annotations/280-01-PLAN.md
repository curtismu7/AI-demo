<objective>
Wire MCP tool annotations (destructive, idempotent flags from BankingToolRegistry.ts) through the Python pipeline to BaseTool.metadata, and add an annotation-aware instruction to the agent system prompt.

Purpose: The MCP server already emits rich annotations per tool (destructive, idempotent, readable, openWorld). The LangChain layer currently discards them. This phase captures annotations at every hop — JSON-RPC response -> ToolInfo -> MCPTool.metadata — and uses them to inform agent behaviour via the system prompt.

Output:
- connection.py: captures annotations from tools/list response
- tool_registry.py: ToolInfo gains annotations field
- mcp_tool_provider.py: MCPTool.metadata populated as {"destructive": bool, "idempotent": bool, "readable": bool}
- langchain_mcp_agent.py: system prompt includes instruction to announce intent before calling any destructive tool
- test_mcp_annotations.py: 4 pytest tests verifying the pipeline end-to-end with mocks
</objective>
