**Interactive Flow Diagram (CSS-based, no external libs):**
- Horizontal flow: User → Chat → Orchestrator → Agent → MCP Server → Tool
- Security annotation badges at each connection point:
  - User→Chat: "🔐 OAuth Login (PKCE)"
  - Chat→Agent: "🤖 Agent Identity (Client Credentials)"
  - Agent→MCP: "🔄 Token Exchange (RFC 8693)"
  - MCP→Tool: "🔒 Vault Credentials (Last Mile)"
- Use CSS grid/flexbox with connecting lines (borders/pseudo-elements)

**6 Pillars Section (expandable cards):**
Each card has: pillar name, threat description, how the demo addresses it, status badge (✅ Strong / ⚠️ Partial / ❌ Gap), and "See it in action" link.

1. **Credential Replay Prevention** — ✅ Strong
   - Threat: Tokens leaked to LLM via prompt, man-in-the-middle
   - Demo: BFF token custodian pattern, tokens never reach browser/LLM
   - Link: /pingone-test (Token Chain section)

2. **Rogue Agent Prevention** — ✅ Good
   - Threat: Unauthorized agents spoofing identity
   - Demo: Agent authenticates via OAuth client credentials at PingOne
   - Link: /pingone-test (Exchange 2 - Agent Authentication)

3. **Impersonation Prevention / Delegation** — ✅ Strong
   - Threat: Agent self-asserts user identity without validation
   - Demo: RFC 8693 token exchange produces combined token with act (agent) + subject (user). IdP validates.
   - Link: /actor-token-education, /pingone-test (Exchange 3)

4. **Per-Hop Token Exchange** — ✅ Strong
   - Threat: Single token travels entire chain without validation
   - Demo: Token exchanged at each hop (User→BFF, BFF→MCP), each hop gets fresh token scoped to that segment
   - Link: /pingone-test (all 3 exchange cards)

5. **Least Privilege / Scope Narrowing** — ⚠️ Partial
   - Threat: Agent token has more permissions than needed for the task
   - Demo: Audience restriction (RFC 8707), scope parameters on exchange
   - Gap: No visual showing progressive scope narrowing per hop
   - Link: /pingone-test (Audience section)

6. **Last Mile Vault** — ❌ Concept Only
   - Threat: MCP server stores persistent credentials to backend tools
   - Demo: Not yet implemented. Concept: Use a vault/secrets manager for temporary per-call credentials
   - Gap: Currently uses propagated tokens or config-based credentials
   - Link: N/A (future improvement)

**What's Next Section:**
