  <action>
Add a new section to PingOneTestPage.jsx called "Scope Narrowing — Least Privilege" positioned after the existing exchange cards section. This is a static educational visualization (does NOT call any APIs).

**ScopeNarrowingVisualization inline component:**
A horizontal pipeline showing 3 stages with connecting arrows:

Stage 1: "User Login" card
- Scopes: `openid`, `profile`, `email`, `banking:accounts:read`, `banking:accounts:write`, `banking:transfers`, `banking:admin`
- Badge: "Full user permissions"
- Color: blue

Stage 2: "Agent Exchange" card  
- Scopes: `openid`, `banking:accounts:read`, `banking:transfers`
- Badge: "Agent-scoped subset"
- Color: green
- Arrow from Stage 1 with label "Token Exchange narrows scope"

Stage 3: "MCP Tool Call" card
- Scopes: `banking:accounts:read`
- Badge: "Single tool scope"
- Color: orange
- Arrow from Stage 2 with label "Further restricted per tool"

Each scope shown as a pill/chip. Scopes that were dropped between stages shown with strikethrough in the previous stage's card to make the narrowing visual.

Below the pipeline, add a brief explanation:
"At each token exchange hop, the scope is restricted to only what is needed. The user's full permissions are progressively narrowed — the agent only receives scopes relevant to its task, and the MCP tool call gets the minimum scope for that specific operation. This is the principle of least privilege applied to delegated AI access."

**LastMileSecurityNote inline component:**
Add after the scope narrowing section. A bordered info card with:
- Title: "🔒 Last Mile Security — Tool Credential Management"
- Content explaining: "In a production agentic system, the MCP server should NOT store persistent credentials for backend tools. Instead, a secure vault (e.g., HashiCorp Vault, AWS Secrets Manager) provides temporary, scoped credentials for each tool call. This ensures that even if the MCP server is compromised, tool credentials are not exposed."
- A "Concept" badge (yellow) indicating this is planned but not yet implemented
- Brief note: "This demo currently propagates OAuth tokens to the tool. A vault integration is a future enhancement."

CSS: Use the existing PingOneTestPage.css patterns — card borders, section headers, chip/pill styling matching the existing exchange cards.
