<objective>
Extend `banking_mcp_gateway` with three credential dispositions (`oauth_bearer`, `api_key`, `dual_token`) and add a BFF-internal endpoint that delivers the user's id_token to the gateway server-to-server. Implements requirements 266-R1 (gateway side) and 266-R2 (gateway side) and lays the foundation for 266-R4 (path-aware downstream).

**R2 architecture (this revision):**
- `api_key` disposition is the ONLY Gateway-only marker. It records the swap in `_meta.tokenEvents`, returns a marker payload, no backend call.
- `dual_token` disposition FORWARDS to `GET /api/resource-server/identity` on `banking_resource_server` (defined in Plan 02). The Gateway still synthesizes `_meta.tokenEvents` (chain segments) and stamps `credentialPath: 'dual_token'`, but the response BODY comes from the backend route, not a Gateway-fabricated marker. The Gateway forwards the id_token in JSON-RPC params (not as an HTTP header — header-size limit per RESEARCH §Pitfall 2).
- `oauth_bearer` disposition routes banking-data tools to the NEW split routes `GET /api/resource-server/accounts` and `GET /api/resource-server/transactions` (defined in Plan 02). The existing `/api/resource-server/summary` route is preserved untouched but is NOT the target for the new gateway routes.

Plan 01 is the foundation for Paths A, B, and C. Downstream plans depend on Plan 01 being on main BEFORE they are executed (the router defined here is the sole resolver for tool names like `special_offers` and `user_profile_card`).

Purpose: One gateway, three credential mechanisms — visible end-to-end in the Token Chain.
Output: Extended gateway router + new credentialSwap module + tool→backend-route table + BFF id_token retrieval endpoint (regression + integration tests) + Express sessionStore registration + synthesized tokenEvents in api_key/dual_token responses.

```typescript
'use strict';
import { GatewayConfig } from './config';
import { exchangeTokenForBackend } from './tokenExchange';
import { backendResourceUri } from './router';
import type { BackendTarget } from './router';

export type CredentialKind = 'oauth_bearer' | 'api_key' | 'dual_token';
export type CredentialPath = 'oauth_bearer' | 'api_key' | 'dual_token';

export interface OutboundCredential {
  kind: CredentialKind;
  credentialPath: CredentialPath;
  authorization?: string;
  apiKeyMaskedLast4?: string;   // last 4 chars of the service API key, for Token Chain display
  idToken?: string;             // raw id_token — for dual_token, forwarded in JSON-RPC params to /identity (not a header)
}

export class IdTokenMissingError extends Error {
  code = 'id_token_missing';
  constructor() { super('id_token required for dual_token disposition but absent from session'); }
}

export async function selectCredentialForBackend(
  target: BackendTarget,
  subjectToken: string,
  idToken: string | null,
  config: GatewayConfig,
): Promise<OutboundCredential> {
  if (target === 'apikey') {
    const key = config.demoApiKeyServiceKey || '';
    const last4 = key.length >= 4 ? key.slice(-4) : 'XXXX';
    return { kind: 'api_key', credentialPath: 'api_key', apiKeyMaskedLast4: last4 };
  }
  if (target === 'dualtoken') {
    if (!idToken) throw new IdTokenMissingError();
    return {
      kind: 'dual_token',
      credentialPath: 'dual_token',
      authorization: `Bearer ${subjectToken}`,
      idToken,
    };
  }
  // oauth_bearer path — exchange and forward bearer to banking_resource_server
  const exchangedToken = await exchangeTokenForBackend(subjectToken, backendResourceUri(target, config), config);
  return { kind: 'oauth_bearer', credentialPath: 'oauth_bearer', authorization: `Bearer ${exchangedToken}` };
}
```

```javascript
'use strict';
/**
 * /internal/id-token — gateway-only endpoint
 *
 * Returns the current user's id_token to the MCP Gateway server-to-server.
 * This is the ONLY place the raw id_token leaves the BFF. The endpoint:
 *   - is NOT mounted under /api/* (browser-facing prefix)
 *   - requires a shared secret (x-internal-gateway-secret) matching BFF_INTERNAL_SECRET
 *   - reads the id_token from the session lookup keyed by x-subject-sub
 *
 * The SPA NEVER calls this. CLAUDE.md "Token custody rule" applies.
 */
const express = require('express');
const router = express.Router();

const INTERNAL_SECRET = process.env.BFF_INTERNAL_SECRET || 'dev-shared-secret-change-me';

router.get('/id-token', (req, res) => {
  const presented = req.headers['x-internal-gateway-secret'];
  if (!presented || presented !== INTERNAL_SECRET) {
    return res.status(403).json({ error: 'forbidden' });
  }

  // The session is bound to a cookie the gateway doesn't have; the gateway
  // identifies the user via x-subject-sub from the validated MCP token.
  const sub = req.headers['x-subject-sub'];
  if (!sub) return res.status(400).json({ error: 'missing_sub' });

  // Read the registered sessionStore. server.js calls app.set('sessionStore', sessionStore)
  // after instantiation (only when a real store is created; memory fallback does NOT
  // register, so we get undefined here and return 503 — graceful failure for fresh
  // installs that haven't migrated to a persistent store).
  const sessionStore = req.app.get('sessionStore');
  if (!sessionStore || typeof sessionStore.all !== 'function') {
    return res.status(503).json({ error: 'session_store_unavailable' });
  }

  // sqliteSessionStore.all(cb) signature (verified): cb(err, sessions[]) where
  // sessions is an ARRAY of parsed session objects (JSON.parse already applied).
  sessionStore.all((err, sessions) => {
    if (err) return res.status(500).json({ error: 'session_store_error' });
    const all = Array.isArray(sessions) ? sessions : (sessions ? Object.values(sessions) : []);
    const match = all.find((s) => {
      const tokens = s && s.oauthTokens;
      if (!tokens) return false;
      return tokens.subjectSub === sub || tokens.sub === sub;
    });
    if (!match) return res.status(404).json({ error: 'session_not_found' });
    const idToken = match.oauthTokens && match.oauthTokens.idToken;
    if (!idToken) return res.status(412).json({ error: 'id_token_missing' });
    return res.json({ idToken });
  });
});

module.exports = router;
```
