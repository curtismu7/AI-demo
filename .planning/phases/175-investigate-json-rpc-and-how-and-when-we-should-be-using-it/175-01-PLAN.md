<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent→MCP server | Untrusted MCP protocol format; JSON-RPC could standardize error handling |
| BFF→token exchange | Internal call; JSON-RPC would standardize request/response correlation |

## STRIDE Considerations

Since this is a research phase:
- **Spoofing:** JSON-RPC request IDs help prevent reply confusion; not a threat here
- **Tampering:** No new attack surface if JSON-RPC adopted for internal calls only
- **Repudiation:** Request IDs improve audit trail (positive)
- **Information Disclosure:** No secrets in JSON-RPC structure itself
- **Denial of Service:** Batch requests in JSON-RPC could be DoS vector if not rate-limited (future consideration)
- **Elevation of Privilege:** No privilege impact from JSON-RPC adoption

**Disposition:** Research phase; no implementation means no new threats. Defer threat assessment to Phase 176–180 if JSON-RPC is adopted.
</threat_model>
