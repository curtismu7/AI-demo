# Plan 230-01: Authorize gate skip reason

## Goal

Every `{ ran: false }` from the authorize gate must carry a `reason` string so call sites can log it to the activity log and surface it to the user, not silently continue.

## Scope

3 files changed:
- `banking_api_server/services/transactionAuthorizationService.js` ‚Äî compute skip reason before the single `!SHOULD_RUN` guard; return `{ ran: false, reason }`
- `banking_api_server/services/mcpToolAuthorizationService.js` ‚Äî add reason to each of the 4 early-exit `{ ran: false }` returns
- `banking_api_server/server.js` ‚Äî include `reason` in `authorize_gate_skipped` emit; fire `logAppEvent('authorize', 'warning', ...)`
- `banking_api_server/routes/transactions.js` ‚Äî add `else` branch after `if (authz.ran)` block to fire `logAppEvent('authorize', 'info', ...)` with the skip reason

## Reason taxonomy

### transactionAuthorizationService
| Condition | reason |
|---|---|
| `!AUTHORIZE_ENABLED` | `'authorize_disabled'` |
| `userRole === 'admin'` | `'admin_role_exempt'` |
| `!AUTHORIZE_TYPES.includes(type)` | `'type_not_in_scope'` |
| `!(USE_SIMULATED \|\| PINGONE_READY)` | `'not_configured'` |

### mcpToolAuthorizationService
| Condition | reason |
|---|---|
| `!flag` | `'feature_flag_disabled'` |
| `!agentToken` | `'no_agent_token'` |
| `mcpFirstToolAuthorizeDone` | `'already_evaluated'` |
| `role === 'admin'` | `'admin_role_exempt'` |

## Tasks

### Task 1 ‚Äî transactionAuthorizationService.js: compute and return reason

Replace the existing `!SHOULD_RUN ‚Üí return { ran: false }` block with individual condition checks, each returning `{ ran: false, reason }`.

Conditions are checked in priority order (disabled ‚Üí admin ‚Üí type ‚Üí not_configured).

### Task 2 ‚Äî mcpToolAuthorizationService.js: add reason to each early exit

Add `reason` string to all 4 existing `return { ran: false }` statements.

### Task 3 ‚Äî server.js: surface MCP gate skip reason

In the `if (!mcpAuthz.ran)` block (line ~1321):
- Add `reason: mcpAuthz.reason` to the `authorize_gate_skipped` emit
- Fire `logAppEvent('authorize', 'info', 'Authorize gate skipped ‚Äî ' + (mcpAuthz.reason || 'unknown'), { tag: 'authorize/gate-skipped', metadata: { reason: mcpAuthz.reason } })`

Add `const { logEvent: logAppEvent } = require('./services/appEventService');` if not already imported.

### Task 4 ‚Äî transactions.js: log skip in activity log

After `if (authz.ran) { ... }` block, add:
```js
} else {
  logAppEvent('authorize', 'info',
    'Authorize gate skipped ‚Äî ' + (authz.reason || 'unknown'),
    { tag: 'authorize/gate-skipped', metadata: { reason: authz.reason, type, userId: req.user?.id } });
}
```

Add `const { logEvent: logAppEvent } = require('../services/appEventService');` at top if not already present.
