   - `ANTHROPIC_MODEL = process.env.ANTHROPIC_MODEL || 'claude-3-5-haiku-20241022'`
   - `ANTHROPIC_TIMEOUT_MS = 10000`

3. Add `parseWithAnthropic(userMessage, context = {})` function:
   - If `!ANTHROPIC_API_KEY` return null (skip if no key)
   - Build `systemWithCtx` same pattern as parseWithLmStudio (role-aware system prompt)
   - Use `AbortController` with `ANTHROPIC_TIMEOUT_MS` timeout
   - POST to `https://api.anthropic.com/v1/messages` with headers:
     - `Content-Type: application/json`
     - `x-api-key: ${ANTHROPIC_API_KEY}`
     - `anthropic-version: 2023-06-01`
   - Body: `{ model: ANTHROPIC_MODEL, max_tokens: 256, system: systemWithCtx, messages: [{ role: 'user', content: userMessage }] }`
   - Parse response: `data?.content?.[0]?.text`
   - Strip markdown fences if present (same as Gemini did)
   - JSON.parse the text, return if valid `{ kind: ... }` object
   - Catch AbortError for timeout, log other errors
   - Return null on any failure

4. Update `parseNaturalLanguage` chain — step 3 becomes Anthropic instead of Gemini:
   - Replace the Gemini block with identical structure calling `parseWithAnthropic`
   - Update the source string from `'gemini'` to `'anthropic'`

   `LLM intent parsing — priority: Groq → LM Studio → Anthropic → heuristic regex.`

6. Update the return type JSDoc for parseNaturalLanguage:
