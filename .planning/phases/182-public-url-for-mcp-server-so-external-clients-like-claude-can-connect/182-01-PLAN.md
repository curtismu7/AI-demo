**ingress.yaml** (per D-05, D-06, D-09, D-10, D-11):
```yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: banking-mcp-server
  namespace: banking-demo
  labels:
    app: banking-mcp-server
  annotations:
    # WebSocket support (per D-06) — increase timeouts for long-lived connections
    nginx.ingress.kubernetes.io/proxy-read-timeout: "3600"
    nginx.ingress.kubernetes.io/proxy-send-timeout: "3600"
    nginx.ingress.kubernetes.io/proxy-connect-timeout: "60"
    nginx.ingress.kubernetes.io/websocket-services: "banking-mcp-server"
    # Rate limiting (per D-09) — 60 req/min ≈ 1 req/sec
    nginx.ingress.kubernetes.io/limit-rps: "1"
    nginx.ingress.kubernetes.io/limit-burst-multiplier: "5"
    # TLS — uncomment if using cert-manager
    # cert-manager.io/cluster-issuer: "letsencrypt-prod"
spec:
  ingressClassName: nginx    # Adjust to match your ingress controller
  tls:
    - hosts:
        - api.pingdemo.com
      secretName: api-pingdemo-com-tls    # Pre-existing or cert-manager managed
  rules:
    - host: api.pingdemo.com
      http:
        paths:
          - path: /
            pathType: Prefix
            backend:
              service:
                name: banking-mcp-server
                port:
                  number: 8080
```

## Kubernetes Deployment

Deploy the MCP server to an existing Kubernetes cluster (tested on EKS).

```bash
# Review and apply all manifests
kubectl apply -k k8s/

kubectl -n banking-demo get ingress
```

```bash
# Health check
curl https://api.pingdemo.com/health

# MCP discovery
curl https://api.pingdemo.com/.well-known/mcp-server

# OAuth protected resource metadata (RFC 9728)

- **Sensitive config:** `k8s/secret.yaml` (PingOne credentials, encryption key)
- **Ingress:** `k8s/ingress.yaml` (hostname, TLS, rate limiting, WebSocket timeouts)

See `k8s/` directory for all manifests. Uses Kustomize for management.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Ingress | Untrusted traffic reaches K8s ingress from public internet |
| Ingress → MCP Pod | Ingress forwards to internal ClusterIP service |
| MCP Pod → PingOne | Outbound calls to PingOne for token introspection |
| MCP Pod → Banking API | Outbound calls to BFF for banking operations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-182-01 | Spoofing | Ingress | mitigate | OAuth 2.0 Protected Resource (RFC 9728) — tools require valid PingOne tokens; `/.well-known/oauth-protected-resource` advertises auth requirements |
| T-182-02 | Denial of Service | Ingress | mitigate | nginx rate limiting annotation `limit-rps: 1` (60/min) with burst multiplier 5 |
| T-182-03 | Information Disclosure | Secret YAML | mitigate | PingOne credentials in K8s Secret (not ConfigMap); secret.yaml uses REPLACE_ME placeholders — never committed with real values |
| T-182-04 | Elevation of Privilege | Container | mitigate | `runAsNonRoot: true`, `runAsUser: 1001`, `allowPrivilegeEscalation: false` in deployment securityContext |
| T-182-05 | Tampering | CORS | mitigate | `MCP_ALLOWED_ORIGINS` restricts HTTP Streamable origins to Vercel domain; existing `isOriginAllowed()` in HttpMCPTransport enforces |
| T-182-06 | Repudiation | Audit | accept | AuditLogger already captures tool calls — no additional K8s audit needed for demo |
</threat_model>
