Source: TokenDisplay.css verified hex values + DevToolsDashboard.jsx inline styles.

surface inside DecodedTokenPanel. New audit trail rows must bridge both zones.

|------|-------|-------|
| Dominant (60%) — panel background | #1a202c | Outer floating panel body, tab content area background |
| Secondary (30%) — card / row surfaces | #2d3748 | Tab bar background; audit trail row hover state |
| Accent (10%) — active / highlight | #2563eb | Active tab box-shadow highlight (existing); actor token badge |
| DecodedTokenPanel surface | #f8fafc | Light card background inside each decoded panel (existing) |
| Muted text | #94a3b8 | Timestamps, empty state copy, hint text |

| Destructive / subject badge | #dc2626 | Subject token category badge background; active tab button background |

| Scope pill | bg #e0f2fe, text #0369a1 | Existing `.decoded-scope-badge` — reuse unchanged |

Accent reserved for: active tab box-shadow bottom highlight, actor badge backgrounds.

semantic for "subject/user token" category. There are no destructive actions in this phase.

### Dark-to-light zone transition rule
Audit trail rows render on the dark panel background (#1a202c). When a row expands, the
DecodedTokenPanel cards inside use their existing light surface (#f8fafc). This contrast
is intentional — the white card visually separates expanded detail from the dark row list.
Do not override DecodedTokenPanel's background color in expanded rows.

## Interaction Contract

Two tabs are appended to the existing `TABS` array:

### Audit Trail Tab

**Layout:** Vertical scrollable list. `flex-direction: column`, `overflowY: auto`, `height: 100%`.

**Data source:** `useTokenChainOptional()` → `context.history` (Array of `{ tool, timestamp, events[] }`,
newest first). Also include `context.sessionTokenEvent` as a synthetic entry at the bottom of
the list (oldest) if present, with tool name rendered as `"session"`.

**Empty state:** When `history.length === 0` and no sessionTokenEvent, render centered
muted copy — see Copywriting Contract.

**Row anatomy (collapsed):**

```
[ ▸ ] [ 14:23:05 ] [ get_account_balance ] [ 🔵 actor ] [ 🔴 subject ] [ 🟢 mcp ]
```

- Chevron: `▸` collapsed / `▾` expanded. Color: #94a3b8. Size: 0.75rem.
- Timestamp: `toLocaleTimeString()` of `entry.timestamp`. Style: 12px, #94a3b8, monospace.
- Tool name: `entry.tool`. Style: 14px, weight 600, #f1f5f9.
- Category badges: one per distinct token category present in `entry.events` that has
  `jwtFullDecode != null`. Use `deriveTokenCategory(event.label, event.id)` → background
  from BADGE_BG map. Render as `<span>` with inline background style. Pill shape:
  border-radius 4px, padding 2px 8px, font 0.7rem weight 600, color #fff.
- Row click target: full row width, min-height 40px.
- Hover: background #2d3748.
- Separator: 1px solid #1e293b bottom border on each row.

**Row expand (D-01, D-02, D-03):**

Expand state: `useState(new Set())` keyed by `${entry.tool}-${entry.timestamp}`. Multiple
rows may be open simultaneously (D-03). Toggled by clicking anywhere on the row header.

Expanded section renders directly below the row header (inline expand — no modal, no slide
pane). Contains one `DecodedTokenPanel` per token event in `entry.events` that has
`jwtFullDecode != null`.

**CRITICAL prop adapter** — must be applied at every callsite:
```jsx
const decoded = evt.jwtFullDecode
  ? { header: evt.jwtFullDecode.header, payload: evt.jwtFullDecode.claims }
  : null;
<DecodedTokenPanel decoded={decoded} label={evt.label} />
```

DecodedTokenPanel renders in its default collapsed state inside the expanded row.
User clicks its own "▼ show / ▲ hide" toggle to see claims. This two-level expand
is intentional — the outer expand reveals which tokens were acquired; the inner expand
shows decoded claims.

Expanded area background: #1a202c (same as panel — DecodedTokenPanel provides its own
white card surface #f8fafc).

**Row key:** `${entry.tool}-${entry.timestamp}` (not index — prevents key collisions on
rapid sequential calls with index-based shift). Fallback: index if timestamp is missing.

### Token Decoder Tab

**Layout:** Horizontal scrollable flex row of columns.

```jsx
display: "flex"
flexDirection: "row"
gap: "0.75rem"       // 12px
overflowX: "auto"
padding: "1rem"      // 16px
flex: 1
minHeight: 0
```

**Data source:** `useTokenChainOptional()` → `context.events` (displayEvents alias — current
tool call events, or session token if no tool call yet). Filter to events where
`event.jwtFullDecode != null`.

**Column spec:**
- width: `minWidth: 280px, flex: "0 0 280px"` — fixed, not stretchy.
- Shows 2–3 columns without scroll on default 1100px panel width.
- Each column is one `DecodedTokenPanel` using the prop adapter above.
- DecodedTokenPanel renders in its **default collapsed** state. Users click "▼ show"
  on each column to expand claims. This matches the CONTEXT.md screenshot showing
  highlighted fields — the highlighted state is after the user clicked expand.

**Empty state:** When no events have `jwtFullDecode`, render centered muted copy — see
Copywriting Contract.

**Column header:** `DecodedTokenPanel`'s existing header provides the label and color coding.
No additional column title wrapper is needed.
