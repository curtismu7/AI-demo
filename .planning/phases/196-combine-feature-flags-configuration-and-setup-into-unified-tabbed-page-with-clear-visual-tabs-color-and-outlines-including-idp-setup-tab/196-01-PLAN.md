## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client (browser) → BFF API | Web page displays application configuration to authenticated admins; configuration values are educational/non-sensitive (URLs, environment IDs, client IDs but no secrets) |
| Admin role access | Page requires authentication; should be restricted to admin users only (existing auth middleware in BFF) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-196-01  | Information Disclosure | UnifiedConfigPage displays config | Mitigate | Only display non-sensitive values (no client secrets, API keys); add comment in component "Educational demo — all values visible for transparency" |
| T-196-02  | Spoofing auth access | Unauthenticated access to /config | Mitigate | requiresAuth: true in route; verify React Router guards work (check App.js middleware) |
| T-196-03  | Elevation of privilege | Non-admin user reads admin config | Mitigate | Existing BFF session checks enforce auth level; no code changes required; test with logged-in user role |

</threat_model>
