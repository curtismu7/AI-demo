<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → BFF | Unauthenticated attacker tries to bypass HITL by sending POST /api/transactions without consentChallengeId |
| Session → Authorization Server | Session hijacker could forge a consentChallengeId; storage must use server-side session (not client cookie) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-170-01 | Tampering | consentChallengeId in request body | mitigate | Challenge ID is cryptographically random, 24 bytes (192 bits) entropy; attacker cannot guess; return 428 on mismatch |
| T-170-02 | Spoofing | Attacker claims to own transfer source account | mitigate | Call `verifyAndConsumeChallenge()` which validates `challenge.userId === req.user.id` before allowing execution |
| T-170-03 | Repudiation | User claims they did not consent to transfer | accept | OTP email audit trail + session-backed challenge log provides accountability; out of scope for code changes |
| T-170-04 | Denial | Session expires mid-OTP verify | accept | Challenge TTL is 10 min; OTP TTL is 5 min; user must restart; acceptable trade-off |
</threat_model>
