## RFC 8693 Token Terminology

### Subject Token (RFC 8693 §2.1)
The user's authorization identity requesting an operation.
**In this demo:** User logs in via PingOne, receives access token (subject_token).
**Key claim:** `sub` = user ID

### Actor Token (RFC 8693 §2.2)
The agent's identity acting on behalf of the subject.
**In this demo:** Agent has client credentials token (actor_token); combines with subject_token in dual-exchange.
**Key claim:** `act` = agent ID

### Resource / Audience (RFC 8693 §2.3)
The target API/service the delegated token will access.
**In this demo:** MCP server is the resource; audience = https://mcp-server.pingdemo.com
**Key claim:** `aud` = audience URI

[See TOKEN_TERMINOLOGY_GLOSSARY.md for complete reference]
