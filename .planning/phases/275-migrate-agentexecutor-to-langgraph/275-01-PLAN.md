<objective>
Install langgraph and replace the per-request AgentExecutor rebuild pattern in
LangChainMCPAgent with a single compiled LangGraph StateGraph.

Purpose: The current code rebuilds a full AgentExecutor (including creating a
ConversationBufferMemory object and pre-populating it from ConversationMemory) on
every single message. This is the textbook hand-rolled anti-pattern — LangGraph's
MemorySaver checkpointer eliminates the rebuild by storing conversation state in the
graph itself, keyed by thread_id=session_id. The per-request rebuild also prevents
parallel tool calls (ToolNode) and proper astream_events streaming.

Output:
- requirements.txt gains langgraph>=0.2.0,<1.0.0
- langchain_mcp_agent.py: _get_agent_executor_for_session() deleted; self._graph built once in initialize_tools(); ainvoke in process_message_with_tracing uses thread_id=session_id
- All other public methods (process_message, execute_tool, initialize_session_with_token, etc.) remain intact
</objective>

## STRIDE Threat Register

|-----------|----------|-----------|-------------|-----------------|
| T-275-01 | Information Disclosure | MemorySaver thread_id keying | mitigate | thread_id = session_id which is a UUID already validated by session_manager; MemorySaver is in-process dict — no cross-session leak possible unless caller sends wrong session_id |
| T-275-02 | Tampering | SystemMessage injection per turn | mitigate | Inject SystemMessage only on first turn (empty checkpoint); subsequent turns omit it to prevent prompt injection via accumulated system messages |
