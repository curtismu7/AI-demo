### 2. Wizard Steps — Real PingGateway Tab

**Wizard layout:**
- Wrapper: stacked flex container (`.mgc-panel`)
- Step list: ordered list (1–5), each step row with:
  - Circle number: 24px × 24px, centered, font 12px bold
  - Status indicator: (✓ = complete, ⚠ = needs action, ○ = pending)
  - Step label: 16px, 600 weight
  - Step description: 12px, 400 weight (optional, shown on expand)
  - Expandable form/content area (only Step 2 has form fields)

**Step colors (CSS classes to create):**
- Complete step circle: green background (#28a745), white text
- Pending step circle: light gray background (#e2e3e5), gray text
- Needs-action step circle: yellow background (#ffc107), dark text
- Active step: blue bottom border on label

**Step 1: Verify PingOne Credentials**
- Read-only status display (no form inputs)
- Shows: environment ID, region, token endpoint
- All auto-filled from configStore
- Status: green check (✓) if all present; red (⚠) if missing
- Hint text: "Set your PingOne Environment ID in Configuration first" (with `/config` link, if missing)

**Step 2: Configure Gateway Routes**
- 5 form fields (see below)
- 2 read-only derived fields with "🔗 From PingOne config" chip
- Status: ⚠ (needs input) until "Save to Config" clicked
- Save button: blue (#0070f3), text "⬆ Save to Config" (or "Save" after first save)
- Success/error inline below button

**Step 3: Download Route File**
- Copy button: "📋 Copy mcp.json"
- Download button: "⬇ Download mcp.json" (new for Phase 264)
- Placement hint: "$HOME/.openig/config/routes/mcp.json"
- Status: ✓ (complete, read-only button state)

**Step 4: Configure admin.json**
- Copy button: "📋 Copy admin.json snippet"
- Hint: "Merge into your existing admin.json; streamingEnabled: true required"
- Status: ✓ (complete, read-only)

**Step 5: Point BFF to Real Gateway**
- Ordered list of 5 steps (instructions)
- Copy-able code blocks (env var values)
- Status: ✓ (complete, read-only)

### 3. Form Fields (within Step 2)

**Form structure:**
- Container: `.mgc-push-form` (flex column, gap: 10px)
- Each field: `.mgc-field` with `.mgc-field-label`, `.mgc-input`, `.mgc-field-hint`

**Field 1: pingOneEnvID (read-only)**
- Label: "PingOne Environment ID" (16px, 600 weight)
- Value: auto-derived from configStore
- Chip: "🔗 From PingOne config" (blue text, no background)
- Not editable; gray text (#999)

**Field 2: pingOneResourceID (required input)**
- Label: "PingOne Resource ID" + red "Required" badge (if empty) (16px, 600 weight)
- Placeholder: "e.g., a1b2c3d4-e5f6-7890-abcd-ef1234567890"
- Hint: "Used as `username` in OAuth2ResourceServerFilter for introspection" (12px, 400 weight)
- Red badge: background #ffc107 (yellow), text #333, padding 4px 8px, font 12px, weight 600

**Field 3: gatewayPublicUrl (required input)**
- Label: "PingGateway Public URL" + red "Required" badge (if empty) (16px, 600 weight)
- Placeholder: "https://ig.example.com:8443"
- Hint: "The public HTTPS URL of your PingGateway instance (used as `gatewayUrl` in mcp.json)" (12px, 400 weight)

**Field 4: mcpServerUrl (read-only)**
- Label: "Upstream MCP Server URL" (16px, 600 weight)
- Value: auto-derived from configStore (`mcp_server_url`)
- Chip: "🔗 From PingOne config"
- Not editable

**Field 5: mcpScope (input with default)**
- Label: "MCP Scope" (16px, 600 weight)
- Value: default "banking:mcp:invoke" (pre-filled, editable)
- Hint: "OAuth 2.0 scope required for token exchange; matched against gateway scope requirement" (12px, 400 weight)

**Field 6: introspectEndpoint (read-only derived)**
- Label: "Token Introspection Endpoint" (16px, 600 weight)
- Value: auto-computed as `${pingOneEnvUrl}/as/introspect`
- Chip: "🔗 From PingOne config"
- Not editable

### 4. Live JSON Preview

**Layout:**
- Below Step 2 form, within same section
- Container: `.mgc-code-block`
- JSON displayed in `<pre class="mgc-pre mgc-pre--code">`
- Buttons above JSON: "📋 Copy mcp.json" + "⬇ Download mcp.json" (new)
- Updates client-side in real-time as user types in Step 2 form fields (no API call)

**JSON structure (must match PingGateway 2025.11.1):**
```json
{
  "name": "mcp",
  "condition": "...",
  "properties": {
    "pingOneEnvID": "user input or derived",
    "pingOneResourceID": "user input",
    "gatewayUrl": "user input",
    "mcpServerUrl": "derived",
    "mcpScope": "user input with default"
  },
  "baseURI": "derived from mcpServerUrl",
  "heap": [...],
  "filters": [...]
}
```

### 5. Download mcp.json Button

**New button for Phase 264:**
- Label: "⬇ Download mcp.json"
- Style: same as copy button (.mgc-copy-btn), blue background (#0070f3)
- Placement: alongside "📋 Copy mcp.json" button (float right in `.mgc-code-block`)
- Behavior: triggers browser download of `mcp.json` file (data: URL or fetch)
- Hover: background #0060d0 (darker blue, same as copy button)

## Implementation Notes

### Pre-Population Strategy (D-04)

Auto-derive on GET `/api/admin/mcp-gateway/config`:
- `pingOneEnvUrl` = `https://auth.pingone.${region}/${envId}` (from configStore)
- `upstreamMcpUrl` = `mcp_server_url` (existing)
- `gatewayPublicUrl` = `MCP_GW_RESOURCE_URI` env var (strip `/mcp` suffix if present)
- `pingOneResourceId` = `mcp_gw_client_id` (configStore or env)
- `mcpScope` = default `banking:mcp:invoke`
- `introspectEndpoint` = `${pingOneEnvUrl}/as/introspect` (computed)

### Form Persistence (D-11)

POST `/api/admin/mcp-gateway/config` with new allowed keys:
- `mcp_gw_client_id` (pingOneResourceID)
- `mcp_gw_public_url` (gatewayPublicUrl)
- `mcp_scope` (mcpScope)

Values persisted to configStore, survive BFF restart.

### Test Coverage (D-18, D-19)

Tests to add in Phase 264 execution:
- GET `/api/admin/mcp-gateway/config` → 5 new fields present and correctly derived
- POST `/api/admin/mcp-gateway/config` → new keys accepted and saved
- `pingGatewayJson` response → correct PingGateway 2025.11.1 schema
- Mock gateway 401/403 → `WWW-Authenticate` header present with RFC 9728 format
- Build: `npm run build` exit code 0 (after UI changes)

### Mock Gateway Compliance (D-16)

WWW-Authenticate header format:
```
Bearer realm="PingOne", resource_metadata="https://mcp-gw.ping.demo/.well-known/mcp-server"
```

(Only in mock gateway responses; BFF responses unchanged)
