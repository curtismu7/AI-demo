# Plan 239-01 — PingOne API Audit: verify BFF request/response shapes

## Goal
Read the official PingOne API docs for every endpoint the BFF calls and compare against the actual
implementation. Produce a structured FINDINGS.md listing any mismatches. No code changes yet.

## Reference docs to check
- https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token
- https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-authorize
- https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token-introspection
- https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-decision
- https://developer.pingidentity.com/apis.html (index)

## Tasks

### T1 — Audit OAuth token endpoint (`POST /as/token`)
Read `banking_api_server/services/oauthService.js` and `agentMcpTokenService.js`.
Check against PingOne docs:
- `grant_type=authorization_code`: required fields (`code`, `redirect_uri`, `code_verifier`)
- `grant_type=client_credentials`: required fields (`scope`)
- `grant_type=urn:ietf:params:oauth:grant-type:token-exchange`: RFC 8693 fields
  (`subject_token`, `subject_token_type`, `actor_token`, `actor_token_type`, `requested_token_type`, `audience`, `scope`)
- Response shape: `access_token`, `token_type`, `expires_in`, `scope`; error: `error`, `error_description`

For each: document the actual code's request body and compare to the documented curl example.

### T2 — Audit token introspection (`POST /as/introspect`)
Read `banking_api_server/services/tokenIntrospectionService.js`.
Check: `token` param in form body, `Authorization: Basic` header with client_id:client_secret.
Response: RFC 7662 `active` boolean + standard claims. Compare to PingOne docs.

### T3 — Audit PingOne Authorize decisions endpoint
Read `banking_api_server/services/pingOneAuthorizeService.js`.
Check: POST body shape (decisionEndpointId vs policyId path), parameters object, response envelope.
PingOne Authorize response: `result.decision` = "PERMIT" | "DENY", `result.statements`, `result.obligations`.
Compare to simulated response in `simulatedAuthorizeService.js` — list any field mismatches.

### T4 — Audit CIBA / MFA push endpoint
Read `banking_api_server/services/cibaService.js`.
Check `POST /as/bc-authorize` and `POST /as/token` with `grant_type=urn:openid:params:grant-type:ciba`.
Compare required fields to PingOne CIBA docs.
