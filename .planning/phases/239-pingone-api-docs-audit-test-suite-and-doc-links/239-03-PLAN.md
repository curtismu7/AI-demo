# Plan 239-03 — Add PingOne doc links + live API+request display to all test pages

## Goal
Every test page (PingOne Test, MFA Test, Authz Test) must show:
1. The actual PingOne API endpoint URL being called

### T2 — Create shared `ApiCallPreviewCard` component
**File:** `banking_api_ui/src/components/shared/ApiCallPreviewCard.js`
**File:** `banking_api_ui/src/components/shared/ApiCallPreviewCard.css`

```js
// Renders: endpoint URL + request JSON preview + doc link
// Props:
//   method: 'POST' | 'GET'
//   endpoint: string (full URL or path)
//   requestBody: object | null
//   docUrl: string
//   docLabel: string
//   description: string

export default function ApiCallPreviewCard({ method, endpoint, requestBody, docUrl, docLabel, description }) {
  return (
    <div className="acpc-root">
      <div className="acpc-header">
        <span className={`acpc-method acpc-method--${method.toLowerCase()}`}>{method}</span>
        <span className="acpc-endpoint">{endpoint}</span>
        {docUrl && (
          <a href={docUrl} target="_blank" rel="noopener noreferrer" className="acpc-doc-link">
            {docLabel || 'PingOne Docs'} ↗
          </a>
        )}
      </div>
      {description && <div className="acpc-desc">{description}</div>}
      {requestBody && (
        <div className="acpc-body">
          <div className="acpc-body-label">Request body</div>
          <pre className="acpc-json">{JSON.stringify(requestBody, null, 2)}</pre>
        </div>
      )}
    </div>
  );
}
```
