# Plan 239-04 ‚Äî PingOne API integration test suite

## Goal
Build and maintain a BFF-side integration test suite (`banking_api_server/tests/pingone-api.test.js`) that exercises
each PingOne API the BFF calls with real HTTP requests and validates response shapes against the documented
PingOne schema. Tests run with `npm test` in `banking_api_server/`.

Docs reference: https://apidocs.pingidentity.com/pingone/platform/v1/api/

## Scope

- `banking_api_server/tests/pingone-api.test.js`
- `banking_api_server/jest.config.js` ‚Äî testMatch extended to include `**/tests/**/*.test.js`
- `banking_api_server/services/tokenIntrospectionService.js` ‚Äî fixed: credentials moved to `Authorization: Basic` header (RFC 7662 ¬ß2.1)
- `banking_api_server/PINGONE_API_FINDINGS.md` ‚Äî audit doc kept in sync

## Pre-conditions

Tests require live PingOne credentials. The following env vars must be set in `banking_api_server/.env`:

```
PINGONE_ENVIRONMENT_ID
PINGONE_REGION                           # default: com
PINGONE_WORKER_TOKEN_CLIENT_ID
PINGONE_WORKER_TOKEN_CLIENT_SECRET
PINGONE_AUTHORIZE_DECISION_ENDPOINT_ID   # for Authorize suite
PINGONE_AUTHORIZE_WORKER_CLIENT_ID       # for Authorize suite
PINGONE_AUTHORIZE_WORKER_CLIENT_SECRET
PINGONE_MCP_TOKEN_EXCHANGER_CLIENT_ID    # for RFC 8693 suite
PINGONE_MCP_TOKEN_EXCHANGER_CLIENT_SECRET
PINGONE_RESOURCE_MCP_SERVER_URI          # audience for token exchange
```

Tests skip gracefully (not fail) when credentials are absent so CI never breaks.

## Tasks

### T1 ‚Äî Test file with credential setup and helpers

File: `banking_api_server/tests/pingone-api.test.js`

- Load `.env` via `dotenv` at top of file
- Derive `AUTH_URL` from `PINGONE_REGION` + `PINGONE_ENVIRONMENT_ID`
- `PINGONE_WORKER_TOKEN_CLIENT_ID/SECRET` (not `PINGONE_CLIENT_ID`) are the correct env var names
- `getWorkerToken()` helper: POST `/as/token` with `client_credentials` + Basic auth
- `HAVE_CREDENTIALS` guard: skip all live tests if any required var is missing
- Separate `HAVE_AUTHZ`, `HAVE_EXCHANGE` guards for suites that need extra credentials

### T2 ‚Äî POST /as/token ‚Äî client_credentials

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token

- Returns `access_token`, `token_type: Bearer`, `expires_in` (RFC 6749 ¬ß5.1)
- Returns `{ error, error_description }` on bad credentials (HTTP 401)

### T3 ‚Äî POST /as/introspect ‚Äî RFC 7662

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token-introspection

- Returns `active: false` for invalid token (HTTP 200, not 4xx)
- Returns `active: true` + `iss`, `exp`, `iat` for valid CC token
- `sub` may be absent for CC tokens ‚Äî `sub || client_id` must be truthy
- **Note:** PingOne does NOT return `scope` in introspection for CC tokens in all environments ‚Äî assertion must be conditional

### T4 ‚Äî POST /as/token ‚Äî worker token claims

- JWT has 3 dot-separated segments
- Payload contains `iss` (contains "pingone"), `iat`, `exp` (future)
- `token_type: Bearer`, `expires_in` is positive integer
- `scope` narrowing: conditional check (PingOne may omit `scope` from CC introspection)
- Unsupported `grant_type` returns 400 + `{ error }` shape

### T5 ‚Äî GET /v1/environments/{id}/mfaPolicies

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-mfa-policies

- Returns `_embedded.mfaPolicies` array with `id`, `name` fields
- At least one policy has `default: true`
- No-auth request returns 4xx
- **Note:** PingOne returns **403** (not 401) for unauthenticated Management API requests
- **Note:** Worker app requires **MFA Admin role** in PingOne console ‚Äî test skips gracefully with `console.warn` if 403

### T6 ‚Äî GET + User CRUD ‚Äî /v1/environments/{id}/users

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-users

- `GET /users` returns `_embedded.users` array with `id`, `username`
- Filter query `?filter=(username sw "test")` returns 200
- No-auth request returns 401
- Full CRUD lifecycle: POST (201) ‚Üí GET ‚Üí PATCH ‚Üí DELETE (204)
- `GET /users/{id}/devices` returns empty devices array for new user
- `afterAll` safety cleanup if mid-suite failure leaves test user behind

### T7 ‚Äî GET /v1/environments/{id}/decisionEndpoints

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-decision-endpoints

- Returns array with `id`, `name` fields
- Configured `DECISION_ENDPOINT_ID` appears in list

### T8 ‚Äî POST /v1/environments/{id}/decisionEndpoints/{id} ‚Äî Authorize evaluation

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-evaluate-decision-endpoint

Trust Framework params: `Amount`, `TransactionType`, `UserId`, `Timestamp`

- Small transfer ‚Üí decision is one of `PERMIT`, `DENY`, `INDETERMINATE`, `STEP_UP`
- Large transfer ($999,999) ‚Üí decision is `DENY`, `STEP_UP`, or `INDETERMINATE`
- No-auth request returns 4xx
- **Note:** PingOne Authorize rate-limits rapid back-to-back calls ‚Äî add `beforeEach` delay of 1500ms
- **Note:** Response `id`/`decisionId` field is policy-config-dependent and may be absent

### T9 ‚Äî POST /as/token ‚Äî RFC 8693 token exchange

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token  
RFC: https://datatracker.ietf.org/doc/html/rfc8693

Request fields:
- `grant_type: urn:ietf:params:oauth:grant-type:token-exchange`
- `subject_token` + `subject_token_type: urn:ietf:params:oauth:token-type:access_token`
- `requested_token_type: urn:ietf:params:oauth:token-type:access_token`
- `audience` = MCP resource URI
- `scope` = requested scopes
- Exchanger credentials: `PINGONE_MCP_TOKEN_EXCHANGER_CLIENT_ID/SECRET` (post auth method)

Tests:
- Invalid `subject_token` ‚Üí RFC 6749 error shape (`{ error }`)
- Valid exchange ‚Üí success shape: `access_token`, `issued_token_type`, `token_type: Bearer`, `expires_in` ‚Äî or documented policy rejection (4xx, never 5xx)
- Issued JWT `aud` claim narrows to MCP resource URI (RFC 8707)
- Missing `grant_type` ‚Üí HTTP 400 + `{ error }`

### T10 ‚Äî Simulated Authorize parity (always runs, no credentials)

- PERMIT response matches PingOne Authorize envelope shape
- DENY response has identical envelope structure
- Step-up response includes `IDENTITY_REQUIREMENT` obligation

### T11 ‚Äî jest.config.js update

Add `'**/tests/**/*.test.js'` to `testMatch` so Jest discovers `tests/` directory.

### T12 ‚Äî tokenIntrospectionService.js fix

Change from form-body `client_id`/`client_secret` to `Authorization: Basic` header per RFC 7662 ¬ß2.1.
Send only `token` in the request body.

### T13 ‚Äî GET /v1/environments/{id}/resources ‚Äî Resource Servers

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-resources

- Returns `_embedded.resources` array with `id`, `name`
- Unauthenticated request returns 4xx

### T14 ‚Äî GET /v1/environments/{id}/resources/{id}/scopes ‚Äî Resource Scopes

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-resource-scopes

- Uses first resource ID from T13 response
- Returns `_embedded.scopes` array with `id`, `name`

### T15 ‚Äî GET /v1/environments/{id}/applications ‚Äî Applications

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-applications

- Returns `_embedded.applications` array with `id`, `name`, `protocol`
- OIDC filter `protocol eq "OPENID_CONNECT"` returns only OIDC apps
- Unauthenticated request returns 4xx

### T16 ‚Äî GET /v1/environments/{id}/applications/{id} ‚Äî Single Application

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-one-application

- Uses first app ID from T15 list response
- Returns object with `id`, `name`, `enabled`, `protocol`

### T17 ‚Äî GET /v1/environments/{id}/applications/{id}/grants ‚Äî Application Grants

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-application-resource-grants

- Uses first OIDC app ID from T15 filter response
- Returns `_embedded.grants` array (may be empty for apps with no resource grants)

### T18 ‚Äî GET /v1/environments/{id}/populations ‚Äî Populations

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-populations

- Returns `_embedded.populations` array with `id`, `name`
- Unauthenticated request returns 4xx

### T19 ‚Äî GET /as/.well-known/openid-configuration ‚Äî OIDC Discovery

Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-discovery-endpoint
RFC: https://www.rfc-editor.org/rfc/rfc8414

- URL: `{AUTH_URL}/as/.well-known/openid-configuration` (NOT `/.well-known/` ‚Äî PingOne returns 403 without `/as/` prefix)
- Returns standard OIDC fields: `issuer`, `authorization_endpoint`, `token_endpoint`, `jwks_uri`, `response_types_supported`, `subject_types_supported`, `id_token_signing_alg_values_supported`
- Optional RFC 9728: `introspection_endpoint` if present must be a string
- Optional RFC 8693: `grant_types_supported` if present must contain `authorization_code`
