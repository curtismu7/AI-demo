## Streaming Response Format

Current: SSE with `data: {...}` lines (works over HTTP/1.1 and HTTP/2)
Proposed: Same format, now multiplexed over HTTP/2 streams instead of polling

Example stream payload after POST /api/mcp/tool:
```
data: {"type":"flow_event","step":"tool_start","tool":"get_my_accounts","timestamp":1713...}
data: {"type":"flow_event","step":"tool_end","result":{...},"timestamp":1713...}
data: {"type":"stream_close","status":"success"}
```

## Current Polling Limitation
- Agent makes POST /api/mcp/tool
- Response is JSON final result
- BFF publishes SSE to /api/mcp/tool/events
- Agent polls SSE in parallel (separate connection)
- Flow diagram updates delayed by polling interval

## HTTP/2 Streaming Improvement
- Agent makes POST /api/mcp/tool
- Response is chunked stream with embedded flow events
- No separate SSE poll needed
- Events arrive in real time as MCP server processes

2. **Detect stream vs. non-stream response:**
   ```javascript
   if (response.headers.get('content-type')?.includes('application/stream+json')) {
     // New: HTTP/2 streaming response
     return parseStreamingResponse(response.body);
   } else {
     // Fallback: traditional JSON response
     return response.json();
   }
   ```

3. **Implement streaming parser:**
   - Create helper: `parseStreamingResponse(readableStream)`
   - Buffer incoming data chunks
   - On complete JSON object (ends with `}`), parse and emit
   - Handle mid-transfer fragmentation (JSON split across chunks)
   - Extract flow events (if present in stream) and send to agentFlowDiagram
   - Return final result object when stream ends

4. **Flow event extraction:**
   - Look for `flow_event` type in stream objects
   - Pass to `agentFlowDiagram.applyServerEvent(data)`
   - Continue parsing until `stream_close` marker

5. **Error handling:**
   - If stream breaks mid-parse, throw with clear message
   - Timeout if stream idle > 30s (tool hang)
   - Preserve existing error codes (401, 403, 504, etc.)

6. **Backward compatibility:**
   - If server doesn't send `content-type: application/stream+json`, fall back to JSON
   - Existing offline/fallback paths unchanged

Example code structure:
```javascript
export async function callMcpTool(tool, params = {}) {
  // ... existing setup ...
  
  const response = await fetch('/api/mcp/tool', fetchOpts);
  
  if (response.headers.get('content-type')?.includes('stream+json')) {
    // HTTP/2 streaming
    const result = await parseStreamingResponse(response.body);
    return result;
  } else {
    // HTTP/1.1 JSON
    return response.json();
  }
}

async function parseStreamingResponse(readableStream) {
  const reader = readableStream.getReader();
  const decoder = new TextDecoder();
  let buffer = '';
  let finalResult = null;
  
  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    
    buffer += decoder.decode(value, { stream: true });
    
    // Split on complete objects
    while (buffer.includes('}')) {
      const endIdx = buffer.indexOf('}') + 1;
      const chunk = buffer.substring(0, endIdx);
      buffer = buffer.substring(endIdx);
      
      try {
        const obj = JSON.parse(chunk);
        if (obj.type === 'flow_event') {
          agentFlowDiagram.applyServerEvent(obj);
        } else if (obj.type === 'result') {
          finalResult = obj.data;
        }
        // stream_close handled by loop termination
      } catch (e) {
        console.warn('Failed to parse stream chunk', e);
      }
    }
  }
  
  return finalResult || { error: 'no result received' };
}
```

Update the POST /api/mcp/tool handler to embed flow events in the streamed response:

1. **Response format:**
   - Set `Content-Type: application/stream+json`
   - Stream flow event objects as newline-delimited JSON
   - Final object: `{"type":"result","data":{...}}`
   - Terminal: `{"type":"stream_close","status":"success"}`

2. **Implementation in handler:**
   ```javascript
   router.post('/api/mcp/tool', authenticateToken, async (req, res) => {
     const { tool, params, flowTraceId } = req.body;
     
     // Setup response as streaming
     res.setHeader('Content-Type', 'application/stream+json; charset=utf-8');
     res.setHeader('Transfer-Encoding', 'chunked');
     
     // Create flow event listener
     const flowHandler = (evt) => {
       res.write(JSON.stringify({ type: 'flow_event', ...evt }) + '\\n');
     };
     mcpFlowSseHub.on(`flow-${flowTraceId}`, flowHandler);
     
     try {
       // Forward to MCP via HTTP/2 bridge
       const session = http2Bridge.createHttp2Session(...);
       const stream = await http2Bridge.forwardToolCall(session, tool, params, flowTraceId);
       
       // Wait for MCP response
       const result = await http2Bridge.handleMcpResponse(stream);
       
       // Send final result in stream
       res.write(JSON.stringify({ type: 'result', data: result }) + '\\n');
       res.write(JSON.stringify({ type: 'stream_close', status: 'success' }) + '\\n');
     } catch (err) {
       res.write(JSON.stringify({ type: 'error', message: err.message }) + '\\n');
     } finally {
       mcpFlowSseHub.off(`flow-${flowTraceId}`, flowHandler);
       res.end();
     }
   });
   ```

   - SSE /api/mcp/tool/events route still works (for debugging/fallback)
