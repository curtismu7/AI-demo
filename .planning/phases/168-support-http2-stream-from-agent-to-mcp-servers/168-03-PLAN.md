<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Stream frame integrity | HTTP/2 uses checksums; tampering detectable |
| Concurrent request isolation | Each request has separate stream ID; no cross-contamination |
| Connection pooling trust | Shared session across multiple requests (same auth context) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-168-09 | Denial of Service | Stream timeout | mitigate | 30s idle timeout; cancel long-running streams; max 5 connections |
| T-168-10 | Tampering | JSON parsing in stream | mitigate | Use native `JSON.parse` (throws on malformed); discard invalid frames |
| T-168-11 | Information Disclosure | Connection reuse across users | mitigate | Connection is per-BFF-process, not per-user; requires common auth token (acceptable for shared service) |

</threat_model>
