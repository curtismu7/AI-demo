  <name>Task 1: Create http2McpBridge service with connection pooling</name>
  <files>banking_api_server/services/http2McpBridge.js</files>
  <action>
Create a new HTTP/2 adapter service that:

1. **Exports interface:**
   ```typescript
   export function createHttp2Session(mcpServerUrl, bearerToken)  // Returns persistent H2 session
   export async function forwardToolCall(session, tool, params, flowTraceId)  // Forward to MCP
   export function handleMcpResponse(stream)  // Stream response back to BFF
   export function closeSession(session)  // Cleanup
   ```

2. **Implementation details:**
   - Use Node.js `http2` module (https://nodejs.org/api/http2.html)
   - Create persistent HTTP/2 connection to MCP server (connection pooling)
   - Store session keyed by `{mcpServerUrl}:{bearerToken}` to reuse across tool calls
   - Forward POST body as HTTP/2 stream with Authorization header
   - Return stream object to caller (for response reading + flow trace SSE)

3. **Headers sent to MCP server:**
   - `Authorization: Bearer {agentToken}`
   - `Content-Type: application/json`
   - `MCP-Session-Id: {uuid}` (MCP spec requirement for session tracking)

4. **Error handling:**
   - Connection timeout → reject with `ECONNREFUSED`
   - 401/403 from MCP → rethrow with `error.code` preserved
   - Stream errors → emit `error` event for caller

5. **Connection pool:**
   - Max 5 concurrent connections to MCP server per process
   - Reuse sessions across tool calls (don't close after each call)
   - Graceful shutdown: drain and close on SIGTERM

  <name>Task 2: Update BFF POST /api/mcp/tool route to use HTTP/2 bridge</name>
  <files>banking_api_server/routes/agent.js</files>
  <action>
Modify the existing `POST /api/mcp/tool` endpoint to use the HTTP/2 bridge instead of WebSocket:

1. **Find the current handler** (likely called `handleMcpTool` or similar):
   - Should extract tool name, params, and flowTraceId from request body
   - Currently proxies via WebSocket to MCP server

2. **Replace WebSocket proxy with HTTP/2 bridge:**
   ```javascript
   const http2Bridge = require('../services/http2McpBridge');
   
   router.post('/api/mcp/tool', authenticateToken, async (req, res) => {
     const { tool, params, flowTraceId } = req.body;
     const agentToken = req.auth.bearerToken; // From middleware
     const mcpServerUrl = process.env.MCP_SERVER_URL || 'http://localhost:8081';
     
     try {
       // Get or create H2 session
       const session = http2Bridge.createHttp2Session(mcpServerUrl, agentToken);
       
       // Forward tool call as HTTP/2 stream
       const stream = await http2Bridge.forwardToolCall(session, tool, params, flowTraceId);
       
       // Pipe stream response back to client (preserves SSE/chunked encoding)
       stream.pipe(res);
       
       // Handle stream errors
       stream.on('error', (err) => {
         if (!res.headersSent) {
           res.status(err.statusCode || 500).json({ error: err.message });
         }
       });
     } catch (err) {
       res.status(err.statusCode || 500).json({ error: err.message, code: err.code });
     }
   });
   ```

3. **Preserve backward compatibility:**
   - Keep SSE events flow unchanged (still use `mcpFlowSseHub` for `/api/mcp/tool/events`)
   - HTTP/2 bridge only affects the direction: Agent → BFF → MCP
   - Fallback: if H2 fails, can still use WebSocket (not required yet, but useful)

4. **Key changes:**
   - Remove WebSocket proxy calls
   - Add http2Bridge import and session creation
   - Pipe stream response directly to HTTP response object
   - Error handling for stream failures

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent → BFF | Agent sends bearer token in Authorization header |
| BFF → MCP | BFF forwards token; controls scope via token exchange |
| MCP → Backend APIs | MCP validates token scopes and act claims |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-168-01 | Tampering | HTTP/2 stream body | mitigate | HTTP/2 enforces frame integrity via checksums; TLS required in production |
| T-168-02 | Spoofing | BFF → MCP connection | mitigate | HTTP/2 TLS certificate validation; fail if cert invalid (`rejectUnauthorized: true`) |
| T-168-03 | Information Disclosure | Token in bearer auth | mitigate | Only send over encrypted HTTP/2 (TLS); never log bearer token value |
| T-168-04 | Denial of Service | Connection pool exhaustion | mitigate | Max 5 concurrent connections per process; timeout on idle streams (30s) |
| T-168-05 | Elevation of Privilege | Agent token scope | mitigate | BFF does NOT modify token; passes as-is to MCP; MCP introspects and validates |

</threat_model>
