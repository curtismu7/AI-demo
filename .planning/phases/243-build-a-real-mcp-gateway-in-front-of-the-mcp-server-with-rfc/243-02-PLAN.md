<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller token -> gateway validator | Untrusted bearer token is checked before any side effects |
| gateway -> PingOne Authorize | External policy decision service boundary |
| gateway -> token endpoint | Security-sensitive exchange for next-hop token |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-243-04 | S | inbound bearer token | mitigate | Validate `aud`, `exp`, `sub`, and actor claims before Authorize or forwarding |
| T-243-05 | E | PingOne Authorize bypass | mitigate | Require Authorize permit before exchange/forwarding; deny or fail closed on unavailable policy path |
| T-243-06 | T | upstream token misuse | mitigate | Exchange every forwarded call for the configured upstream audience and add tests that reject reused wrong-hop tokens |
</threat_model>
