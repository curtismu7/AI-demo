PingOne API endpoint URL pattern:
- Token endpoint (all grant types): `POST https://auth.pingone.{region}/{envId}/as/token`
- Management API: `GET https://api.pingone.{region}/v1/environments/{envId}/applications`
- Management API: `GET https://api.pingone.{region}/v1/environments/{envId}/resources`
- Management API: `GET https://api.pingone.{region}/v1/environments/{envId}/resources/{resourceId}/scopes`
- Management API: `GET https://api.pingone.{region}/v1/environments/{envId}/users`

The actual envId/region are dynamic (from config), so endpoint strings should use placeholder notation
like `POST https://auth.pingone.{region}/{envId}/as/token` (static descriptive strings, not resolved values —
the live URL is already visible in the response JSON from the tracker).

PingOne docs URLs to use:
- Token endpoint (client_credentials): https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token
- Token endpoint (RFC 8693 exchange): https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token
- Worker token (client_credentials): https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-token
- List applications: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-applications
