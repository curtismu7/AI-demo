PingOne MFA API endpoints (for docs links and endpoint display):
- Initiate MFA challenge (create device auth): POST https://api.pingone.{region}/v1/environments/{envId}/users/{userId}/mfaAuthenticators
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-create-mfa-authenticators
- Verify OTP (check device auth): POST https://api.pingone.{region}/v1/environments/{envId}/users/{userId}/mfaAuthenticators/{deviceAuthId}/check
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-check-mfa-authenticator
- List devices: GET https://api.pingone.{region}/v1/environments/{envId}/users/{userId}/devices
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#get-read-all-mfa-devices
- Enroll SMS: POST https://api.pingone.{region}/v1/environments/{envId}/users/{userId}/devices
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-create-mfa-device
- FIDO2 registration: POST https://api.pingone.{region}/v1/environments/{envId}/users/{userId}/fido2Credentials
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-create-fido2-credential

PingOne Authorize decision endpoint:
- POST https://api.pingone.{region}/v1/environments/{envId}/decisionEndpoints/{endpointId}
  Docs: https://apidocs.pingidentity.com/pingone/platform/v1/api/#post-evaluate-decision

Note: The actual URLs come from the live `pingoneRequest.url` field — the endpoint string shown is a
descriptive placeholder. Always prefer `pingoneRequest.url` when available for display.
