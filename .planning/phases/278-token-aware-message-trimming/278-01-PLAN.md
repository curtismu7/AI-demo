<objective>
Replace the hard 100-message count cap in ConversationMemory with token-aware truncation using langchain_core.messages.trim_messages(). Add MAX_CONTEXT_TOKENS to ChatConfig (default 4096 for Ollama compat) and document it in .env.example. The count cap (max_messages_per_session) is retained as a coarse upper bound; trim_messages() runs first to enforce the token budget.

Purpose: 100 messages of banking tool call outputs silently overflow Ollama's context window. Token-aware trimming keeps the most recent messages within the model's context budget.

Output:
- conversation_memory.py with trim_messages() in _trim_session_messages()
- settings.py ChatConfig with max_context_tokens field
- .env.example with LANGCHAIN_MAX_CONTEXT_TOKENS entry
- test_conversation_memory.py with token-aware trimming test
</objective>

async def _trim_session_messages(self, session_id: str) -> None:
