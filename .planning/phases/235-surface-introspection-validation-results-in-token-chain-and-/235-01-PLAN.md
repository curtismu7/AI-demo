1. In the Activity Log — when PingOne was called to validate a token and what it returned
2. In the Token Chain — a visual indicator that the validation mode is introspection (PingOne is actively confirming tokens, not just decoding them locally)
</objective>

## Task 2 — Log introspection events in tokenIntrospectionService.js

**File:** `banking_api_server/services/tokenIntrospectionService.js`

Add import at top:
```js
const { logEvent: logAppEvent } = require('./appEventService');
```

In `validateToken()`:

After `introspectionCache.set(...)` (successful network call, result cached), add:
```js
logAppEvent('introspection', result.valid ? 'info' : 'warning',
  result.valid ? 'Token introspected — PingOne confirmed active' : 'Token introspected — PingOne returned inactive',
  { tag: result.valid ? 'introspection/active' : 'introspection/inactive',
    metadata: { active: result.valid, sub: result.sub || null, client_id: result.client_id || null, scopeCount: result.scopes?.length || 0, scopes: result.scopes || [], exp: result.exp || null } });
```

In the catch block (after the existing logger error call), add:
```js
logAppEvent('introspection', 'error', `Token introspection failed: ${error.message}`,
  { tag: 'introspection/error', metadata: { error: error.message } });
```

Do NOT log on cache hits — too noisy.

## Task 3 — Log introspection events in tokenIntrospection.js middleware

**File:** `banking_api_server/middleware/tokenIntrospection.js`

Add import at top (after existing requires):
```js
const { logEvent: logAppEvent } = require('../services/appEventService');
```

In `tokenIntrospectionMiddleware`, after `req.tokenIntrospection = {...}` is set and before `next()`:
```js
logAppEvent('introspection', 'info', 'Token validated via PingOne introspection (middleware)',
  { tag: 'introspection/middleware-validated',
    metadata: { active: true, sub: introspectionResult.sub || null, path: req.path, scope: introspectionResult.scope || null } });
```

After `logger.warn('Inactive token rejected', ...)` and before `return next(new Error(...))`:
```js
logAppEvent('introspection', 'warning', 'Token rejected — PingOne returned inactive',
  { tag: 'introspection/middleware-inactive',
    metadata: { sub: introspectionResult.sub || null, path: req.path } });
```

In the catch block, after `logger.error(...)`:
```js
logAppEvent('introspection', 'error', `Token introspection middleware failed: ${error.message}`,
  { tag: 'introspection/middleware-error', metadata: { error: error.message, path: req.path } });
```

## Task 5 — Show introspection hint badge in TokenChainDisplay.js

**File:** `banking_api_ui/src/components/TokenChainDisplay.js`

The token chain panel fetches from `/api/token-chain` via `TokenChainContext`. The context already stores `mcpToolCalls`. We need to also store `validationMode` from the response.

**In `TokenChainContext.js`** — the `fetchMCPToolCalls` function already reads `data.mcpToolCallsChain`. Update it to also capture `data.validationMode`:
```js
setMCPToolCalls(data.mcpToolCallsChain || []);
// NEW:
if (data.validationMode) setValidationMode(data.validationMode);
```

Add `const [validationMode, setValidationMode] = useState(null);` state.
Expose `validationMode` in the context value.

**In `TokenChainDisplay.js`** — in the `EventRow` component, add an introspection hint:
```js
const introspectionHint =
  (event.tokenType === 'user_token' || event.id === 'user-token' || event.eventType === 'auth') && validationMode === 'introspection'
    ? { text: '🔬 PingOne verified', cls: 'ok' }
    : null;
```

Then include in the hints row alongside existing hints:
```js
{introspectionHint && <span className={`tcd-event-hint tcd-event-hint--${introspectionHint.cls}`}>{introspectionHint.text}</span>}
```

`validationMode` is passed into EventRow as a prop from the parent.
