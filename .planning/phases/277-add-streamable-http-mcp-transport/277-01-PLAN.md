Add streamable_http as a selectable MCP transport in the Python LangChain agent, alongside the existing WebSocket transport.

Purpose: Align with MCP spec 2025-03-26 which deprecated WebSocket in favour of Streamable HTTP (HTTP POST for client→server; optional SSE stream for server→client). - Unit tests covering initialize handshake and tools/call over HTTP
- Doc comments explaining when to use each transport

Add the StreamableHttpMCPConnection class after MCPConnectionPool (at end of file). It must implement the MCPClient interface (same abstract base used by MCPConnection and LocalMCPConnection):

Class structure:
- __init__(self, server_config: MCPServerConfig, connection_timeout: float = 30.0)
  - self.server_config = server_config
  - self.connection_timeout = connection_timeout
  - self._mcp_session_id: Optional[str] = None
  - self._available_tools: List[str] = []
  - self._tool_schemas: Dict[str, Dict[str, Any]] = {}
  - self._is_connected: bool = False
  - self._http_base_url: str  — the endpoint value as-is (e.g. "http://localhost:8080")
  - self._mcp_url: str — f"{self._http_base_url}/mcp"
  - self._authorization_header: Optional[str] = None  — set from server_config or external caller

- is_connected property: returns self._is_connected

- async connect(self, server_config: MCPServerConfig) — send initialize, capture session id:
  - Build initialize JSON-RPC: {"jsonrpc":"2.0","id":str(uuid4()),"method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{"tools":{"listChanged":False}}}}
  - POST to self._mcp_url; headers: Content-Type: application/json; Authorization: Bearer {self._authorization_header} if set
  - Expect 200; parse JSON-RPC response; extract Mcp-Session-Id from response.headers["mcp-session-id"]
  - Store in self._mcp_session_id; set self._is_connected = True
  - Call self._refresh_tools() after session established
  - On httpx.HTTPError or non-200 status: set self._is_connected = False, raise ConnectionError

- async disconnect(self):
  - If self._mcp_session_id: send DELETE to self._mcp_url with header mcp-session-id: self._mcp_session_id (best-effort, swallow errors)
  - Set self._is_connected = False, self._mcp_session_id = None

- async call_tool(self, tool_call: MCPToolCall) -> Dict[str, Any]:
  - If tool_call.agent_token: set self._authorization_header = tool_call.agent_token.token
  - If not self._is_connected: await self.connect(self.server_config)
  - Build tools/call JSON-RPC: {"jsonrpc":"2.0","id":str(uuid4()),"method":"tools/call","params":{"name":tool_call.tool_name,"arguments":tool_call.parameters}}
  - POST with headers: Content-Type, Authorization, mcp-session-id: self._mcp_session_id, mcp-protocol-version: "2025-03-26"
  - On 404: raise MCPConnectionClosedError (session expired; caller must reconnect)
  - On error in JSON-RPC response: raise Exception with error message
  - Return response["result"]

- async list_tools(self) -> List[str]:
  - If not self._is_connected: await self.connect(self.server_config)
  - Return self._available_tools.copy()

- async get_tool_schema(self, tool_name: str) -> Optional[Dict[str, Any]]:
  - Return self._tool_schemas.get(tool_name)

- async _refresh_tools(self):
  - POST tools/list JSON-RPC; parse result["tools"]; populate self._available_tools and self._tool_schemas

- async _post_rpc(self, message: Dict[str, Any]) -> httpx.Response:
  - Use async with httpx.AsyncClient(timeout=self.connection_timeout) as client: return await client.post(...)
  - Include headers: Content-Type: application/json; mcp-session-id if set; mcp-protocol-version; Authorization if set

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python agent → MCP server HTTP | Agent sends JSON-RPC over plain HTTP POST; bearer token in Authorization header |
| Mcp-Session-Id header | Session token issued by server; if intercepted allows tool calls on agent's behalf |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-277-01 | Information Disclosure | StreamableHttpMCPConnection._authorization_header | mitigate | Token logged only at DEBUG level via masked_fingerprint() pattern; never at INFO. Mirror BL-01 redaction from MCPConnection.call_tool() — redact auth header in log lines. |
| T-277-02 | Tampering | Mcp-Session-Id header | accept | Session ID is server-issued UUID; not a secret (server validates it against in-memory map). Loss of session on 404 triggers reconnect — no data loss. |
| T-277-03 | Spoofing | HTTP endpoint vs WebSocket endpoint misconfiguration | mitigate | HI-04 production guard in get_mcp_server_configs() already rejects plain ws:// in production; HTTP transport must use https:// in production (existing env validation catches this). |
| T-277-04 | Denial of Service | Long-lived httpx.AsyncClient per request | accept | AsyncClient created per-request inside _post_rpc(); timeout enforced by connection_timeout (30s default). Acceptable for demo load. |
| T-277-SC | Tampering | No new npm/pip/cargo installs | accept | httpx already in requirements.txt; no new packages added. Package legitimacy gate not required. |
</threat_model>
