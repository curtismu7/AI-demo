**1. Create `banking_api_ui/src/services/spinnerActivityService.js`:**

A lightweight client-side service that manages polling lifecycle for spinner activity events. Pattern matches spinnerService's imperative singleton approach.

Key design:
- `start()` — begins polling `/api/admin/app-events?since={timestamp}&limit=20` every 2 seconds. Records `sinceTimestamp` as the moment `start()` is called (only show events from this spinner session).
- `stop()` — clears polling interval, resets event buffer.
- `addClientEvent(method, url)` — adds a client-side interceptor event immediately (no polling delay). Called from spinnerService.increment so the in-flight API call appears instantly.
- `subscribe(fn)` / unsubscribe — same pattern as spinnerService. Notifies with array of events `[{ id, icon, timeDelta, message, source }]`.
- `getEvents()` — returns current event buffer.
- On fetch error (401/403 for non-admin), silently stop polling (no retries, no console errors). This makes the service safe for non-admin users.
- Max 20 events in buffer. Newest at bottom.
- Each event object in the feed: `{ id, icon, timeDelta, message, source: 'server'|'client' }`.
- Map categories to icons: oauth→🔑, token_exchange→🔄, session→💾, jwks→🛡️, mcp→🤖, auth_lifecycle→🔐, client→📡.
- `timeDelta` = seconds since `start()` with 1 decimal (e.g., "+0.2s", "+1.5s").
- Use the existing `apiClient` (from `banking_api_ui/src/services/apiClient.js`) for the fetch, but IMPORTANT: the fetch itself must NOT trigger the spinner (would cause infinite loop). Check how apiClient interceptors work — if they always call spinner.increment, then use raw `axios` instance without interceptors for this polling call. Create a private axios instance without interceptors.
