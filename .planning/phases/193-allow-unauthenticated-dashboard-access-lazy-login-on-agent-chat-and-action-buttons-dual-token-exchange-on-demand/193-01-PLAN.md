**App.js routing change:**

Add an explicit `/dashboard` route in the OUTER `<Routes>` block (alongside `/setup/pingone`, `/marketing`, etc.) — BEFORE the `*` catch-all. Place it right after the `/marketing` route block (~line 610). The pattern matches how `/marketing` is handled: render LandingPage wrapper when !user, or full layout when user.

For `/dashboard` when `!user`:
```jsx
<Route
  path="/dashboard"
  element={
    !user ? (
      loading ? null : (
        <>
          <TopNav user={user} onLogout={logout} />
          <main className="main-content">
            <UserDashboard user={null} onLogout={logout} />
          </main>
        </>
      )
    ) : null
  }
/>
```

IMPORTANT: This only handles the `!user` case. When `user` exists, this route element renders `null`, and React Router falls through to the `*` catch-all which already renders UserDashboard at `/dashboard` inside the authenticated block. This avoids duplicating the authenticated rendering logic.

Wait — React Router v6 does NOT fall through from a matched route. Once `/dashboard` matches, it renders that element. So the route must handle BOTH cases:

```jsx
<Route
  path="/dashboard"
  element={
    loading ? null : (
      !user ? (
        <>
          <TopNav user={user} onLogout={logout} />
          <main className="main-content">
            <UserDashboard user={null} onLogout={logout} />
            <WebMcpPanel />
          </main>
        </>
      ) : (
        <>
          <AdminSideNav user={user} />
          <TopNav user={user} onLogout={logout} />
          <main className="main-content">
            <UserDashboard user={user} onLogout={logout} />
            <WebMcpPanel />
          </main>
        </>
      )
    )
  }
/>
```

Place this BEFORE the `*` catch-all route. Then REMOVE the `/dashboard` route from inside the nested authenticated `<Routes>` block (around line 636: `<Route path="/dashboard" element={<><UserDashboard ... /><WebMcpPanel /></>} />`). If you leave both, the outer one wins and the inner one is dead code — cleaner to remove.
