Create new React component at src/components/SidebarWrapper.js that:

1. **Component interface**:
   ```javascript
   export default function SidebarWrapper({
     user,                    // { firstName, lastName, role, expiresAt }
     onLogout,               // () => void
     onThemeToggle,          // () => void
     onTokenInspector,       // () => void
     onSwitchRole,           // (targetRole: string) => Promise
     onAutoRefreshToggle,    // (enabled: boolean) => void
     autoRefreshEnabled,     // boolean
     children,               // ReactNode (main-content element)
   })
   ```

2. **State management**:
   - `collapsed` (boolean) — whether sidebar is hidden on mobile
   - Read from `localStorage.getItem('sidebarCollapsed')` on mount
   - Save to localStorage on change
   - Key: `'sidebarCollapsed'` (for theme persistence across sessions)

3. **Responsive behavior**:
   - Desktop (>768px): Always show sidebar, ignore collapsed state
   - Mobile (<768px): Show hamburger icon, use collapsed state to hide/show
   - Add CSS classes for each state: `.sidebar-wrapper--desktop`, `.sidebar-wrapper--mobile`, `.sidebar-wrapper--collapsed`

4. **Toggle handler**:
   - `toggleSidebarMobile()` function that:
     - Only affects mobile (<768px)
     - Toggles `collapsed` state
     - Saves to localStorage
     - Desktop ignores toggle

5. **Layout structure**:
   - Outer `.sidebar-wrapper` container (grid layout — see Task 2 for CSS)
   - Left: `<Sidebar user={user} onLogout={onLogout} ... />`
   - Right: Pass `children` as-is (the main-content)
   - Mobile: Add hamburger button above sidebar (left-align, only visible <768px)

6. **Hamburger button**:
   - Icon: ☰ (three horizontal bars, or use React Icons MdMenu)
   - Placement: Top-left, above sidebar
   - Only visible on mobile (<768px)
   - Click: Call `toggleSidebarMobile()`
   - Styling: Small, neutral, no shadow initially

7. **No routing — pass handlers to Sidebar**:
   - All navigation wiring happens in Sidebar component
   - SidebarWrapper just manages visibility and layout
   - Props passed through: onLogout, onThemeToggle, onTokenInspector, onSwitchRole, onAutoRefreshToggle, autoRefreshEnabled

  <name>Task 2: Create SidebarWrapper.css with responsive grid layout</name>
  <files>banking_api_ui/src/components/SidebarWrapper.css</files>
  <action>
Create new CSS file at src/components/SidebarWrapper.css with:

1. **Desktop layout** (>768px):
   ```css
   .sidebar-wrapper {
     display: grid;
     grid-template-columns: 240px 1fr;  /* Sidebar 240px, main-content flex */
     gap: 0;
     min-height: 100vh;
   }
   
   .sidebar-wrapper .sidebar {
     grid-column: 1;
     position: relative;
   }
   
   .sidebar-wrapper > * :not(.sidebar) {
     grid-column: 2;
   }
   ```

2. **Mobile layout** (<768px):
   ```css
   .sidebar-wrapper {
     display: block;
     position: relative;
   }
   
   .sidebar-wrapper .sidebar {
     position: fixed;
     left: 0;
     top: 60px;  /* Below topnav */
     width: 100%;
     height: calc(100vh - 60px);
     background: white;
     z-index: 100;
     transform: translateX(-100%);
     transition: transform 0.3s ease;
   }
   
   .sidebar-wrapper.sidebar-wrapper--mobile:not(.sidebar-wrapper--collapsed) .sidebar {
     transform: translateX(0);
   }
   ```

3. **Hamburger button** (mobile only):
   ```css
   .sidebar-wrapper__mobile-toggle {
     display: none;  /* Hidden on desktop */
     @media (max-width: 767px) {
       display: inline-flex;
       align-items: center;
       justify-content: center;
       width: 40px;
       height: 40px;
       background: transparent;
       border: none;
       cursor: pointer;
       position: fixed;
       left: 12px;
       top: 12px;
       z-index: 110;
     }
   }
   ```

4. **Responsive breakpoint media query**:
   - Desktop: `@media (min-width: 768px)` — show full sidebar
   - Mobile: `@media (max-width: 767px)` — hide sidebar by default, show only when not collapsed
   - Use CSS Grid on desktop for proper layout, fixed positioning on mobile overlay

5. **Main-content adjustment**:
   - On desktop: Already takes grid-column: 2 (sits beside sidebar)
   - On mobile: Full width (sidebar is overlay)

6. **Prevent double-scroll**:
   - Sidebar on mobile has `overflow-y: auto` (scrollable)
   - Main content also has `overflow-y: auto`
   - Body/container should NOT have overflow hidden (allow both to scroll independently)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sidebar → Logout | onLogout callback must call actual logout function (clear tokens, call /api/auth/logout) |
| Sidebar → Role Switch | onSwitchRole must validate role on server (not client-only) |
| Sidebar → Theme Toggle | Theme preference should be persisted (localStorage or backend) |
| Layout → Mobile Overlay | Sidebar overlay must not cover critical interactive elements (topnav buttons) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-155-02-01 | Spoofing | Role Switch Handler | Mitigate | Server validates role change; BFF restricts to current user only; no privilege escalation |
| T-155-02-02 | Information Disclosure | Token Inspector | Mitigate | Token modal already exists; ensure new handler respects auth (user can only see their own tokens) |
| T-155-02-03 | Denial of Service | Sidebar Collapse/Expand | Accept | Toggle localStorage only; no API calls; minimal DOM churn |
| T-155-02-04 | Isolation | Mobile Sidebar Overlay | Mitigate | z-index 100+ ensures sidebar doesn't interfere with critical auth flows; modal patterns preserved |

</threat_model>
