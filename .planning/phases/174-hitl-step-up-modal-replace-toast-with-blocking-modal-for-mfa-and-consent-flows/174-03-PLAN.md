<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Passkey credential (user → modal) | Untrusted: WebAuthn response from browser |
| FIDO support detection | Semi-trusted: based on `window.PublicKeyCredential` availability |
| Method selection (user) | Trusted: user chooses OTP or FIDO fallback consciously |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-174-06 | Tampering | Credential response | mitigate | Server validates WebAuthn attestation and signature; client-side serialization only |
| T-174-07 | Spoofing | FIDO support detection | mitigate | Server sends explicit `step_up_method` in response; don't rely on client capability alone |
| T-174-08 | Information Disclosure | Method fallback | mitigate | Log method fallback server-side; don't expose FIDO failure reasons in UI |
| T-174-09 | Denial of Service | WebAuthn timeout | mitigate | 60s timeout with clear user feedback; fallback to OTP always available |

</threat_model>
