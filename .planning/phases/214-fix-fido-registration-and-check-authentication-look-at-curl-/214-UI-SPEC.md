## Component Contract: PingOneCurlCard

### Structure

```
<div class="test-card-raw">
  <button class="test-card-raw-toggle">
    {open ? "▾ Hide curl — {label}" : "▸ Show curl — {label}"}
  </button>
  {open && (
    <pre class="test-card-raw-json test-card-raw-json--curl">
      {curlCommand}
    </pre>
  )}
  {open && copySupported && (
    <button class="test-card-raw-copy">
      Copy curl
    </button>
  )}
</div>
```

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `label` | string | yes | Display label shown in toggle button (e.g. `"Enroll FIDO2 Device"`) |
| `curlCommand` | string or null | yes | Full curl string to display; if null or empty, component renders nothing |

### Behavior

1. Renders nothing when `curlCommand` is falsy (null, empty string, undefined). This handles the "curl context not yet loaded" state — no spinner needed, just absence.
2. Toggle starts closed (`open = false`). User must click to expand.
3. When open, the full curl string is rendered inside a `<pre>` block using React text children — no `dangerouslySetInnerHTML`.
4. `whiteSpace: pre-wrap` and `wordBreak: break-all` on the pre block ensure long URLs and headers wrap rather than overflow horizontally.
5. Copy button is shown only when `open === true` AND `navigator.clipboard` is defined. On click, calls `navigator.clipboard.writeText(curlCommand).catch(() => {})`. Button label is "Copy curl". No success feedback state required.
6. Copy button is placed below the `<pre>` block, not inside it.

### State

```jsx
const [open, setOpen] = useState(false);
const copySupported = typeof navigator !== "undefined" && !!navigator.clipboard;
```

### Token Security

- The `Authorization` header in all curl output MUST read `Bearer $WORKER_TOKEN` — the literal shell-variable string. The dollar sign and variable name are hardcoded in template builder functions, never derived from runtime token values.
- The BFF endpoint `GET /api/mfa/test/curl-context` returns `{ envId, region, userId }`. The worker token is NEVER returned from this endpoint.

## Curl Templates: Required Placeholders

When a dynamic value is not yet available in component state, these exact placeholder strings MUST be used:

| Dynamic Value | Placeholder String |
|---------------|-------------------|
| FIDO2 device ID (pre-enrollment) | `{DEVICE_ID}` |
| SMS/Email device ID (pre-enrollment) | `{DEVICE_ID}` |
| DA ID (challenge, pre-initiation) | `{DA_ID}` |
| User ID (pre-curlCtx load) | `{USER_ID}` |
| Env ID (pre-curlCtx load) | `{ENV_ID}` |

Placeholder curls are still shown before a step runs — this lets developers see the curl structure before executing each step. The component does not suppress itself just because a device ID is unknown.

## CSS Additions

Two new CSS rules are required in `MFATestPage.css`. All other styles reuse existing classes.

### Rule 1: curl pre modifier

```css
.test-card-raw-json--curl {
  white-space: pre-wrap;
  word-break: break-all;
}
```

This overrides the `white-space: pre` from `.test-card-raw-json` to prevent horizontal overflow on long curl lines.

### Rule 2: copy button

```css
.test-card-raw-copy {
  background: none;
  border: 1px solid #6b7280;
  color: #6b7280;
  font-size: 0.75rem;

  margin-top: 0.25rem;
  display: inline-block;

.test-card-raw-copy:hover {

}
```

No other new CSS rules are needed for this phase.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Toggle — closed state | `▸ Show curl — {label}` | `{label}` is the step label from the placement table above |
| Toggle — open state | `▾ Hide curl — {label}` | Matches "▸ Show P1 Response" / "▾ Hide P1 Response" pattern |
| Copy button | `Copy curl` | Verb + noun — no icon, no success animation required |
| Empty / not-yet-loaded state | (nothing rendered) | Component renders null when `curlCommand` is falsy |
| Error state | (none per component) | Curl context load failure is surfaced at section level, not per-curl-card |
| Authorization header placeholder | `Bearer $WORKER_TOKEN` | Literal string — never a real token value |
| Placeholder for unknown IDs | `{DEVICE_ID}`, `{DA_ID}`, `{USER_ID}`, `{ENV_ID}` | Shown until state provides real values |

No destructive actions exist in this phase's UI surface. No confirmation dialogs required.

## Implementation Notes for Executor

1. Define `PingOneCurlCard` as a named function component directly in `MFATestPage.jsx` — same file pattern as `TestCard` and `WhatIsHappening`. No separate file needed.
2. Define curl builder functions (`buildSmsOtpCurl`, `buildFido2InitCurl`, etc.) as module-level pure functions, not inside the component body. Signature: `(curlCtx, ...dynamicIds) => string | null`.
3. Load curl context once in a `useEffect` at component mount. Store in `const [curlCtx, setCurlCtx] = useState(null)`. Call `GET /api/mfa/test/curl-context`.
4. All curl builder functions check `if (!curlCtx?.envId) return null` as their first line — this is the guard that keeps the component hidden until context is ready.
5. Apply the `test-card-raw-json--curl` modifier in addition to `test-card-raw-json`, not instead of it: `className="test-card-raw-json test-card-raw-json--curl"`.
6. Do NOT modify the `TestCard` component's props or internals. `PingOneCurlCard` is a sibling, not a child, of `TestCard`.
7. The Content-Type shown in the `buildFido2CompleteCurl` template must reflect whatever vendor type the FIDO2 fix applies to `mfaService.js` — update the curl template to match the fixed header, not the current broken `application/json`.
