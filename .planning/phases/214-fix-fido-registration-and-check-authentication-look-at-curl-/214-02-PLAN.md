<objective>
Add dynamic PingOne Management API curl display to all MFA test page sections (SMS OTP, Email OTP, FIDO2 enrollment, FIDO2 authentication). Implement: (1) a new BFF route `GET /api/mfa/test/curl-context` that returns `{ envId, region, userId }` for building curl URLs; (2) a `PingOneCurlCard` component in MFATestPage.tsx that renders a collapsible curl command with copy-to-clipboard support; (3) curl builder functions for each MFA section; (4) placement of PingOneCurlCard under each relevant MFATestCard.
</objective>

<must_haves>
  <truths>
    - Worker token must NEVER be returned from the curl-context endpoint or embedded in curl output — always use the literal string `$WORKER_TOKEN`
    - `PingOneCurlCard` is a sibling of `MFATestCard`, not a child — do NOT modify MFATestCard or MFATestPage's existing props
    - MFATestPage.tsx is TypeScript — all new code must be valid TypeScript (proper types for props, state, event handlers)
    - The component file is `demo_api_ui/src/components/MFATestPage.tsx` — NOT `.jsx`. All references in this plan mean the `.tsx` file.
    - `cd demo_api_ui && npm run build` must exit 0 after all UI changes
    - `cd demo_api_server && npm test` must exit 0 after the curl-context route is added
    - No emoji other than ⚠️ ✅ ❌ in UI text
    - `configStore.getEffective()` is the source for `envId` and `region` — do NOT read from env vars directly in the route handler
  </truths>
  <goal_backward>
    Each MFA test section shows a collapsible `▸ Show curl — {label}` card after each TestCard step. The curl shows the actual PingOne API call with real envId, real userId, $WORKER_TOKEN placeholder, and device IDs from state (or `{DEVICE_ID}` placeholder when not yet enrolled).
  </goal_backward>
</must_haves>

      - `buildFido2AuthPollCurl(ctx: CurlCtx | null, daId: string | null): string | null` — GET `https://auth.pingone.${ctx.region}/${ctx.envId}/deviceAuthentications/${daId || '{DA_ID}'}?expand=embedded`
      - `buildFido2AuthVerifyCurl(ctx: CurlCtx | null, daId: string | null): string | null` — POST `https://auth.pingone.${ctx.region}/${ctx.envId}/deviceAuthentications/${daId || '{DA_ID}'}` body `{ assertion: "<base64 WebAuthn assertion>", origin: "<browser origin>", compatibility:"FULL" }` Content-Type: `application/vnd.pingidentity.assertion.check+json`
