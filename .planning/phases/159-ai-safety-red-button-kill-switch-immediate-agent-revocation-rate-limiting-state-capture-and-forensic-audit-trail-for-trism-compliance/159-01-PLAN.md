Create killSwitchService.js (350 lines) exporting:
- killAgent(agentId, reason) → async function calling:
  1. revokeTokenAtPingOne(agentId) — POST /as/revoke with agent's refresh token; PingOne invalidates token
  2. invalidateSessionsInRedis(agentId) — Delete all sessions for agentId from session store (Redis/Upstash)
  3. captureAgentState(agentId) → returns state snapshot object:
     - timestamp, agent_id
     - token info (client_id, scopes, expiry)
     - active_sessions (list of current operations)
     - last_requests (last 50 API requests from this agent)
     - config (rate limits, approved resources, permissions)
     - metrics (request count, errors, rate limit hits in last 5min)
     - recent_actions (transfers, API calls in last 5min)
  4. recordKillEvent(agentId, reason, state, revoke_time_ms) → log via auditLogService
  5. Return { success: true, revoked_at: timestamp, state_snapshot_id, time_to_revoke_ms }

- revokeTokenAtPingOne(agentId) → Function that:
  1. Retrieves agent's refresh token from session store (key: agent:{agentId}:refresh_token)
  2. Makes POST request to PingOne /as/revoke with:
     - token: refresh_token
     - token_type_hint: refresh_token
     - client_id: ADMIN_CLIENT_ID (or management client)
     - client_secret: ADMIN_CLIENT_SECRET
  3. Per oauth-pingone skill: revoke endpoint is at https://auth.pingone.{region}/{envId}/as/revoke
  4. Returns { revoked: true, timestamp }; throws if fails (status !== 200)

- captureAgentState(agentId) → Async function:
  1. Get agent's current token: decode from DB or session (keys: agent:{agentId}:token_*)
  2. Query session store for active sessions matching agentId
  3. Query request log (Redis sorted set or DB) for last 50 requests from agentId
  4. Get agent config from configStore (rate_limit, resources, max_tx, expiry)
  5. Compute metrics from logs (request count, error count, rate limit violations in last 5 min)
  6. Get recent actions from transaction log (transfers, delete operations, read ops in last 5 min)
  7. Sanitize sensitive data (remove PII, account numbers→XXXX-1234)
  8. Return state_snapshot object (typed, not just object)

- invalidateSessionsInRedis(agentId) → Async:
  1. Pattern match: sessions:{agentId}:* in Redis
  2. Delete all matched keys (UNLINK, not DEL, for speed)

- isAgentRevoked(agentId) → Check if agent's token is revoked:
  1. Query session store for agent:{agentId}:token_state
  2. If key doesn't exist → assume revoked (true)
  3. If key exists but has value "revoked" → true
  4. Otherwise false

- Helper: getAgentRefreshToken(agentId) → Get from session store with fallback to DB

Per 159-CONTEXT.md REQ-159-01/02/03: This service must be decoupled (not called from agent code), use PingOne revocation (not local token deletion), and complete within 500ms.

Create agentRateLimit.js middleware (250 lines) exporting:
- agentRateLimitMiddleware(req, res, next) → Express middleware that:
  1. Extract agentId from req.user.client_id (MUST be set by prior OAuth middleware)
  2. If not an agent (no client_id) → call next() (bypass for non-agents)
  3. Get agent's rate limit config from configStore (default: 10 requests per 60 seconds)
  4. Redis operation: INCR redis:agent:{agentId}:requests in current 60-second window
  5. On first request (count=1), SET expiry to 60 seconds
  6. Compare count vs limit:
     - count <= limit → call next()
     - count > limit → call recordRateLimitViolation(agentId, count, limit)
  7. If rate limit exceeded:
     - Check violation threshold (auto-kill trigger): getViolationCount(agentId, '5m') >= 5 violations?
     - If threshold reached → call await killSwitchService.killAgent(agentId, 'Auto-triggered: rate limit violations')
     - Return 429 with JSON: { error: 'agent_killed', message, auto_kill_reason: 'rate_limit_violations', admin_notification: 'Red button auto-triggered...' }
     - Otherwise return 429: { error: 'rate_limit_exceeded', message, current_count, limit, window: '60s' }
  8. Log audit event: recordRateLimitViolation(agentId, count, limit) for each violation

Per 159-CONTEXT.md REQ-159-03/06: Rate limiting is external (not in agent code), cannot be bypassed by agent, and auto-triggers kill at threshold.

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent → BFF | Agent requests BFF to call MCP tools; must be auth'd with agent token |
| BFF → PingOne | BFF sends token revocation requests; must use admin credentials |
| Admin → Kill Switch | Admin UI calls kill switch endpoint; must be admin-authenticated |
| Kill Switch → Session Store | Kill switch invalidates sessions; must have write permission to Redis/DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-159-01 | Spoofing | Agent claims different identity | Mitigate | Token claims validated by existing OAuth middleware; agent_id = client_id from token (immutable) |
| T-159-02 | Tampering | Agent modifies kill switch endpoint | Mitigate | Kill switch logic decoupled from agent code; endpoint in separate admin service; agent cannot reach |
| T-159-03 | Tampering | Agent clears audit log | Mitigate | Audit log is append-only (no DELETE permission for agent credentials); stored in DB with immutability constraints |
| T-159-04 | Tamper | Rate limit middleware disabled by agent | Mitigate | Rate limit middleware runs at BFF layer (before agent handler); agent cannot bypass |
| T-159-05 | Repudiation | Admin denies clicking kill button | Accept | Kill event logged with admin_id + timestamp; sufficient for compliance/audit |
| T-159-06 | Info Disclosure | State snapshot contains sensitive data | Mitigate | sanitizeState() removes PII before storage; account numbers masked (XXXX-1234), customer IDs hashed |
| T-159-07 | Info Disclosure | Audit log accessible to non-admins | Mitigate | getAuditTrail() requires admin role; endpoint access controlled by isAdminUser middleware |
| T-159-08 | Denial of Service | Agent makes N kill-switch requests to exhaust storage | Mitigate | Kill switch endpoint rate-limited (max 1 per agent per 60s); audit log pruning (keep 90 days) |
| T-159-09 | Elevation of Privilege | Non-admin calls kill switch endpoint | Mitigate | isAdminUser middleware checks req.user.role === 'admin'; 403 if not |
| T-159-10 | Information Disclosure | PingOne revocation fails silently | Mitigate | revokeTokenAtPingOne() returns error; killAgent() logs failure + retries 3 times; after 3rd failure, escalate to admin alert |

</threat_model>
