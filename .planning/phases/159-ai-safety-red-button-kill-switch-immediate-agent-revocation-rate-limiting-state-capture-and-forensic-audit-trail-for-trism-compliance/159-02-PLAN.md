<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin UI → Kill Switch Endpoint | Admin sends kill decision; must be authenticated admin |
| Kill Switch Endpoint → Token Revocation | Kill switch passes request to Wave 1 backend; must validate |
| Audit Viewer → Audit Log Retrieval | Admin views audit trail; must have read permission |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-159-11 | Spoofing | Non-admin clicks red button | Mitigate | RedButton only shown if req.user.role === 'admin'; endpoint blocks non-admins with 403 |
| T-159-12 | Tampering | User modifies kill reason in UI | Accept | Reason is display-only customer preference; actual reason logged server-side from request |
| T-159-13 | Tampering | User intercepts onConfirm call | Mitigate | Kill endpoint validates admin auth; non-admins rejected with 403 |
| T-159-14 | Repudiation | Admin modified audit trail | Mitigate | Audit logs immutable backend (appends only, no UPDATE/DELETE); frontend cannot edit |
| T-159-15 | Info Disclosure | Audit trail shows sensitive data | Mitigate | State snapshots sanitized (PII masked, account numbers XXXX); audit endpoint requires admin role |
| T-159-16 | DoS | User makes many confirmation modals | Accept | UI rate-limit: cooldown after kill (disable button 60s), or accept user fatigue prevents spam |

</threat_model>
