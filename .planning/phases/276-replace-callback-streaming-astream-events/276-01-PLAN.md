<objective>
Replace the WebSocketStreamCallbackHandler callback class with a native LangGraph astream_events v2 async streaming loop.

The current implementation attaches a BaseCallbackHandler subclass to AgentExecutor.callbacks. This pattern does not propagate reliably through nested LangGraph sub-graphs and requires manual run_id correlation to match tool_start with tool_end events. LangGraph's astream_events(version="v2") API produces typed StreamEvent dicts (on_tool_start, on_tool_end, on_chat_model_stream) with automatic parent/child run tracking.

Phase 276 delivers the complete migration: langgraph added to requirements, AgentExecutor replaced with create_react_agent compiled graph, callback-based streaming replaced by an async for loop over astream_events, and WebSocketStreamCallbackHandler deleted.

Purpose: Reliable streaming that works with LangGraph's architecture; no run_id bookkeeping; cleaner code surface.
Output: Modified langchain_mcp_agent.py, deleted websocket_stream_callback.py body (file kept as tombstone import guard), updated requirements.txt, passing tests.
</objective>

   response_parts = []
   ws_handler = stream_context.get("websocket_handler") if stream_context else None
   stream_tools = getattr(self.config.langchain, "stream_mcp_tool_events", True)
   stream_tokens = getattr(self.config.langchain, "stream_llm_tokens", True)

   async for event in graph.astream_events(agent_input, config=config, version="v2"):
       event_name = event.get("event")

       if event_name == "on_tool_start" and stream_tools and ws_handler:

           envelope = {
               "type": "stream_event",
               "session_id": session_id,
               "event": "tool_start",
               "tool": tool_name,
               "input": event.get("data", {}).get("input"),
           }
           await ws_handler.send_message_to_session(session_id, envelope)

       elif event_name == "on_tool_end" and stream_tools and ws_handler:
           output = event.get("data", {}).get("output", "")
           output_str = str(output) if not isinstance(output, str) else output
           preview = output_str[:400] + "..." if len(output_str) > 400 else output_str
           envelope = {
               "type": "stream_event",
               "session_id": session_id,
               "event": "tool_end",
               "output_preview": preview,
           }
           await ws_handler.send_message_to_session(session_id, envelope)

       elif event_name == "on_chat_model_stream" and stream_tokens and ws_handler:
           chunk = event.get("data", {}).get("chunk")
           token = getattr(chunk, "content", "") if chunk is not None else ""
           if token:
               envelope = {
                   "type": "stream_event",
                   "session_id": session_id,
                   "event": "llm_token",
                   "token": token,
               }
               await ws_handler.send_message_to_session(session_id, envelope)

       elif event_name == "on_chain_end":
           # Capture final output from the root chain end event
           output = event.get("data", {}).get("output")
           if isinstance(output, dict) and "messages" in output:
               msgs = output["messages"]
               if msgs:
                   last = msgs[-1]
                   content = getattr(last, "content", "")
                   if isinstance(content, str) and content:
                       response_parts.append(content)

   response = response_parts[-1] if response_parts else "I'm sorry, I couldn't process your request."

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| astream_events loop → WebSocket send | Streamed event data (tool names, output previews, LLM tokens) flows from LangGraph into the user's WebSocket channel |
| event["data"] field access | StreamEvent dicts are produced internally by LangGraph — not user-controlled, but defensive access avoids AttributeError on unexpected shapes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-276-01 | Information Disclosure | on_tool_end preview (output[:400]) | accept | Output content originates from MCP tool results already scoped to the authenticated session; 400-char preview is same policy as deleted callback |
| T-276-02 | Information Disclosure | on_chat_model_stream token forwarding | accept | Tokens are LLM output for the authenticated session; no cross-session leakage because ws_handler.send_message_to_session is keyed by session_id bound at connection init |
| T-276-03 | Tampering | astream_events event dict key access | mitigate | Use event.get() with defaults throughout the loop body — never direct dict key access — so a malformed event cannot raise KeyError and bypass the guard |
| T-276-04 | Denial of Service | Unbounded astream_events iteration | mitigate | RunnableConfig(recursion_limit=self.config.langchain.max_iterations) propagated to graph invocation caps loop depth |
| T-276-SC | Tampering | pip install langgraph | mitigate | Verify package at pypi.org/project/langgraph before install; langgraph is a first-party LangChain ecosystem package (langchain-ai GitHub org) |
</threat_model>
