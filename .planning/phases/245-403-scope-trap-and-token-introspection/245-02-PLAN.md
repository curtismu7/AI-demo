<objective>
Consolidate the two parallel token introspection implementations into one authoritative service. The middleware (`tokenIntrospection.js`) currently has its own cache (collision-vulnerable 20-char prefix key) and credential resolution that conflicts with the service. Fix by: (1) removing the middleware's local cache and `introspectToken()` function, delegating all introspection to `tokenIntrospectionService.validateToken()`; (2) adding periodic cache eviction to the service; (3) fixing the type guard and log constant in the service.

Purpose: A single introspection path means revoked tokens can't be active in one cache and expired in another. The SHA-256 cache keys in the service are already correct; the middleware's 20-char prefix is a security issue (session boundary collision risk).

Output: Modified `middleware/tokenIntrospection.js` + `services/tokenIntrospectionService.js`.
</objective>
