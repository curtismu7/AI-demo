Crypto choices (FROZEN per RESEARCH.md):
- AES-256-GCM via `node:crypto` (zero new heavy dep; matches existing configStore.js pattern).
- Argon2id KDF via `argon2@^0.44.0` (m=65536 / t=3 / p=4 / hashLen=32 — OWASP current).
- Per-entry DEK design: master KEK wraps each entry's random DEK; entry values are sealed with the
  DEK. Adding/deleting an entry leaves OTHER entries' ciphertexts byte-identical.
- Whole-file HMAC-SHA256 over canonical JSON (alphabetical key sort) using an HKDF-SHA256 sub-key
  derived from KEK + 'fileHmac/v1'. Catches structural tampering that per-entry GCM tags can't.
- Magic 'BNKV' + integer version 1 file header for future format migrations.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host-fs ↔ vault file | A flipped byte on disk crosses this boundary into our parser |
| operator ↔ CLI process stdin | Untrusted password input (but operator is trusted) |
| process memory ↔ KEK | KEK must not outlive its closure |
| process memory ↔ audit log | Audit log writer must not see decrypted values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-269-01 | Spoofing | Stolen vault file → offline brute-force | mitigate | Argon2id m=65536 t=3 p=4 hashLen=32 (Object.freeze'd in crypto.js); ~100ms per attempt on commodity GPU |
| T-269-02 | Tampering | AEAD nonce reuse | mitigate | Per-entry random DEK + crypto.randomBytes(12) IV per encrypt; assertion in aeadSeal that key.length===32. Nonce space per key is 2^96; never reused. |
| T-269-03 | Tampering | Structural tampering between AEAD'd entries | mitigate | Whole-file HMAC-SHA256 via HKDF sub-key from KEK; verified BEFORE any per-entry AEAD; crypto.timingSafeEqual for compare |
| T-269-04 | Spoofing | Forgotten-password "backdoor" feature creep | mitigate | No-recovery design documented in REQUIREMENTS.md REQ-VAULT-06; no `--recover` / `--admin-reset` flag; CLI prints warning (Plan 02). Code review: any future PR adding such flags must be rejected. |
| T-269-05 | Tampering | Audit log injection / log forging | mitigate | recordAudit takes only structured object; throws on unexpected fields; writes one appendFileSync of `JSON.stringify(...) + '\n'` per call; no format-string interpolation |
| T-269-06 | Information disclosure | Secrets leak via /proc/<pid>/environ | mitigate | This plan: load values into configStore cache via setRaw({persist:false}) — they do not stay in process.env. Plan 03 deletes VAULT_PASSWORD env after open. |
| T-269-07 | Tampering | Accidental git commit of vault file | mitigate | .gitignore adds secrets.vault, secrets.vault.tmp, secrets.vault.audit.log. Encrypted at rest, so commit is recoverable via password rotation. |
| T-269-08 | Information disclosure | KEK lives in memory longer than needed | mitigate | close() fills KEK + every DEK Buffer with 0s. Plan 03 must call close() in `finally` after BFF startup vault read. |
| T-269-09 | Information disclosure | Wrong-password error oracle | mitigate | Generic message 'vault: open failed (bad password or tampered file)' — does not mention argon2/kek/dek; same message for tampered file. Acceptance criterion enforces no leak via grep. |
| T-269-10 | Information disclosure | Decrypted value leaks into audit log | mitigate | audit.js does NOT import crypto.js or format.js (no path to plaintext); recordAudit signature rejects any field other than {op,key,result,caller}; golden fixture grep test asserts no known plaintext appears in log. |
</threat_model>

    - `deriveKek(password, saltBuf)`: `await argon2.hash(password, {...KDF_PARAMS, salt: saltBuf, raw: true})` → returns 32-byte Buffer
    - `aeadSeal(plaintext, key)`: validate `key.length === 32` (throw 'key must be 32 bytes'); `iv = crypto.randomBytes(12)`; cipher = `crypto.createCipheriv('aes-256-gcm', key, iv)`; `ct = Buffer.concat([cipher.update(plaintext), cipher.final()])`; `tag = cipher.getAuthTag()`; return `{iv, tag, ct}`
    - `aeadOpen({iv, tag, ct}, key)`: validate lengths 32/12/16; decipher = `crypto.createDecipheriv('aes-256-gcm', key, iv)`; `decipher.setAuthTag(tag)`; return `Buffer.concat([decipher.update(ct), decipher.final()])` (throws on tag mismatch — DO NOT catch and re-throw with a leaky message)

    'use strict';
    const fs = require('node:fs');
    const os = require('node:os');
    const ALLOWED = new Set(['op','key','result','caller']);
    function recordAudit(filePath, entry) {
      for (const k of Object.keys(entry)) {
        if (!ALLOWED.has(k)) throw new Error('recordAudit: unexpected field ' + k);
      }
      const line = JSON.stringify({
        ts: new Date().toISOString(),
        op: entry.op,
        key: entry.key ?? null,
        pid: process.pid,
        caller: entry.caller,
        host: os.hostname(),
        result: entry.result,
      }) + '\n';
      try { fs.appendFileSync(filePath, line, { flag: 'a' }); }
      catch (err) { console.warn('[vault.audit] write failed:', err.message); }
    }
    module.exports = { recordAudit };

      if (!password) throw new VaultPasswordRequiredError('vault: password required');

      return {

          // (We keep the same Buffer reference so future set()/save() see the rotated key.)
          kek.fill(0);
          newKek.copy(kek);
          newKek.fill(0);
          recordAudit(auditPath(filePath), { op: 'rotate', key: null, caller: 'vault.js', result: 'ok' });
        },
        async save() {
          ensureOpen();
          // Rebuild on-disk entries map; values' ciphertexts are unchanged unless set/delete touched them.
          const outEntries = {};
          for (const [name, e] of entries) {

              valueTag:   e.valueTag.toString('base64'),

              updatedAt:  e.updatedAt,
              ...(e.note ? { note: e.note } : {}),
            };
          }
          envelope.entries = outEntries;
          envelope.fileHmac = computeFileHmac(envelope, hkdfFileHmacKey(kek));
          const tmp = filePath + '.tmp';
          await fsp.writeFile(tmp, canonicalJson(envelope), { mode: 0o600 });
          await fsp.rename(tmp, filePath);
          recordAudit(auditPath(filePath), { op: 'save', key: null, caller: 'vault.js', result: 'ok' });
        },
        close() {
          if (closed) return;
          closed = true;
          kek.fill(0);
          for (const [, e] of entries) e.dek.fill(0);
          entries.clear();

        },
      };
    }
