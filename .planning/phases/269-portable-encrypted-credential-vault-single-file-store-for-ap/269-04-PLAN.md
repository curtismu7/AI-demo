<objective>
Add a vault-load step to the MCP Gateway's startup so the gateway can read its own credentials
(client secret + future AI provider keys) from `secrets.vault` instead of `.env`.

**Scope decision documented (open question 1 from RESEARCH.md "Open Questions"):** Phase 269 wires
the MCP Gateway as an OPTIONAL consumer with these constraints:

1. The gateway has no Helix dependency today (per RESEARCH.md "Summary"). HELIX_API_KEY remains in
   the BFF only (Plan 03 handles it). This plan does NOT add Helix to the gateway.
2. The gateway DOES read its own OAuth credentials (`MCP_GW_CLIENT_ID`, `MCP_GW_CLIENT_SECRET`,
   `BFF_INTERNAL_SECRET`, etc.) — these are the secrets the prompt explicitly wants to move out of
   `.env`. By wiring the vault here, any operator who runs `npm run vault:set MCP_GW_CLIENT_SECRET ...`
   in the BFF (Plan 02 CLI) gets that secret automatically loaded by the gateway on next restart.
3. Future provider keys (`PROVIDER_*` prefix — Anthropic, OpenAI, etc.) are also recognized via the
   allowlist regex `/^(MCP_GW_|PROVIDER_|HELIX_)[A-Z0-9_]+$/`. The HELIX_ prefix is allowed in case a
   future gateway tool needs a Helix key directly; today no such tool exists, so it's just a
   forward-compatibility hook.

**Why this scoping is conservative:**

- The gateway is TypeScript; the vault library is CommonJS. We use a small TS shim file
  (`banking_mcp_gateway/src/vault.ts`) that does the cross-package require with `// @ts-ignore`
  where needed, then re-exports a typed API. This avoids duplicating the crypto code in TS.
- The allowlist prevents a stolen-vault scenario where an attacker could write entries like
  `LD_PRELOAD` or `NODE_OPTIONS` to inject behavior at startup. Only prefixes we OWN are loaded.
- The vault load happens BEFORE `loadConfig()` so the existing `process.env.MCP_GW_*` resolution
  works unchanged — vault values land in process.env, then config.ts reads process.env as before.
  Zero new code paths in config.ts.

**Dependencies:** Plan 01 (vault library), Plan 02 (CLI — operators need the CLI to populate the
vault). Plan 03 is NOT a hard dependency, but runs in parallel; both server.js (Plan 03) and the
gateway (Plan 04) call the same `lib/vault/openVault` function from their respective startups.

Output: gateway loads `MCP_GW_*` / `PROVIDER_*` / `HELIX_*` entries from vault at startup; existing
behavior preserved when no vault file exists; .env.example documents the new env vars.

**Setup-script coverage:** The gateway reads VAULT_PATH and VAULT_PASSWORD from `process.env` —
same contract as the BFF (Plan 03). Plan 05 Task 3 writes VAULT_PATH into the BFF's `.env`; the
gateway's own `.env.example` is updated here with the same variable so operators know the gateway
honors it. `run-bank.sh` already propagates VAULT_PATH/VAULT_PASSWORD via process inheritance —
no shell-wrapper edits needed. Do not edit `banking_api_server/scripts/setupFresh.js` here.
</objective>

    ```typescript
    'use strict';
    /**
     * MCP Gateway vault loader (Phase 269).
     *
     * Reads selected entries from the BFF-side encrypted vault at
     * `secrets.vault` and copies them into `process.env` BEFORE the gateway's
     * existing loadConfig() runs. This lets operators store the gateway's
     * OAuth client secret (MCP_GW_CLIENT_SECRET) and future AI provider keys
     * (PROVIDER_OPENAI_KEY, HELIX_API_KEY, ...) in the vault instead of .env.
     *
     * Allowlist: only entries whose NAME matches /^(MCP_GW_|PROVIDER_|HELIX_|
     * BFF_INTERNAL_)[A-Z0-9_]+$/ are copied. Non-matching entries are logged
     * and skipped. This is critical: a stolen vault file with a entry like
     * `LD_PRELOAD=/evil.so` MUST NOT set process.env.LD_PRELOAD.
     *
     * Vercel: bypassed when VERCEL=1 — consistent with the BFF (Plan 03).
     */
    import { existsSync } from 'node:fs';
    import { resolve, join } from 'node:path';

    // The vault library is CommonJS over in banking_api_server. We require it
    // via a relative path. No TS types exist for it — use @ts-expect-error.
    // eslint-disable-next-line @typescript-eslint/no-var-requires
    // @ts-expect-error — banking_api_server/lib/vault is CJS, no .d.ts
    import vaultLib = require('../../banking_api_server/lib/vault');

    const REPO_ROOT = resolve(__dirname, '..', '..');
    const DEFAULT_VAULT_PATH = join(REPO_ROOT, 'secrets.vault');
    const DEFAULT_ALLOWED = /^(MCP_GW_|PROVIDER_|HELIX_|BFF_INTERNAL_)[A-Z0-9_]+$/;

    export interface VaultLoadResult {
      loaded: boolean;
      entries: number;
      reason?: 'vercel' | 'no_vault_file';
    }

    export interface VaultLoadOpts {
      vaultPath?: string;
      password?: string;
      logger?: Pick<Console, 'log' | 'warn' | 'error'>;
      allowedPrefixes?: RegExp;
      isVercel?: boolean;
    }

    export async function loadVaultIntoEnv(opts: VaultLoadOpts = {}): Promise<VaultLoadResult> {
      const vaultPath = opts.vaultPath ?? process.env.VAULT_PATH ?? DEFAULT_VAULT_PATH;
      const password  = opts.password  ?? process.env.VAULT_PASSWORD;
      const logger    = opts.logger    ?? console;
      const allowed   = opts.allowedPrefixes ?? DEFAULT_ALLOWED;
      const isVercel  = opts.isVercel  ?? (process.env.VERCEL === '1');

      if (isVercel) {
        logger.log('[GW vault] Vercel detected — skipping vault load (use Encrypted Environment Variables)');
        return { loaded: false, entries: 0, reason: 'vercel' };
      }

      if (!existsSync(vaultPath)) {
        logger.log('[GW vault] no vault file at ' + vaultPath + ' — using process.env only');
        return { loaded: false, entries: 0, reason: 'no_vault_file' };
      }

      if (!password) {
        const msg = '[GW vault] secrets.vault exists but VAULT_PASSWORD not set — refusing to start';
        logger.error(msg);
        throw new Error(msg);
      }

      let vault;
      try {
        vault = await vaultLib.openVault(vaultPath, password);
      } catch (err) {
        // Log err.message only (no err.stack — would leak argon2 / kek internal names)
        const e = err as Error;
        logger.error('[GW vault] open failed:', e.message);
        throw err;
      }

      let entryCount = 0;
      try {
        for (const name of vault.list() as string[]) {
          if (!allowed.test(name)) {
            logger.warn('[GW vault] skipping non-allowlisted entry: ' + name);
            continue;
          }
          process.env[name] = vault.read(name);
          entryCount++;
        }
      } finally {
        try { vault.close(); } catch (_) {}
        delete process.env.VAULT_PASSWORD;
      }

      logger.log('[GW vault] loaded ' + entryCount + ' entries from ' + vaultPath);
      return { loaded: true, entries: entryCount };
    }
    ```
