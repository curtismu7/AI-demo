<objective>
Create the Option D agent-facing delegation endpoint (`POST /api/agent/delegate`) at the BFF.

External agent platforms (N8N, AWS Bedrock, Glean) cannot perform RFC 8693 token exchange themselves. This endpoint lets them pre-fetch a delegated token with `act` claim by sending a user Bearer token. They then use the returned token as a standard Bearer header to MCP.

**Purpose:** Enable external agent platform integration without requiring those platforms to implement RFC 8693.

**Output:**
- `banking_api_server/routes/agentDelegation.js` — New route file
- `banking_api_server/server.js` — Route registration
</objective>

<task type="auto">
  <name>Task 1: Create agentDelegation.js route</name>
  <files>banking_api_server/routes/agentDelegation.js</files>
  <action>
Create `banking_api_server/routes/agentDelegation.js` with:

1. **POST /delegate** handler:
   - Extract Bearer token from Authorization header
   - If missing: return HTTP 401 `{ error: 'missing_token' }`
   - Decode JWT payload (no signature verify — BFF trusts PingOne; exchange call will reject invalid tokens)
   - Extract `sub`, `scope`, `exp` from token claims
   - If `req.body.scope` provided: intersect with token's scope (only grant scopes the user actually has)
   - Call `oauthService.performTokenExchangeWithActor(userToken, agentCreds, scope)` — reuse existing function
     - agentCreds from env: `AGENT_OAUTH_CLIENT_ID`, `AGENT_OAUTH_CLIENT_SECRET`
   - On PingOne error (invalid token, expired): return HTTP 401 with PingOne error message
   - On success: return HTTP 200 with `{ access_token, token_type: 'Bearer', expires_in, scope, act }`

2. **Rate limiting:**
   - Use `express-rate-limit` (already a dependency) 
   - Key by decoded `sub` claim (user identity)
   - 10 requests per minute per user
   - On limit: HTTP 429 `{ error: 'rate_limit_exceeded' }`

3. **Audit logging:**
   - Console log (structured JSON): `{ event: 'agent_delegation', sub, act_sub: AGENT_CLIENT_ID, scope, agent_client_id: req.headers['x-agent-client-id'], timestamp }`

4. **Security:**
   - No session cookie required — accepts raw Bearer only
   - Delegated token TTL: use whatever PingOne returns (respects exchange policy)
   - Do NOT log the actual tokens — only claims metadata

**Pattern reference:** Look at `banking_api_server/routes/agentIdentity.js` for route structure patterns used in this BFF.
