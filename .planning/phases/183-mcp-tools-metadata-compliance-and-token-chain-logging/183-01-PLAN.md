<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP Server → Client | Tool metadata is transmitted to untrusted clients; ensure annotations don't leak sensitive implementation details |
| Icon data URIs | SVG embedded as data URIs — no external CSS/image fetch, no XSS vector |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-183-01 | I (Information Disclosure) | annotations field | mitigate | Annotations are operational metadata (readOnly/destructive hints), not user/account data. Safe to transmit to all MPC clients. No PII or credentials in field. |
| T-183-02 | T (Tampering) | icons data URIs | accept | SVG embedded inline, cannot be modified in transit. Data URI cannot be swapped by MITM. Client-side rendering only. |
| T-183-03 | D (Denial of Service) | icons size | mitigate | Each tool icon < 1 KB (SVG data URI). Total 9 tools < 10 KB. Within MCP protocol limits. No risk of payload explosion. |

</threat_model>
