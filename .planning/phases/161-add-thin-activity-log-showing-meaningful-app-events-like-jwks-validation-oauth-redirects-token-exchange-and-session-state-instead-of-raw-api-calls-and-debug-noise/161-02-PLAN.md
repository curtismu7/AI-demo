  <name>Task 1: Instrument OAuth login/callback and session events</name>
  <files>banking_api_server/routes/oauthUser.js, banking_api_server/services/sqliteSessionStore.js</files>
  <action>
**oauthUser.js instrumentation** — Add `const appEventService = require('../services/appEventService');` at the top.

At each existing `console.log('[oauth/user/...]...')` call site, ADD a corresponding `appEventService.logEvent()` call. Do NOT remove existing console.log — keep it for developer terminal output.

Key instrumentation points (per D-01):
1. **Login initiation** `[oauth/user/login]` → `logEvent('auth_lifecycle', 'info', 'User login initiated', { tag: 'oauth/user/login', metadata: { username: loginHint }, flowId })`
2. **PingOne redirect** `Redirecting end user to PingOne Core` → `logEvent('oauth', 'info', 'Redirecting to PingOne authorize', { tag: 'oauth/user/redirect', metadata: { authorizeUrl: truncated }, flowId })`
3. **Callback received** `[oauth/user/callback]` → `logEvent('oauth', 'info', 'OAuth callback received', { tag: 'oauth/user/callback', flowId })`
4. **Token received** `Token received for end user` → `logEvent('token_exchange', 'info', 'User tokens received from PingOne', { tag: 'oauth/user/callback', metadata: { scopes, expiresIn, hasRefreshToken: boolean }, flowId })`
5. **Login success** `End user OAuth login successful` → `logEvent('auth_lifecycle', 'info', 'User login successful', { tag: 'oauth/user/callback', metadata: { username }, flowId })`
6. **Session save failure** `Session save FAILED` → `logEvent('session', 'error', 'Session save failed after OAuth callback', { tag: 'oauth/user/callback', metadata: { error: message }, flowId })`
7. **Step-up auth** `[StepUp]` → `logEvent('oauth', 'info', 'Step-up authentication triggered', { tag: 'StepUp', flowId })`

Generate the `flowId` once at the start of the login route handler using `appEventService.generateFlowId()` and pass it through all related events in that request lifecycle.

**sqliteSessionStore.js instrumentation** — Add `const appEventService = require('./appEventService');` (relative path since it's in services/).

Instrument:
1. **Init** `[sqlite-session-store] ... initialized` → `logEvent('session', 'info', 'SQLite session store initialized', { tag: 'sqlite-session-store' })`
2. **Cleanup error / DBMOVED** `[sqlite-session-store] Cleanup error` → `logEvent('session', 'warning', 'Session store cleanup error: DBMOVED', { tag: 'sqlite-session-store', metadata: { code: error.code } })`
3. **Reconnect** → `logEvent('session', 'info', 'Session store reconnected', { tag: 'sqlite-session-store' })`

Do NOT log every session get/set/destroy — only state changes and errors per D-02 (exclude noise).

<task type="auto">
  <name>Task 2: Instrument token exchange and JWKS events</name>
  <files>banking_api_server/services/oauthService.js, banking_api_server/server.js</files>
  <action>
**oauthService.js instrumentation** — Add `const appEventService = require('./appEventService');`

At each existing `console.log('[TokenExchange...]...')` call site, ADD a corresponding logEvent. Key points:

1. **Token exchange initiated** `[TokenExchange]` → `logEvent('token_exchange', 'info', 'RFC 8693 token exchange initiated', { tag: 'TokenExchange', metadata: { grantType, subjectTokenType, actorTokenType }, flowId })`
2. **Token exchange ID token** `[TokenExchange:ID_TOKEN]` → `logEvent('token_exchange', 'info', 'ID token extracted from exchange', { tag: 'TokenExchange:ID_TOKEN', metadata: { subject, issuer } })`
3. **Actor token exchange** `[TokenExchange+Actor]` → `logEvent('token_exchange', 'info', 'Actor token attached to exchange', { tag: 'TokenExchange+Actor' })`
4. **Agent client credentials** `[AgentClientCredentials]` → `logEvent('token_exchange', 'info', 'Agent client credentials obtained', { tag: 'AgentClientCredentials' })`
5. **MCP exchanger token** `[McpExchangerToken]` → `logEvent('token_exchange', 'info', 'MCP exchanger token obtained', { tag: 'McpExchangerToken' })`
6. **Token refresh** `[TokenRefresh]` → `logEvent('token_exchange', 'info', 'Token refresh performed', { tag: 'TokenRefresh' })`
7. **Token revocation** `[RFC7009]` → `logEvent('token_exchange', 'info', 'Token revoked (RFC 7009)', { tag: 'RFC7009' })`

For exchange events, include sanitized metadata (scopes, token type, expiry) but NEVER include actual token values.

**server.js JWKS instrumentation** — Search for JWKS validation logic (jwks-rsa, jose, or similar). If JWKS validation is in a middleware or utility:
- Cache hit → `logEvent('jwks', 'info', 'JWKS cache hit', { tag: 'jwks', metadata: { kid } })`
- Cache miss / fetch → `logEvent('jwks', 'info', 'JWKS key fetched from PingOne', { tag: 'jwks', metadata: { kid, issuer } })`
- Validation failure → `logEvent('jwks', 'error', 'JWKS validation failed', { tag: 'jwks', metadata: { error } })`

If JWKS validation is not directly in server.js, find where it lives (likely `oauthService.js` or a middleware) and instrument there instead. Note in SUMMARY which file was instrumented.
