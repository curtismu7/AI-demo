<task type="auto">
  <name>Task 1: Create appEventService with ring buffer and category system</name>
  <files>banking_api_server/services/appEventService.js</files>
  <action>
Create `banking_api_server/services/appEventService.js` following the in-memory Map pattern from `tokenChainService.js`. Per D-01 and D-03:

**EVENT_CATEGORIES** enum/object:
- `oauth` — OAuth redirects, authorization, callbacks
- `token_exchange` — RFC 8693 exchanges (subject/actor)
- `session` — Session save/fail/reconnect/DBMOVED
- `jwks` — JWKS validation cache hit/miss, key rotation
- `mcp` — MCP tool calls, results
- `auth_lifecycle` — Login/logout lifecycle events

**EVENT_SEVERITIES:** `info`, `warning`, `error`

**Storage:** Single array ring buffer (not per-user — these are global app events). Default max 500 entries. When full, oldest entries are evicted.

**Each event object shape:**
```js
{
  id: crypto.randomUUID(),
  timestamp: new Date().toISOString(),
  category: string,         // from EVENT_CATEGORIES
  severity: string,         // from EVENT_SEVERITIES
  message: string,          // human-readable summary
  tag: string,              // original [tag] label for traceability
  metadata: object|null,    // optional structured data (token claims excerpt, error details, timing)
  flowId: string|null,      // optional grouping key for related events (e.g., login flow)
  username: string|null     // optional user association
}
```

**Exports:**
- `logEvent(category, severity, message, { tag, metadata, flowId, username })` — adds event to ring buffer
- `getEvents({ category, severity, limit, since })` — returns filtered events array (newest first)
- `getEventsByCategory()` — returns event counts grouped by category
- `clearEvents()` — clears the buffer
- `EVENT_CATEGORIES` and `EVENT_SEVERITIES` constants
- `generateFlowId()` — returns a short random ID for grouping related events in a flow

Do NOT use `console.log` inside this service to avoid recursion if we later hook console. Use direct array operations only.
