## Problem Statement (from user brief)

The floating agent is too messy and UI-complicated. Two specific pain points:

1. **Left rail chip overload** — a 172px scrolling column of chips (suggestions, collapsible action groups, session buttons, server-status chips) competes with the chat area and shrinks it.
2. **Header toolbar overload** — 7 controls visible simultaneously (expand, popout, appearance select, page-theme toggle, sign-out text, token-chain toggle, collapse) clutter every interaction.

Desired outcome: clean chat-first panel; all action chips behind a single "Actions" button; header stripped to only what the user touches in a normal session.

## Redesign Scope

### What is removed from the always-visible UI

| Current Element | Disposition |
|----------------|-------------|
| Left rail (172px chip column) | Removed entirely — no persistent left column in float mode |
| Appearance `<select>` dropdown in header | Moved to actions popout "Settings" section |

| Token chain toggle (🔗) in header | Moved to actions popout "View" section |
| Suggestion chips in left rail | Moved to actions popout "Suggestions" section |
| Action group chips in left rail | Moved to actions popout, grouped (Account / Transaction / Admin / Testing) |
| Session row (Refresh / Sign out) in left rail | Moved to actions popout "Session" section |
| Server-status chips footer | Moved to actions popout bottom status bar |

### What stays in the always-visible header

| Control | Justification |
|---------|---------------|
| Status dot + title + subtitle | Identity/state — always needed |
| "Actions" trigger button | Single entry point to all chips and settings |
| Expand/restore (⊞/⊟) | Layout affordance — no equivalent elsewhere |
| Collapse to FAB (▼) | Essential close affordance |

### What stays in the chat input row

| Control | Justification |
|---------|---------------|
| Text input | Primary interaction surface |
| Send button | Primary CTA |

- Touch target minimum: 28px height for all clickable elements in header.

## Interaction Contracts

### Actions trigger button

- **Default state:** "Actions ▾" pill, accent fill (`#4169e1`), 28px height.
- **Active/open state:** "Actions ▴" pill, darker fill (`#3158d3`), same dimensions. Class `active` on `.ba-actions-trigger`.
- **Hover:** darken to `#3158d3`.
- **Keyboard:** `Enter` / `Space` toggles popout. `Escape` while popout is open closes popout and returns focus to trigger.

### Actions popout

- **Open trigger:** click `.ba-actions-trigger`.
- **Close triggers:** click trigger again, click `.ba-popout-backdrop`, press `Escape`.
- **Escape behavior (two-step):** first `Escape` clears search input (if non-empty); second `Escape` closes popout. Matches existing `discoverySearch` / `showDiscovery` logic.
- **Position:** `position: absolute; bottom: calc(100% + 8px); right: 0` relative to `.ba-header`. The popout appears above the header trigger row.
- **Width:** 320px fixed; matches panel width minus 2 × 16px padding = fits within 380px panel.
- **Max-height:** `min(70vh, 480px)`. Overflow-y: auto. Scrollbar-width: none (matches existing panel scroll pattern).
- **Animation:** `opacity 0 → 1` + `transform: translateY(4px) → translateY(0)` over 0.15s ease. Matches `agent-slide-up` spirit.
- **Chip click:** calls existing `handleActionClick(action.id)`, closes popout, sends message to chat.
- **Suggestion chip click:** injects text into NL input, submits, closes popout.

### Header toolbar (after redesign)

Three controls only, left to right: Actions trigger — Expand toggle — Collapse.

```
[⊞ Actions ▾]                    [⊞] [▼]
```

- Expand toggle and Collapse button retain existing `.ba-icon-btn` styling.
- Actions trigger gets its own class `.ba-actions-trigger` with identical pill dimensions but wider (`padding: 6px 12px` vs square icon).

### Left column in float mode

Hidden via CSS (`display: none`). No animation — the body simply expands to fill the full width. The token chain column (middle column, `.ba-token-chain-col`) is also conditionally hidden when `showTokenChain` is false — this does not change.

### Panel dimensions

No change to existing 380px × 480px defaults. With the left column removed, the chat area gains the 172px that was the left column — approximately 45% more horizontal space for messages.

## States to Implement

| State | Trigger | Visual |
|-------|---------|--------|
| Panel closed | FAB | FAB visible, panel hidden |
| Panel open — default | FAB click | Header + full-width chat + input |
| Actions popout open | Actions button click | Popout overlay above header |
| Actions popout search | Type in popout search | Filtered chip list in real time |
| Chip clicked | Click any action chip | Popout closes; message sent; loading spinner in chat |
| Logged-out | No session | Header subtitle: "Sign in to get started"; Actions popout shows limited chips (no banking actions) |
| Consent blocked | `consentBlocked = true` | Existing consent-denied banner unchanged; Actions button disabled |
| Token chain visible | Toggle in popout | Middle column appears (existing behavior) |
| Expanded mode | ⊞ button | Panel expands to `ba-expanded` class (existing behavior) |
