<objective>
Distinguish "insufficient scope" (valid token, wrong scope) from "no token at all" (needs
auth) in the MCP server's tool-call pipeline. Return JSON-RPC error code -32005 for the
former so the BFF can propagate a clean 403, while preserving the existing auth challenge /
CIBA path for the latter.

Purpose: Currently both cases produce an `authChallenge`, which the BFF silently swallows by
falling back to the local tool handler — effectively bypassing scope enforcement. This plan
makes the server tell the difference.

Output: `authResult.insufficientScope === true` path in MCPMessageHandler → JSON-RPC -32005
with structured `{ tool, requiredScopes, missingScopes, availableScopes }` data.
</objective>
