<objective>
Wire the MCP server's -32005 error through the BFF to the client UI. Three changes:
1. `mcpWebSocketClient.js`: preserve the -32005 code on the rejected Error so the BFF can
   distinguish it from a network/auth error.
2. `server.js POST /api/mcp/tool`: when caught error has `code === 'mcp_insufficient_scope'`,
   return HTTP 403 `{ error: 'mcp_scope_denied', tool, requiredScopes, missingScopes }` instead
   of falling back to the local tool handler.
3. `BankingAgent.js`: detect 403 `mcp_scope_denied` in `handleActionClick` and trigger the
   existing `setScopeErrorModal()`; display a chat message in `handleNaturalLanguage`.

Purpose: Complete the enforcement loop — the scope error the MCP server now emits must reach
the user with a clear explanation of what scope is missing.

Output: scope denial visible in the UI as a modal (chip path) or inline message (NL path).
</objective>
