Purpose: Enable agent to recover from auth failures by obtaining a fresh ID token and performing dual-token exchange per RFC 8693.

Output: Backend route handlers and service methods for 401→auth→exchange flow, tested with verified error handling.
</objective>

   - After auth completes (callback route), check for pending call and retry

5. Per D-01 (Transparent to user): The app frontend should handle redirect transparently (existing pattern; backend just provides redirect URL)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → App BFF | Session cookie (httpOnly) carries user context; app verifies session before exchange |
| App BFF → PingOne | ID token obtained via OIDC; app validates signature (existing middleware) |
| App BFF → MCP Server | MCP token obtained via token exchange; app sends in Authorization header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-186-01 | Tampering | ID token exchange | Mitigate | Validate ID token signature at PingOne JWKS endpoint (existing middleware); verify issuer and audience claims; reject if signature invalid |
| T-186-02 | Information Disclosure | ID token in session | Mitigate | Store ID token in httpOnly session cookie only; never expose to frontend; clear after exchange per D-07 |
| T-186-03 | Information Disclosure | Agent CC token reuse | Accept | Agent CC token is server-side secret; not exposed in exchange responses; standard OAuth practice |
| T-186-04 | Elevation of Privilege | Forged ID token | Mitigate | PingOne signature validation gates this; forged token rejected during exchange validation |
| T-186-05 | Denial of Service | Repeated 401 → auth cycles | Mitigate | Implement rate limiting on OIDC auth endpoint (existing); fail gracefully if auth limit exceeded |
| T-186-06 | Repudiation | Token exchange without audit | Mitigate | Log token exchange attempt (endpoint, user ID, timestamp) to audit trail; tie to agent request ID |

</threat_model>
