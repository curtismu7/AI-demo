# Sketch 001: Phase 246 — Floating Agent Redesign

## Design Question
Does moving all chips and controls into an Actions popout, while stripping the header down to 3 controls, give the clean chat-first experience the brief asks for?

## How to View
```
open .planning/sketches/001-phase-246-agent-redesign/index.html
```

## Variants
- **A: Before vs After** — Side-by-side comparison of the 7-control + left-column layout vs the redesigned 3-control + full-width chat layout.
- **B: Interactive Prototype** — Fully interactive mockup: open/close popout, type in search to filter chips, click any chip to send it, expand panel, Escape to close, TESTING section collapsible.
- **C: Popout Open States** — Three static snapshots: popout closed, popout open (all sections), search active ("trans").

## What to Look For
- Does the full-width chat area feel noticeably better for reading messages?
- Does the Actions pill button feel discoverable enough?
- Is the popout easy to scan? Is the section order (Suggestions → Account → Transaction → Testing → Session → View → Settings → Status) logical?
- Does the TESTING section being collapsed by default feel right?
- Does ⧉ New window (vs the old ↗ in the header) feel acceptable in the VIEW section?
- Are there any sections or controls you'd order differently or remove?
